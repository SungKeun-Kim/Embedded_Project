# Modbus RTU 통신 프로토콜 사양서

**적용 장치**: STM32G474RE 메가소닉 발진기 메인보드  
**프로토콜**: Modbus RTU (표준 준거)  
**문서 버전**: 2.0  
**최종 수정**: 2026-03-23

> **참고**: 본 문서는 기존 HS AUP Multi Generator 매뉴얼의 RS-422 ASCII/BCC 통신 프로토콜을
> RS-485 Modbus RTU로 현대화한 사양입니다. 기존 매뉴얼의 모든 명령(W, STA, VER, C)에
> 대응하는 Modbus 레지스터를 제공합니다.

---

## 1. 물리 계층 (Physical Layer)

### 1.1 하드웨어 인터페이스

| 항목         | 사양                                        |
| ------------ | ------------------------------------------- |
| 통신 규격    | RS-485 반이중 (Half-Duplex)                 |
| 트랜시버     | MAX3485 / SP3485 (3.3V 동작)                |
| 커넥터       | 2선식 (A, B) + GND                          |
| 종단 저항    | 120Ω (필요 시 보드 내 점퍼/저항 장착)       |
| 최대 노드 수 | 32 (표준), 256 (고입력 임피던스 IC 사용 시) |
| 최대 케이블  | 1,200 m (9600 bps 기준)                     |

> **기존 매뉴얼과의 차이**: RS-422 (D-SUB 9핀, 4선식 전이중) → RS-485 (2선식 반이중)로 변경.
> 기존 매뉴얼의 D-SUB 9핀 커넥터(1:Tx+, 2:Tx-, 3:Rx+, 4:Rx-)는 RS-485 A/B 2선으로 대체.

### 1.2 UART 설정

| 항목      | 기본값        | 설정 범위                        |
| --------- | ------------- | -------------------------------- |
| 통신 속도 | 9600 bps      | 9600 / 19200 / 38400 / 115200    |
| 데이터    | 8 bits        | 고정                             |
| 패리티    | None          | None / Even / Odd                |
| 정지 비트 | 2 bits (N 시) | N→2비트, E/O→1비트 (Modbus 규격) |

> **기존 매뉴얼 통신 사양 대비**:
>
> - 기존: 9600 bps 고정, 7비트 데이터, Even 패리티, 1 Stop, ASCII 코드 + BCC
> - 변경: 9600 bps 기본(변경 가능), 8비트 데이터, None/Even/Odd 선택, Modbus RTU

### 1.3 RS-485 방향 제어 핀

| 핀  | 포트        | 기능                            |
| --- | ----------- | ------------------------------- |
| PA2 | USART2_TX   | RS-485 TXD                      |
| PA3 | USART2_RX   | RS-485 RXD                      |
| PC13 | GPIO_Output | DE (Driver Enable, HIGH=송신)   |
| PC14 | GPIO_Output | /RE (Receiver Enable, LOW=수신) |

**방향 전환 타이밍:**

```
대기/수신: DE=LOW,  /RE=LOW   → RXNE 인터럽트로 바이트 수신
송신 시작: DE=HIGH, /RE=HIGH  → 데이터 전송
송신 완료: TC 플래그 확인 후 DE=LOW, /RE=LOW 복귀 (필수!)
```

## 2. 데이터 링크 계층 (Modbus RTU 프레임)

### 2.1 프레임 구조

표준 Modbus RTU 프레임 형식을 따릅니다.

```
┌──────────┬──────────┬────────────────┬───────────┐
│ 슬레이브 │ 펑션코드 │   데이터 필드  │  CRC-16   │
│ 주소     │          │  (가변 길이)   │           │
├──────────┼──────────┼────────────────┼───────────┤
│ 1 byte   │ 1 byte   │  N bytes       │ 2 bytes   │
└──────────┴──────────┴────────────────┴───────────┘
```

> **기존 ASCII 프로토콜과의 핵심 차이점:**
>
> - **인코딩**: ASCII(7비트+BCC) → **바이너리 RTU**
> - **에러 체크**: BCC (짝수 수평 패리티) → **CRC-16/Modbus** (다항식 0xA001)
> - **프레임 구분**: CR(0x0D) 딜리미터 → **3.5 캐릭터 타임 묵음 구간**
> - **바이트 순서**: Modbus RTU는 **Big Endian** (MSB 우선)

### 2.2 슬레이브 주소

| 주소    | 용도                             |
| ------- | -------------------------------- |
| 0       | 브로드캐스트 (응답 없음, 쓰기만) |
| 1–247   | 개별 장치 주소 (기본값: 1)       |
| 248–255 | 예약                             |

> **기존 매뉴얼 대비**: 기존 ADDRESS 0~E(0~14)를 Modbus 주소 1~15로 매핑.
> 기존 ID=0 → Modbus 주소 1, 기존 ID=E(14) → Modbus 주소 15.

### 2.3 프레임 간 묵음 구간

| 통신 속도  | 3.5 캐릭터 시간 | 1.5 캐릭터 시간 |
| ---------- | --------------- | --------------- |
| 9600 bps   | 4.01 ms         | 1.72 ms         |
| 19200 bps  | 2.01 ms         | 0.86 ms         |
| 38400 bps  | 1.75 ms (최소)  | 0.75 ms (최소)  |
| 115200 bps | 1.75 ms (최소)  | 0.75 ms (최소)  |

> 19200 bps 이상에서는 타이머 기반 고정 시간(1.75 ms / 0.75 ms)을 사용합니다 (Modbus 표준 권고).

### 2.4 CRC-16 계산

Modbus RTU 표준 CRC-16을 사용합니다.

- **다항식**: 0xA001 (반전 표현), 원래 다항식 x^16+x^15+x^2+1
- **초기값**: 0xFFFF
- **바이트 순서**: CRC Low byte 먼저, CRC High byte 나중 (LSB-first)

```c
/* CRC-16/Modbus 테이블 룩업 방식 (256 엔트리) */
uint16_t Modbus_CRC16(const uint8_t *data, uint16_t length)
{
    uint16_t crc = 0xFFFF;
    while (length--) {
        uint8_t index = (uint8_t)(crc ^ *data++);
        crc = (crc >> 8) ^ crc16_table[index];
    }
    return crc;  /* CRC Low 먼저, CRC High 나중에 전송 */
}
```

## 3. 지원 펑션 코드 (Function Codes)

### 3.1 펑션 코드 목록

| 코드 | 명칭                     | 설명                     | 기존 ASCII 명령 대응 |
| ---- | ------------------------ | ------------------------ | -------------------- |
| 0x03 | Read Holding Registers   | 1개 이상의 레지스터 읽기 | STA, VER 명령        |
| 0x06 | Write Single Register    | 단일 레지스터 쓰기       | W, C 명령            |
| 0x10 | Write Multiple Registers | 복수 레지스터 연속 쓰기  | (신규 확장)          |

### 3.2 FC 0x03 — Read Holding Registers (레지스터 읽기)

연속된 1개 이상의 Holding Register(16비트)를 읽습니다.

**요청 프레임:**

```
┌───────┬───────┬─────────────┬─────────────┬────────┐
│ 주소  │ 0x03  │ 시작 레지스터│ 레지스터 수 │ CRC-16 │
│ 1byte │ 1byte │ 2bytes (MSB)│ 2bytes (MSB)│ 2bytes │
└───────┴───────┴─────────────┴─────────────┴────────┘
```

**정상 응답 프레임:**

```
┌───────┬───────┬───────────┬───────────────────┬────────┐
│ 주소  │ 0x03  │ 바이트 수 │ 레지스터 데이터들  │ CRC-16 │
│ 1byte │ 1byte │ 1byte     │ N×2 bytes (MSB)   │ 2bytes │
└───────┴───────┴───────────┴───────────────────┴────────┘
```

**예시 — STA 명령 대응: 운전 상태 일괄 읽기 (0x0000~0x0009, 10개):**

```
요청: [01] [03] [00 00] [00 0A] [C5 CD]
응답: [01] [03] [14] [00 01] [00 32] [00 30] [00 00] [13 88] [00 01] [00 00] [00 01] [00 00] [00 03] [CRC CRC]
→ 0x0000=1(발진ON), 0x0001=50(0.50W), 0x0002=48(0.48W), 0x0003=0(Ch0),
   0x0004=5000(500.0kHz), 0x0005=플래그, 0x0006=0(NORMAL), 0x0007=1(주소1),
   0x0008=0(로컬허용), 0x0009=외부입력상태
```

### 3.3 FC 0x06 — Write Single Register (단일 레지스터 쓰기)

단일 Holding Register에 값을 기록합니다.

**요청 프레임:**

```
┌───────┬───────┬─────────────┬─────────────┬────────┐
│ 주소  │ 0x06  │ 레지스터주소│ 기록할 값   │ CRC-16 │
│ 1byte │ 1byte │ 2bytes (MSB)│ 2bytes (MSB)│ 2bytes │
└───────┴───────┴─────────────┴─────────────┴────────┘
```

**정상 응답**: 요청 프레임과 동일한 내용을 그대로 에코합니다.

**예시 — W 명령 대응: 출력 0.70W 설정 (레지스터 0x0001, 값 70):**

```
요청: [01] [06] [00 01] [00 46] [59 DF]
응답: [01] [06] [00 01] [00 46] [59 DF]  ← 동일 에코
→ 값 70 = 0.70W (×0.01W)
```

**예시 — C 명령 대응: 주파수 채널 3으로 변경 (레지스터 0x0003, 값 3):**

```
요청: [01] [06] [00 03] [00 03] [79 CB]
응답: [01] [06] [00 03] [00 03] [79 CB]
→ Ch3 선택 (750.0 kHz)
⚠ 채널 변경 시 발진 1초 정지 후 새 주파수로 재개
```

### 3.4 FC 0x10 — Write Multiple Registers (복수 레지스터 쓰기)

연속된 여러 Holding Register에 한 번에 값을 기록합니다.

**요청 프레임:**

```
┌───────┬───────┬─────────────┬─────────────┬───────────┬────────────────┬────────┐
│ 주소  │ 0x10  │ 시작 레지스터│ 레지스터 수 │ 바이트 수 │ 레지스터 값들  │ CRC-16 │
│ 1byte │ 1byte │ 2bytes      │ 2bytes      │ 1byte     │ N×2 bytes      │ 2bytes │
└───────┴───────┴─────────────┴─────────────┴───────────┴────────────────┴────────┘
```

**정상 응답:**

```
┌───────┬───────┬─────────────┬─────────────┬────────┐
│ 주소  │ 0x10  │ 시작 레지스터│ 레지스터 수 │ CRC-16 │
│ 1byte │ 1byte │ 2bytes      │ 2bytes      │ 2bytes │
└───────┴───────┴─────────────┴─────────────┴────────┘
```

**예시 — 8POWER Step 1~3 출력값 일괄 설정 (0x0021~0x0023):**

```
요청: [01] [10] [00 21] [00 03] [06] [00 1E] [00 28] [00 32] [CRC CRC]
→ Step1=0.30W, Step2=0.40W, Step3=0.50W 동시 기록
```

### 3.5 예외 응답 (Exception Response)

요청이 유효하지 않을 경우, 펑션코드의 최상위 비트(MSB)에 0x80을 OR하여 에러 응답합니다.

```
┌───────┬──────────────┬───────────┬────────┐
│ 주소  │ FC + 0x80    │ 예외 코드 │ CRC-16 │
│ 1byte │ 1byte        │ 1byte     │ 2bytes │
└───────┴──────────────┴───────────┴────────┘
```

| 예외 코드 | 명칭                 | 설명                            | 기존 ASCII 에러 대응 |
| --------- | -------------------- | ------------------------------- | -------------------- |
| 0x01      | Illegal Function     | 지원하지 않는 펑션코드          | COMERR               |
| 0x02      | Illegal Data Address | 존재하지 않는 레지스터 주소     | COMERR, NUMERR       |
| 0x03      | Illegal Data Value   | 허용 범위를 벗어난 값           | NUMERR               |
| 0x04      | Slave Device Failure | 장치 내부 오류로 요청 처리 불가 | —                    |

> **기존 ASCII 에러 대응**: PARERR→CRC불일치(무응답), FRMERR→프레임에러(무응답),
> OVRERR→오버런(무응답), COMERR→0x01/0x02, NUMERR→0x03, BCCERR→CRC불일치(무응답).

> **접근 제한**: NORMAL/REMOTE 모드에서 쓰기(FC06/10) 요청 시 예외 응답 0x04 반환.
> EXT 모드에서만 쓰기 가능 (매뉴얼 준수).

> **무응답 상황**: CRC 불일치, 주소 불일치, 프레임 훼손 등의 경우 어떤 응답도 하지 않습니다.
> 마스터는 타임아웃으로 통신 에러를 판단합니다.

## 4. 레지스터 맵 (Register Map)

레지스터 주소는 기능별로 **16개(0x10) 단위 블록**으로 정렬하여 그룹 내 연속 읽기(`FC 0x03`)에 최적화하였습니다.

### 4.1 레지스터 맵 전체 요약

#### 그룹 0x00: 운전 제어 (Operational Control) — 10개 연속 읽기 가능 (STA 명령 대응)

| 주소   | R/W | 명칭                | 단위/범위                            | 기존 ASCII 대응 |
| ------ | --- | ------------------- | ------------------------------------ | --------------- |
| 0x0000 | R/W | 동작 ON/OFF         | 0=OFF, 1=ON                          | STA: Rx 필드    |
| 0x0001 | R/W | 출력 전력 설정값    | ×0.01 W (20~100 → 0.20~1.00W)        | W command       |
| 0x0002 | R   | 현재 실측 출력 전력 | ×0.01 W                              | STA: Mxxx 필드  |
| 0x0003 | R/W | 주파수 채널 번호    | 0~6 (Ch0~Ch6)                        | C command       |
| 0x0004 | R   | 현재 주파수         | ×0.1 kHz (예: 5000=500.0kHz)         | STA: Fx 필드    |
| 0x0005 | R   | 동작 상태 플래그    | 비트 필드 (§4.2 참조)                | STA 종합        |
| 0x0006 | R/W | 동작 모드           | 0=NORMAL, 1=REMOTE, 2=EXT            | STA: Ex 필드    |
| 0x0007 | R/W | 슬레이브 주소       | 1~247 (기존 0~14 → 1~15 매핑)        | [ID] 필드       |
| 0x0008 | R/W | 로컬 입력 금지      | 0=허용, 1=금지                       | —               |
| 0x0009 | R   | 외부 입력 상태      | bit0=REMOTE, bit1=SENSOR, bit2~4=BCD | STA: Rx,Sx 필드 |

#### 그룹 0x10: 알람 및 진단 (Alarm & Diagnostics) — 8개 연속 읽기 가능

| 주소   | R/W | 명칭             | 단위/범위                                 | 기존 ASCII 대응 |
| ------ | --- | ---------------- | ----------------------------------------- | --------------- |
| 0x0010 | R/W | HIGH 알람 설정값 | ×0.01 W (25~150 → 0.25~1.50W)             | STA: Wx 필드    |
| 0x0011 | R/W | LOW 알람 설정값  | ×0.01 W (0~95 → 0.00~0.95W)               | STA: Wx 필드    |
| 0x0012 | R   | 에러 상태 코드   | 비트 필드 (§4.3 참조)                     | STA: Wx 필드    |
| 0x0013 | W   | 에러 리셋        | 0x0001 기록 시 에러 클리어                | —               |
| 0x0014 | R   | 출력 전압        | 0~9 (9단계, 매뉴얼 Vx 대응)               | STA: Vx 필드    |
| 0x0015 | R   | 출력 전류        | ×1 mA (0~999)                             | STA: Axxx 필드  |
| 0x0016 | R   | 임피던스         | ×1 Ω (50~999)                             | STA: Oxxx 필드  |
| 0x0017 | R   | 펌웨어 버전      | 상위=Major, 하위=Minor (예: 0x0100=v1.00) | VER command     |

#### 그룹 0x20: 8 POWER Step 0~7 출력 설정값

| 주소   | R/W | 명칭                  | 단위/범위                       |
| ------ | --- | --------------------- | ------------------------------- |
| 0x0020 | R/W | Step 0 출력 전력 설정 | ×0.01 W (= NORMAL설정값과 동일) |
| 0x0021 | R/W | Step 1 출력 전력 설정 | ×0.01 W (20~100)                |
| 0x0022 | R/W | Step 2 출력 전력 설정 | ×0.01 W (20~100)                |
| 0x0023 | R/W | Step 3 출력 전력 설정 | ×0.01 W (20~100)                |
| 0x0024 | R/W | Step 4 출력 전력 설정 | ×0.01 W (20~100)                |
| 0x0025 | R/W | Step 5 출력 전력 설정 | ×0.01 W (20~100)                |
| 0x0026 | R/W | Step 6 출력 전력 설정 | ×0.01 W (20~100)                |
| 0x0027 | R/W | Step 7 출력 전력 설정 | ×0.01 W (20~100)                |

> **Step 0**: NORMAL 모드 출력값(0x0001)과 자동 연동. 0x0020 기록 시 0x0001도 동시 갱신.

#### 그룹 0x30: 8 POWER Step 0~7 HIGH 알람 설정값

| 주소          | R/W | 명칭               | 단위/범위                          |
| ------------- | --- | ------------------ | ---------------------------------- |
| 0x0030        | R/W | Step 0 HIGH 알람   | ×0.01 W (= NORMAL HIGH알람과 동일) |
| 0x0031~0x0037 | R/W | Step 1~7 HIGH 알람 | ×0.01 W (25~150)                   |

#### 그룹 0x40: 8 POWER Step 0~7 LOW 알람 설정값

| 주소          | R/W | 명칭              | 단위/범위                         |
| ------------- | --- | ----------------- | --------------------------------- |
| 0x0040        | R/W | Step 0 LOW 알람   | ×0.01 W (= NORMAL LOW알람과 동일) |
| 0x0041~0x0047 | R/W | Step 1~7 LOW 알람 | ×0.01 W (0~95)                    |

#### 그룹 0x50: 주파수 채널 테이블 (7개, 500kHz~2MHz, 250kHz 단위)

| 주소   | R/W | 명칭        | 기본값 (×0.1 kHz) | 실제 주파수 |
| ------ | --- | ----------- | ----------------- | ----------- |
| 0x0050 | R/W | Ch 0 주파수 | 5000              | 500.0 kHz   |
| 0x0051 | R/W | Ch 1 주파수 | 7500              | 750.0 kHz   |
| 0x0052 | R/W | Ch 2 주파수 | 10000             | 1,000.0 kHz |
| 0x0053 | R/W | Ch 3 주파수 | 12500             | 1,250.0 kHz |
| 0x0054 | R/W | Ch 4 주파수 | 15000             | 1,500.0 kHz |
| 0x0055 | R/W | Ch 5 주파수 | 17500             | 1,750.0 kHz |
| 0x0056 | R/W | Ch 6 주파수 | 20000             | 2,000.0 kHz |

> **주파수 범위**: 500.0~2000.0 kHz (×0.1 kHz 단위, 값 5000~20000).
> **주파수 변경 시**: 발진 1초 정지 후 새 주파수로 재개 (매뉴얼 준수).

#### 그룹 0x60: 통신 설정 (Communication)

| 주소   | R/W | 명칭                 | 단위/범위                          |
| ------ | --- | -------------------- | ---------------------------------- |
| 0x0060 | R/W | Modbus 슬레이브 주소 | 1~247                              |
| 0x0061 | R/W | 통신 속도 인덱스     | 0=9600, 1=19200, 2=38400, 3=115200 |
| 0x0062 | R/W | 패리티 설정          | 0=None, 1=Even, 2=Odd              |

> **통신 설정 변경 적용**: 0x0060~0x0062 기록 후 **다음 전원 리셋 시** 적용됨.
> 즉시 적용이 필요하면 0x00FF에 0x1234를 기록하여 소프트 리셋 수행.

#### 시스템 명령 (System Command)

| 주소   | R/W | 명칭        | 단위/범위                  |
| ------ | --- | ----------- | -------------------------- |
| 0x00FF | W   | 시스템 리셋 | 0x1234 기록 시 소프트 리셋 |

### 4.2 동작 상태 플래그 (0x0005) 비트 정의

기존 매뉴얼의 STA 명령 응답 필드(Rx, Ex, Sx, Wx, Fx)를 하나의 비트 필드로 통합합니다.

| 비트  | 명칭         | 설명                   | 기존 STA 필드  |
| ----- | ------------ | ---------------------- | -------------- |
| 0     | RUN          | 발진 동작 중           | Rx (R1=운전중) |
| 1     | REMOTE       | REMOTE 입력 SHORT 상태 | Rx             |
| 2     | EXT_MODE     | EXT 모드 활성          | Ex (E1=EXT)    |
| 3     | SENSOR_OK    | SENSOR 정상 (SHORT)    | Sx (S1=정상)   |
| 4     | HIGH_ALARM   | 출력 HIGH 알람 상태    | Wx (W1=HIGH)   |
| 5     | LOW_ALARM    | 출력 LOW 알람 상태     | Wx (W2=LOW)    |
| 6     | TRANS_ALARM  | 진동자 알람 상태       | —              |
| 7     | POWER_8STEP  | 8POWER 모드 활성       | —              |
| 8     | FREQ_DISPLAY | 주파수 표시 중         | —              |
| 9     | COMM_ACTIVE  | RS-485 통신 활성       | —              |
| 10    | SOFT_START   | 소프트 스타트 진행 중  | —              |
| 11~15 | (예약)       | 향후 확장용 (0 고정)   | —              |

> **💡 활용 팁**: `FC 0x03`으로 0x0000~0x0009 (10개)를 한 번에 읽으면 기존 STA 명령과 동일한
> 정보를 모두 얻을 수 있습니다. 기존 ASCII STA 명령의 1회 통신을 Modbus FC03 1회로 대체.

### 4.3 에러 상태 코드 (0x0012) 비트 정의

매뉴얼의 에러 코드 체계를 비트 필드로 표현합니다.

| 비트 | 에러 코드 | 명칭               | 내용                                |
| ---- | --------- | ------------------ | ----------------------------------- |
| 0    | Err1      | LOW ALARM          | 출력 < LOW 알람값, 5초 유지 → 정지  |
| 1    | Err2      | HI ALARM           | 출력 > HIGH 알람값, 5초 유지 → 정지 |
| 2    | Err5      | TRANSDUCER ALARM   | 진동자 과부하 5초 유지 → 정지       |
| 3    | Err6      | SETTING ERROR      | 설정 불가능한 값 입력 시도          |
| 4    | Err7      | SENSOR ALARM (OFF) | SENSOR 단자 OPEN (정지 중 발생)     |
| 5    | Err8      | SENSOR ALARM (RUN) | SENSOR 단자 OPEN (발진 중 발생)     |
| 6~15 | (예약)    |                    | 향후 확장용 (0 고정)                |

> **에러 리셋**: 0x0013에 0x0001 기록 시 에러 클리어 (Err7 제외).
> Err7은 SENSOR 단자 SHORT 후 전원 재투입으로만 해제 가능.

### 4.4 외부 입력 상태 (0x0009) 비트 정의

| 비트 | 명칭   | 설명                                    |
| ---- | ------ | --------------------------------------- |
| 0    | REMOTE | REMOTE 입력 상태 (0=OPEN, 1=SHORT)      |
| 1    | SENSOR | SENSOR 입력 상태 (0=OPEN/에러, 1=SHORT) |
| 2    | BCD_B1 | 8POWER BCD BIT1 (0=OPEN, 1=SHORT)       |
| 3    | BCD_B2 | 8POWER BCD BIT2                         |
| 4    | BCD_B3 | 8POWER BCD BIT3                         |
| 5~15 | (예약) | 0 고정                                  |

## 5. 기존 ASCII 명령 → Modbus RTU 전환 가이드

### 5.1 명령별 전환 매핑

| 기존 ASCII 명령 | 기능             | Modbus RTU 전환                              |
| --------------- | ---------------- | -------------------------------------------- |
| **W command**   | 출력 전력 설정   | FC 0x06, reg=0x0001, val=전력값(×0.01W)      |
| **STA command** | 상태 전체 읽기   | FC 0x03, reg=0x0000, qty=10 (10개 연속 읽기) |
| **VER command** | 펌웨어 버전 읽기 | FC 0x03, reg=0x0017, qty=1                   |
| **C command**   | 주파수 채널 설정 | FC 0x06, reg=0x0003, val=채널번호(0~6)       |

### 5.2 STA 명령 응답 필드 → Modbus 레지스터 매핑

기존 STA 응답 형식:

```
I[ID],M[xxx]W,Rx,Ex,Sx,Wx,Fx,Vx,Axxx,Oxxx,[BCCH],[BCCL],[CR]
```

| STA 필드 | 의미                    | Modbus 레지스터              |
| -------- | ----------------------- | ---------------------------- |
| M[xxx]W  | 실측 출력 전력 (×0.01W) | 0x0002                       |
| Rx       | REMOTE 상태             | 0x0005 bit1 또는 0x0009 bit0 |
| Ex       | EXT 모드 상태           | 0x0005 bit2 또는 0x0006      |
| Sx       | SENSOR 상태             | 0x0005 bit3 또는 0x0009 bit1 |
| Wx       | 알람 상태 (0/1/2)       | 0x0005 bit4~5 또는 0x0012    |
| Fx       | 주파수 채널 (0~6)       | 0x0003                       |
| Vx       | 전압 (0~9)              | 0x0014                       |
| Axxx     | 전류 (0~999mA)          | 0x0015                       |
| Oxxx     | 임피던스 (50~999Ω)      | 0x0016                       |

### 5.3 전환 예시

**① 기존: 출력 0.10W 설정 (ID=1)**

```
기존 ASCII: "I1,W010W,4,9,[CR]"
Modbus RTU: [01] [06] [00 01] [00 0A] [D8 04]
→ reg=0x0001, val=10 (0.10W)
```

**② 기존: 채널 1 설정 (ID=1)**

```
기존 ASCII: "I1,C01C,7,9,[CR]"
Modbus RTU: [01] [06] [00 03] [00 01] [B8 0A]
→ reg=0x0003, val=1 (Ch1)
```

**③ 기존: 상태 읽기 (ID=1)**

```
기존 ASCII: "I1,STA,3,E,[CR]"
Modbus RTU: [01] [03] [00 00] [00 0A] [C5 CD]
→ reg=0x0000, qty=10 (운전 제어 전체 읽기)
```

**④ 기존: 버전 읽기 (ID=1)**

```
기존 ASCII: "I1,VER,x,x,[CR]"
Modbus RTU: [01] [03] [00 17] [00 01] [35 CE]
→ reg=0x0017, qty=1 (버전 레지스터 읽기)
응답값 예: 0x0100 → v1.00
```

### 7.2 슬레이브 측 수신 처리

- **프레임 완료 감지**: USART IDLE 인터럽트 또는 TIM4 (3.5T 타임아웃)
- **CRC 불일치**: 수신 프레임 무시 (무응답)
- **주소 불일치**: 수신 프레임 무시 (무응답)
- **브로드캐스트(주소 0)**: 쓰기 명령만 수행, 응답 전송 안 함
- **모드 제한**: NORMAL/REMOTE 모드에서 쓰기 요청 → 예외 응답 0x04

## 10. 마스터 구현 참고 코드 (C 언어)

### 10.1 STA 명령 대응 — 운전 상태 일괄 읽기

```c
/* 마스터 → 슬레이브: 운전 제어 레지스터 10개 일괄 읽기 (STA 명령 대체) */
void Modbus_ReadStatus(uint8_t *buf, uint8_t slave_addr)
{
    buf[0] = slave_addr;
    buf[1] = 0x03;            /* FC: Read Holding Registers */
    buf[2] = 0x00; buf[3] = 0x00;  /* 시작 레지스터: 0x0000 */
    buf[4] = 0x00; buf[5] = 0x0A;  /* 레지스터 수: 10개 */
    uint16_t crc = Modbus_CRC16(buf, 6);
    buf[6] = crc & 0xFF;
    buf[7] = (crc >> 8) & 0xFF;
    /* 총 8바이트 전송 → 응답 25바이트 (1+1+1+20+2) */
}

/* 응답 파싱 */
typedef struct {
    uint16_t run_state;        /* 0x0000: 0=OFF, 1=ON */
    uint16_t power_set;        /* 0x0001: ×0.01W */
    uint16_t power_actual;     /* 0x0002: ×0.01W */
    uint16_t freq_channel;     /* 0x0003: 0~6 */
    uint16_t freq_current;     /* 0x0004: ×0.1kHz */
    uint16_t status_flags;     /* 0x0005: 비트필드 */
    uint16_t mode;             /* 0x0006: 0/1/2 */
    uint16_t slave_addr;       /* 0x0007: 1~247 */
    uint16_t local_lock;       /* 0x0008: 0/1 */
    uint16_t ext_input;        /* 0x0009: 비트필드 */
} MegasonicStatus_t;
```

### 10.2 W 명령 대응 — 출력 전력 설정

```c
/* 마스터 → 슬레이브: 출력 전력 설정 (W 명령 대체) */
void Modbus_SetPower(uint8_t *buf, uint8_t slave_addr, uint16_t power_x100)
{
    /* power_x100: 20~100 (0.20~1.00W) */
    buf[0] = slave_addr;
    buf[1] = 0x06;            /* FC: Write Single Register */
    buf[2] = 0x00; buf[3] = 0x01;  /* 레지스터: 0x0001 */
    buf[4] = (power_x100 >> 8) & 0xFF;
    buf[5] =  power_x100       & 0xFF;
    uint16_t crc = Modbus_CRC16(buf, 6);
    buf[6] = crc & 0xFF;
    buf[7] = (crc >> 8) & 0xFF;
    /* EXT 모드에서만 동작. NORMAL/REMOTE에서는 예외 응답 */
}
```

### 10.3 C 명령 대응 — 주파수 채널 변경

```c
/* 마스터 → 슬레이브: 주파수 채널 설정 (C 명령 대체) */
void Modbus_SetChannel(uint8_t *buf, uint8_t slave_addr, uint16_t channel)
{
    /* channel: 0~6 (Ch0~Ch6) */
    buf[0] = slave_addr;
    buf[1] = 0x06;            /* FC: Write Single Register */
    buf[2] = 0x00; buf[3] = 0x03;  /* 레지스터: 0x0003 */
    buf[4] = (channel >> 8) & 0xFF;
    buf[5] =  channel       & 0xFF;
    uint16_t crc = Modbus_CRC16(buf, 6);
    buf[6] = crc & 0xFF;
    buf[7] = (crc >> 8) & 0xFF;
    /* ⚠ 채널 변경 시 발진 1초 정지 발생 */
}
```
