> 내용 출처: `docs/circuit_ref_64pin.html` 섹션 14 + 개발 과정에서 발견된 함정들.
> 마지막 갱신: 2026-04-08

## 🔴 Critical — 미충족 시 기능 불가

> 아래 항목 중 하나라도 틀리면 보드가 동작하지 않거나 즉시 파손됩니다.

- [ ] **PF0/PF1은 HSE 크리스탈 전용** — GPIO_Init에서 GPIOF 초기화 절대 금지.
  - `GPIO_CLOCKS_ENABLE()`에 `__HAL_RCC_GPIOF_CLK_ENABLE()` 없는지 확인
  - `gpio_init.c`에서 PF 포트 초기화 코드 없는지 확인

- [ ] **PA13/PA14은 SWD 전용** — GPIO로 사용 금지 (디버깅 불가).

- [ ] **PA8/PA9 핀 AF13** (HRTIM1_CHA1/CHA2), **GPIO_SPEED_FREQ_VERY_HIGH** — 낮은 속도로 설정하면 파형 왜곡.

- [ ] **HRTIM DLL 캘리브레이션**이 HRTIM 사용 전 반드시 호출됨 — 미호출 시 Period×32 배율 미적용 → 주파수 32배 오류.

  ```c
  HAL_HRTIM_DLLCalibrationStart(&hhrtim, HRTIM_CALIBRATIONRATE_14);
  HAL_HRTIM_PollForDLLCalibration(&hhrtim, HAL_MAX_DELAY);
  ```

- [ ] **UCC21520 VSSA ≠ VSSB** — VSSA는 SW 노드(Q1 소스)에, VSSB는 파워 GND에 각각 별도 연결. 공통 연결 시 관통 전류로 즉시 소손.

- [ ] **하이사이드 VDDA는 반드시 절연 DC-DC에서 공급** (R1215S 등) — 비절연 전원 사용 불가.

- [ ] **LM5005 FB 분압 저항 값 확인** — 기존 회로 값(R79=5.11kΩ, R80=16kΩ) 절대 사용 금지.
  - 신규 설계값: **R1=11kΩ, R2=1kΩ, R3=1.8kΩ**
  - 잘못된 값 사용 시 출력 전력 제어 범위가 완전히 틀어짐

- [ ] **LC_RELAY3/4는 PC7/PC8에 연결** — 예전 핀(PF0/PF1)에 연결하면 HSE 크리스탈 발진 불가.

- [ ] **RS-485 DE → PC13, /RE → PC14** — 예전 핀(PA4/PA5) 혼용 금지.
  - PA4는 현재 **DAC1_OUT1** (BUCK 전력 제어용)

- [ ] **PA4는 DAC1_OUT1 전용** — 다른 신호와 충돌 없는지 확인.

## 🟠 Important — 미충족 시 오동작 또는 측정 오류

- [ ] **HSE 크리스탈 부하 커패시터(CL)** — 크리스탈 데이터시트에서 권장 CL 확인 후 선택 (보통 9~22pF). PCB 기생 커패시턴스 고려.

- [ ] **UCC21520 바이패스 커패시터** — VDDA/VSSA 핀, VDDB/VSSB 핀 바로 옆에 각각 10µF + 100nF 쌍 배치.

- [ ] **STM32 VDDA 바이패스** — VDDA 핀 바로 옆에 100nF + 1µF 쌍 배치. ADC/DAC/컴프 정밀도에 직결.

- [ ] **ADC 캘리브레이션** 코드 펌웨어에 있는지 확인.

  ```c
  HAL_ADCEx_Calibration_Start(&hadc1, ADC_SINGLE_ENDED);
  HAL_ADCEx_Calibration_Start(&hadc2, ADC_SINGLE_ENDED);
  ```

- [ ] **UCC21520 DT 핀(100kΩ)** 사용 시 HRTIM 내장 데드타임은 0으로 설정 — 이중 사용 금지 (과도한 듀티 손실).

- [ ] **RS-485 종단 저항 120Ω** — 버스 맨 끝 장치에만 배치. 중간 장치에 넣으면 신호 약해짐.

- [ ] **RS-485 송신 완료 후 DE→LOW** — USART2 TC 플래그 확인 후 전환하는 코드 있는지 확인.

- [ ] **BOOT0 핀 10kΩ 풀다운** — 배치 안 하면 플래시 부팅 안 되고 부트로더 모드로 진입.

- [ ] **LCD1602 R/W 핀 GND 고정** — HIGH로 두면 쓰기 불가.

- [ ] **LCD1602 V0 핀 가변저항(10kΩ)** — 없으면 대비가 0이라 아무것도 안 보임.

- [ ] **버튼 PC0~PC4 외부 10kΩ 풀업** 설정 확인 (디스플레이 보드 탑재, GPIO_NOPULL, Active LOW 방식).

- [ ] **LED 전류 제한 저항** (330Ω~1kΩ) — 없으면 MCU GPIO 과전류 → 핀 소손.

- [ ] **ULN2003A COM 핀 → 릴레이 코일 전원(5V)** 연결 확인.

- [ ] **알람 릴레이 접점 정격** — AC 100V/DC 100V, 0.3A 초과 부하 연결 금지.

- [ ] **광커플러 TLP281-4 전류 제한 저항** 적절한지 확인 (외부 전원 전압에 따라 계산).

- [ ] **RF 전력 검출기 DC 출력** ≤ 3.3V 확인 — VDDA(3.3V) 초과 시 ADC 핀 소손.

## 🟡 권장 — 성능·신뢰성·EMI

- [ ] 2MHz 동작 시 **Si MOSFET(IPB200N15N3) 발열 계산** → 방열판 필요 여부 결정.
- [ ] **HRTIM PWM 트레이스(PA8→UCC21520)** 최단 거리 배선, GND 플레인 위에 배치.
- [ ] **전력 루프(Vbus→Q1→SW노드→Q2→GND)** 면적 최소화.
- [ ] **UCC21520 1차/2차측 PCB 절연 거리** 최소 8mm — 슬릿 가공 권장.
- [ ] **HSE 크리스탈↔MCU 트레이스** 20mm 이내, 고속 신호 라인과 2mm 이격.
- [ ] **LM5005 R_T = 330kΩ** (f_sw = 100kHz) — 48V 입력 시 최소 ON-time 여유 확보.
- [ ] 4층 PCB 또는 연속 **GND 플레인** 확보.
- [ ] **RELAY_OPERATE** 부하: 24V 0.1A 이상 접점 용량 사용.
- [ ] **디버그 USART1(PB6/PB7)** 테스트 패드 또는 2.54mm 헤더 노출.

- [ ] **C60 DC 블로킹 캡** → 0.1µF에서 **0.47µF/100V PP 필름**으로 변경 — 500kHz에서 Xc 3.18Ω→0.68Ω (§21 참조).
- [ ] **C58/C59** 유전체 **C0G(NP0)** 확인 + 10nF/100V C0G 병렬 추가 권장.
- [ ] **FC1/FC2** 1000µF→**330~470µF/50V 저ESR** + 1µF/50V 세라믹 병렬 추가 권장.
- [ ] **R29/R30/R31** 와트 정격 과잉 — 실소비 전력 대비 확인 (R29/R30: 0.5W, R31: 1~2W 충분).

## 🏗️ 2보드 분리 설계 (MCU보드 ↔ 파워보드 하네스)

> STM32G474 MCU 보드와 IPB200N15N3 MOSFET 파워 보드가 **물리적으로 분리**되어 하네스 케이블로 연결된다.

### 커넥터 위치 결정 — MCU ↔ UCC21520 사이 (필수)

- [ ] **UCC21520 게이트 드라이버는 반드시 파워 보드(MOSFET 측)에 배치** — MCU 보드에 두면 안 됨.
  - 게이트 루프 인덕턴스 = 케이블 인덕턴스(수백 nH) → 게이트 링잉 → Vgs 초과 → MOSFET 파손.
  - `V = L × dI/dt` : 20cm 케이블(~200nH), 2MHz, dI/dt≈20A/µs → V_ring ≈ 4V (Vgs 기준 허용 불가).
- [ ] **커넥터는 MCU보드와 파워보드 사이**에 설치 — 게이트 드라이버↔MOSFET 사이에 커넥터 금지.

### 하네스 케이블 핀 할당

| 방향           | 신호     | MCU 핀 | 설명                             |
| -------------- | -------- | ------ | -------------------------------- |
| MCU → 파워보드 | INA      | PA8    | HRTIM CHA1 (하이사이드 입력)     |
| MCU → 파워보드 | INB      | PA9    | HRTIM CHA2 (로우사이드 입력)     |
| MCU → 파워보드 | DAC_FB   | PA4    | BUCK FB 제어 (DAC1_OUT1, 0~3.3V) |
| MCU → 파워보드 | VCC 3.3V | —      | UCC21520 VCC 공급                |
| MCU → 파워보드 | GND      | —      | 공통 GND                         |
| 파워보드 → MCU | ADC_FWD  | PA0    | 순방향 전력 검출 DC 출력         |
| 파워보드 → MCU | ADC_REV  | PA1    | 역방향 전력 검출 DC 출력         |
| 파워보드 → MCU | ADC_CUR  | PA6    | BUCK 출력 전류 (센스 저항)       |
| 파워보드 → MCU | ADC_VOL  | PA7    | BUCK 출력 전압 (분압)            |

> **커넥터 권장**: JST XH 2.5mm 10핀 또는 Molex 51021-1000 (로킹형, 진동 환경 권장).

### 하네스 신호 보호 회로

#### INA/INB 보호 소자 — 값 및 배치 근거

| 소자              | **값**              | **배치 보드** | 위치                             | 목적                    |
| ----------------- | ------------------- | ------------- | -------------------------------- | ----------------------- |
| 직렬 저항 R_S     | **47Ω, 1%, 0402**   | **MCU 보드**  | PA8/PA9 핀에서 2mm 이내          | 소스 종단 + 반사파 억제 |
| 쉬트 커패시터 C_F | **22pF, C0G, 0402** | **파워 보드** | UCC21520 INA/INB 핀에서 2mm 이내 | HF 노이즈 필터          |

```
MCU 보드                   하네스 케이블              파워 보드
                           (~100Ω, 10~20cm)
PA8 ─[R_S 47Ω]─────────────────────────────────────┬── INA (UCC21520 핀5)
                                                    ├──[C_F 22pF]── GND
PA9 ─[R_S 47Ω]─────────────────────────────────────┬── INB (UCC21520 핀6)
                                                    └──[C_F 22pF]── GND
```

쌍 1: INA + GND (꼬임) ─┐
쌍 2: INB + GND (꼬임) ─┤─── 전체 실드 ─── MCU 보드 GND에만 접속

**직렬 저항 MCU 보드에 배치하는 이유**:
케이블 끝에서 반사된 전압 스파이크가 GPIO 핀에 도달하기 전에 R_S로 흡수됨.
파워 보드에 두면 케이블 전 구간을 무종단으로 통과한 반사파가 MCU 핀을 직격함.

**쉬트 커패시터 파워 보드에 배치하는 이유**:
케이블을 지나는 동안 유도된 HF 노이즈를 UCC21520 입력 직전에 GND로 바이패스.
MCU 보드에 두면 케이블 구간에서 노이즈가 다시 실림.

**설계 검증 (최악 조건 2MHz)**:

- GPIO Rout ≈ 15Ω + R_S 47Ω = 62Ω → 케이블 100Ω 소스 종단 (반사 계수 Γ ≈ 0.23)
- R-C LPF: f_c = 1/(2π × 62Ω × 22pF) = **116MHz** (2MHz의 58배 → 감쇠 negligible)
- 2MHz에서 R_S×CL 지연 = 47Ω × 25pF = 1.2ns (period 500ns 대비 0.24% → 무시 가능)

**⚠️ 주의사항**:

- C_F는 반드시 **C0G(NP0)** 유전체 사용 — X5R/X7R는 전압 계수 있어 실효 용량 변동
- R_S와 C_F를 **같은 보드에 나란히 배치하면 효과 반감** — 반드시 각각 분리 배치

- [ ] **R_S 47Ω (1%, C0G 불필요)** — MCU 보드 PA8/PA9 핀 바로 옆 직렬 삽입.
- [ ] **C_F 22pF (C0G, 0402)** — 파워 보드 UCC21520 INA/INB 핀 바로 옆 GND 연결.
- [ ] **케이블 길이** — 10cm 이내 이상적, 10~20cm 허용(보호 회로 필수), 20cm 초과 시 신호 무결성 별도 분석 필요.
- [ ] **ADC 신호선(PA0/PA1/PA6/PA7)** — 저주파 DC 신호이므로 케이블 길이에 덜 민감. 단, 쉴드 처리 또는 Twisted pair 권장 (노이즈 픽업 방지).
