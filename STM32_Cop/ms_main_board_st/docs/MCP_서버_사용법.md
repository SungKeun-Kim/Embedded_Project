## 1. JLCPCB MCP 서버

LCSC/JLCPCB 공식 부품 라이브러리에서 부품을 검색하고 KiCad 라이브러리에 자동으로 설치한다.
설치된 심볼/풋프린트는 `~/Documents/KiCad/{버전}/3rdparty/jlc_mcp/` 경로에 저장된다.

---

### 1.1 부품 검색

**도구:** `mcp_jlcpcb_component_search`

LCSC/JLCPCB 공식 라이브러리 또는 EasyEDA 커뮤니티 라이브러리에서 키워드로 부품을 검색한다.

| 파라미터     | 타입    | 필수 | 설명                                                  |
| ------------ | ------- | ---- | ----------------------------------------------------- |
| `query`      | string  | ✅   | 검색어 (예: `"STM32G474"`, `"0805 100nF"`, `"ESP32"`) |
| `source`     | string  | ❌   | `"lcsc"` (기본, 공식) 또는 `"community"` (커뮤니티)   |
| `basic_only` | boolean | ❌   | `true` 시 JLCPCB 기본 부품만 반환 (조립 비용 절감)    |
| `in_stock`   | boolean | ❌   | `true` 시 재고 있는 부품만 반환                       |
| `limit`      | number  | ❌   | 결과 최대 개수 (기본 10, 최대 50)                     |

**반환값:** LCSC ID, 제조사 부품번호, 설명, 패키지, 가격, 재고, 데이터시트 URL 등

**사용 예시:**

```
"STM32G474MET6 부품 검색해줘"
"0805 100nF 커패시터 재고 있는 것만 찾아줘"
"MAX3485 RS-485 트랜시버 기본 부품으로 검색해줘"
"커뮤니티 라이브러리에서 XIAO RP2040 찾아줘"
```

**검색 소스 선택 가이드:**

- `lcsc` (기본): JLCPCB 조립 주문 시 사용 가능한 공식 부품. LCSC ID 보유.
- `community`: Arduino 모듈, 개발보드 등 사용자가 기여한 EasyEDA 라이브러리. 비표준 부품에 유용.

---

### 1.2 라이브러리 설치

**도구:** `mcp_jlcpcb_library_install`

검색한 부품을 KiCad 글로벌 라이브러리에 자동 설치한다. 심볼(.kicad_sym)과 풋프린트(.kicad_mod)가 함께 설치된다.

| 파라미터       | 타입    | 필수 | 설명                                            |
| -------------- | ------- | ---- | ----------------------------------------------- |
| `id`           | string  | ✅   | LCSC 부품번호 (예: `"C2040"`) 또는 EasyEDA UUID |
| `project_path` | string  | ❌   | 프로젝트 경로 (EasyEDA UUID 설치 시 필수)       |
| `include_3d`   | boolean | ❌   | 3D 모델 포함 여부 (기본 `false`)                |
| `force`        | boolean | ❌   | 이미 설치된 경우에도 재설치 (기본 `false`)      |

**반환값:** `symbol_ref` (심볼 참조명), `footprint_ref` (풋프린트 참조명)

**설치 위치 (카테고리별 자동 분류):**

```
~/Documents/KiCad/{버전}/3rdparty/jlc_mcp/
├── JLC-MCP-Resistors.kicad_sym
├── JLC-MCP-Capacitors.kicad_sym
├── JLC-MCP-ICs.kicad_sym
└── ...
```

**사용 예시:**

```
"C2040 부품 KiCad에 설치해줘"
"MAX3485 LCSC 번호 찾아서 설치해줘"
"C14663 3D 모델 포함해서 설치해줘"
```

---

### 1.3 라이브러리 일괄 설치

**도구:** `mcp_jlcpcb_library_batch_install`

BOM(부품 목록) 전체를 한 번에 설치할 때 사용한다. 최대 10개 부품을 병렬로 설치한다.

| 파라미터     | 타입     | 필수 | 설명                                       |
| ------------ | -------- | ---- | ------------------------------------------ |
| `ids`        | string[] | ✅   | LCSC 부품번호 배열 (최대 10개)             |
| `include_3d` | boolean  | ❌   | 3D 모델 포함 여부 (기본 `false`)           |
| `force`      | boolean  | ❌   | 이미 설치된 경우에도 재설치 (기본 `false`) |

**반환값:** 설치 성공 / 스킵(이미 설치) / 실패 부품 목록 요약

**사용 예시:**

```
"다음 LCSC 부품들 한번에 설치해줘: C2040, C5446, C14663, C84861"
"BOM 부품 목록 전부 KiCad에 설치해줘"
```

---

### 1.4 설치된 부품 정보 조회

**도구:** `mcp_jlcpcb_library_get_component`

이미 설치된 부품의 심볼/풋프린트 참조명과 파일 경로를 조회한다.

| 파라미터 | 타입   | 필수 | 설명                          |
| -------- | ------ | ---- | ----------------------------- |
| `id`     | string | ✅   | LCSC 부품번호 (예: `"C2040"`) |

**반환값:** 심볼 참조명, 풋프린트 참조명, 파일 경로, 핀/패드 수

**사용 예시:**

```
"C2040 부품 설치 상태 확인해줘"
"MAX3485 심볼과 풋프린트 참조명 알려줘"
```

---

### 1.5 심볼 수정

**도구:** `mcp_jlcpcb_library_fix`

설치된 심볼의 핀 이름, 타입, 위치 오류를 수정하거나 누락된 핀(예: 노출 패드)을 추가한다.
**주의:** 심볼이 이미 설치된 상태에서만 사용 가능.

| 파라미터       | 타입    | 필수 | 설명                                                      |
| -------------- | ------- | ---- | --------------------------------------------------------- |
| `lcsc_id`      | string  | ✅   | LCSC 부품번호                                             |
| `corrections`  | object  | ✅   | 수정 내용 (아래 액션 타입 참조)                           |
| `project_path` | string  | ❌   | 글로벌 라이브러리가 아닌 프로젝트 라이브러리 수정 시 지정 |
| `force`        | boolean | ❌   | 심볼 미존재 시에도 강제 생성 (기본 `false`)               |

**corrections 액션 타입:**

| 액션     | 설명                             | 필수 파라미터                 |
| -------- | -------------------------------- | ----------------------------- |
| `modify` | 핀 이름 변경 또는 전기 타입 변경 | `number`, `rename`/`set_type` |
| `swap`   | 두 핀 위치 교환                  | `pins: [pin1, pin2]`          |
| `add`    | 핀 추가 (노출 패드 등)           | `number`, `name`, `type`      |
| `remove` | 핀 제거                          | `number`                      |

**핀 전기 타입:** `input`, `output`, `bidirectional`, `power_in`, `power_out`, `passive`, `open_collector`, `open_emitter`, `unconnected`, `unspecified`

**사용 예시:**

```
"C2913199 심볼에 노출 패드(EP=GND) 추가해줘"
"C123456 1번 핀을 power_in 타입으로 바꿔줘"
"C5678 3번 핀과 4번 핀 위치 바꿔줘"
```

## 2. KiCad MCP 서버

KiCad 프로젝트 파일(.kicad_pro)을 읽고 회로도 분석, 넷리스트 추출, DRC 검사, BOM 생성 등을 수행한다.

---

### 2.1 프로젝트 관리

#### 프로젝트 목록 조회

**도구:** `mcp_kicad_list_projects`

시스템에 설치된 모든 KiCad 프로젝트를 찾아 목록으로 반환한다. 파라미터 없음.

```
"시스템에 KiCad 프로젝트 어떤 것들 있어?"
```

#### 프로젝트 구조 조회

**도구:** `mcp_kicad_get_project_structure`

| 파라미터       | 타입   | 필수 | 설명                   |
| -------------- | ------ | ---- | ---------------------- |
| `project_path` | string | ✅   | `.kicad_pro` 파일 경로 |

프로젝트에 포함된 모든 파일(회로도, PCB, 라이브러리 등)의 구조를 반환한다.

```
"C:/KiCad/myboard/myboard.kicad_pro 프로젝트 구조 보여줘"
```

#### 프로젝트 유효성 검사

**도구:** `mcp_kicad_validate_project`

| 파라미터       | 타입   | 필수 | 설명                   |
| -------------- | ------ | ---- | ---------------------- |
| `project_path` | string | ✅   | `.kicad_pro` 파일 경로 |

프로젝트 파일의 기본 유효성을 빠르게 검사한다.

```
"이 KiCad 프로젝트 유효한지 확인해줘"
```

#### 프로젝트 열기

**도구:** `mcp_kicad_open_project`

| 파라미터       | 타입   | 필수 | 설명                   |
| -------------- | ------ | ---- | ---------------------- |
| `project_path` | string | ✅   | `.kicad_pro` 파일 경로 |

KiCad 애플리케이션에서 프로젝트를 직접 열어준다.

---

### 2.2 회로도 분석

#### 회로 패턴 식별 (회로도 파일)

**도구:** `mcp_kicad_identify_circuit_patterns`

| 파라미터         | 타입   | 필수 | 설명                          |
| ---------------- | ------ | ---- | ----------------------------- |
| `schematic_path` | string | ✅   | `.kicad_sch` 회로도 파일 경로 |

회로도에서 일반적인 회로 블록을 자동으로 인식한다.

**인식 가능한 패턴:**

- 전원 회로: 선형 레귤레이터, 스위칭 컨버터 (BUCK/BOOST)
- 증폭기: Op-Amp, 트랜지스터 앰프
- 필터: RC, LC, 액티브 필터
- 디지털 인터페이스: I2C, SPI, UART, RS-485
- 마이크로컨트롤러 회로
- 기타 일반 회로 블록

```
"myboard.kicad_sch 에서 전원 회로 패턴 찾아줘"
"이 회로도에서 마이크로컨트롤러 주변 회로 분석해줘"
```

#### 회로 패턴 식별 (프로젝트)

**도구:** `mcp_kicad_analyze_project_circuit_patterns`

| 파라미터       | 타입   | 필수 | 설명                   |
| -------------- | ------ | ---- | ---------------------- |
| `project_path` | string | ✅   | `.kicad_pro` 파일 경로 |

프로젝트 전체 회로도의 회로 패턴을 분석한다.

#### 넷리스트 추출 (회로도 파일)

**도구:** `mcp_kicad_extract_schematic_netlist`

| 파라미터         | 타입   | 필수 | 설명                          |
| ---------------- | ------ | ---- | ----------------------------- |
| `schematic_path` | string | ✅   | `.kicad_sch` 회로도 파일 경로 |

회로도 파일에서 부품, 연결, 라벨 정보를 포함한 넷리스트를 추출한다.

```
"이 회로도의 넷리스트 추출해줘"
"GND 넷에 어떤 부품들이 연결되어 있어?"
```

#### 넷리스트 추출 (프로젝트)

**도구:** `mcp_kicad_extract_project_netlist`

| 파라미터       | 타입   | 필수 | 설명                   |
| -------------- | ------ | ---- | ---------------------- |
| `project_path` | string | ✅   | `.kicad_pro` 파일 경로 |

#### 연결 분석

**도구:** `mcp_kicad_analyze_schematic_connections`

| 파라미터         | 타입   | 필수 | 설명                          |
| ---------------- | ------ | ---- | ----------------------------- |
| `schematic_path` | string | ✅   | `.kicad_sch` 회로도 파일 경로 |

전원 넷, 신호 경로, 잠재적 연결 문제를 상세히 분석한다.

```
"이 회로도의 전원 연결 문제 없는지 분석해줘"
"신호 경로 분석해서 문제점 찾아줘"
```

#### 특정 부품 연결 조회

**도구:** `mcp_kicad_find_component_connections`

| 파라미터        | 타입   | 필수 | 설명                                        |
| --------------- | ------ | ---- | ------------------------------------------- |
| `project_path`  | string | ✅   | `.kicad_pro` 파일 경로                      |

```

```

## 3. 실전 워크플로우 예시

### 워크플로우 A: 신규 부품 추가 (JLCPCB → KiCad)

```
1. 부품 검색
   "MAX3485 RS-485 트랜시버 JLCPCB에서 검색해줘"
   → LCSC ID 확인 (예: C84861)

2. 라이브러리 설치
   "C84861 KiCad에 설치해줘"
   → symbol_ref, footprint_ref 반환

3. 설치 확인
   "C84861 설치 상태 확인해줘"
   → 파일 경로 및 핀 수 확인
```

## 4. 주의사항

### JLCPCB MCP

- **LCSC ID vs EasyEDA UUID**: LCSC ID(`C숫자`)는 글로벌 라이브러리에 설치되며, EasyEDA UUID는 `project_path` 지정 시 프로젝트 로컬 라이브러리에 설치된다.
- **기본 부품(Basic Parts)**: `basic_only: true` 옵션 사용 시 JLCPCB 조립 기본 부품만 검색된다. 기본 부품은 조립 추가 비용이 없으므로 비용 절감에 유리하다.
- **3D 모델**: `include_3d: true` 설정 시 파일 크기가 커지고 설치 시간이 늘어날 수 있다. 필요한 경우에만 활성화한다.
- **심볼 수정**: `mcp_jlcpcb_library_fix`는 EasyEDA 원본 데이터를 재가공하므로, 핀 번호가 정확해야 한다. 데이터시트에서 핀 번호를 먼저 확인할 것.

### KiCad MCP

- **kicad-cli 필요**: `export_bom_csv`, `generate_pcb_thumbnail`, `run_drc_check` 등 일부 도구는 KiCad CLI 도구가 시스템에 설치되어 있어야 정상 동작한다.
- **파일 경로**: 모든 경로는 `.kicad_pro` 파일을 기준으로 한다. 절대 경로를 사용하는 것이 안전하다.
- **넷리스트 추출**: `.kicad_sch` 파일이 없거나 프로젝트와 분리된 경우 오류가 발생할 수 있다.
- **DRC**: PCB 레이아웃(`.kicad_pcb`)이 없으면 DRC 실행 불가.

### 이 프로젝트(ms_main_board_st)에서의 활용 예시

```
# STM32G474MET6 부품 검색
"STM32G474MET6 JLCPCB에서 검색해줘"

# 주요 부품 일괄 설치 (예시 LCSC ID)
"다음 부품 한번에 설치해줘: MAX3485 LCSC ID, ULN2003A LCSC ID, LM5005 LCSC ID"

# KiCad 프로젝트 생성 후 회로도 분석
"myproject.kicad_pro HRTIM 하프브리지 게이트 드라이버 회로 패턴 찾아줘"
"U1(STM32G474) PA8, PA9 핀 연결 확인해줘"
```
