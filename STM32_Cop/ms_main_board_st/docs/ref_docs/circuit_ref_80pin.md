# 메가소닉 게이트 드라이브 회로 참조 문서

> 2.2MHz 메가소닉 발진기용 하프브리지 게이트 드라이브 회로 설계 참조.
> UCC21520 절연 게이트 드라이버 + IPB200N15N3 Si MOSFET + 절연 DC-DC 컨버터 구성.

---

## 2. UCC21520 절연 게이트 드라이버

### 개요

| 항목                  | 사양                                      |
| --------------------- | ----------------------------------------- |
| 제조사                | Texas Instruments                         |
| 패키지                | SOIC-16W (DW)                             |
| 절연 내압             | 5.7kVrms (강화 절연)                      |
| 전파 지연             | 18ns (typ)                                |
| CMTI                  | 150kV/µs (Common Mode Transient Immunity) |
| 피크 출력 전류        | 4A (source/sink)                          |
| 동작 주파수           | DC ~ 5MHz                                 |
| 입력 전압 (VCC)       | 3.0V ~ 5.5V                               |
| 출력 전압 (VDDA/VDDB) | 9.2V ~ 25V                                |

### 핀 배치 및 기능

```
          ┌─────────────────────────────────────┐
          │            UCC21520DW               │
     Primary / MCU  │                                     │   Secondary / FET
          │                                     │
 VCCI (+3.3V) ──────┤ 8  VCCI                 VDDA   16 ├──── VDDA / High-Side
          │                                     │
 DISABLE      ──────┤ 7  DIS                   OUTA   15 ├──── Gate Q1
          │                                     │
 INB (PA9)    ──────┤ 6  INB                   VSSA   14 ├──── SW node
          │                                     │
 INA (PA8)    ──────┤ 5  INA                   NC     13 ├──── NC
          │                                     │
 GND          ──────┤ 4  GND                   NC     12 ├──── NC
          │                                     │
 NC           ──────┤ 3  NC                    VDDB   11 ├──── VDDB / Low-Side
          │                                     │
 DT           ──────┤ 2  DT                    OUTB   10 ├──── Gate Q2
          │                                     │
 VCCI         ──────┤ 1  VCCI                  VSSB    9 ├──── Power GND
          │                                     │
          └─────────────────────────────────────┘
```

### 전원 도메인 (3개 독립 절연)

| 도메인                   | 전원 핀 | GND 핀 | 기준점       | 전압 범위   |
| ------------------------ | ------- | ------ | ------------ | ----------- |
| **1차측 (입력)**         | VCC     | GND    | 시스템 GND   | 3.0V ~ 5.5V |
| **2차측 A (하이사이드)** | VDDA    | VSSA   | **SW 노드**  | 9.2V ~ 25V  |
| **2차측 B (로우사이드)** | VDDB    | VSSB   | **파워 GND** | 9.2V ~ 25V  |

> ⚠️ **VSSA ≠ VSSB**: VSSA는 SW 노드에, VSSB는 파워 GND에 연결. 절대 공유 금지!

### 데드타임 설정 (DT 핀)

| DT 핀 연결             | 데드타임          |
| ---------------------- | ----------------- |
| GND                    | 35ns (최소)       |
| 저항 50kΩ~200kΩ        | 가변 설정         |
| VCC                    | 100ns 이상        |
| **권장 (IPB200N15N3)** | **100kΩ → ~80ns** |

> Si MOSFET은 GaN 대비 스위칭이 느리므로 50~100ns 데드타임 권장.
> HRTIM 내장 데드타임과 UCC21520 DT 핀 중 **하나만 사용** 권장 (이중 적용 시 과도한 듀티 손실).

## 3. IPB200N15N3 Si MOSFET

### 개요

| 항목                | 사양           |
| ------------------- | -------------- |
| 제조사              | Infineon       |
| 패키지              | D2PAK (TO-263) |
| Vds (최대)          | 150V           |
| Rds(on)             | 20mΩ @ Vgs=10V |
| Id (연속)           | 50A @ 25°C     |
| Qg (총 게이트 전하) | 31nC           |
| Qgd (밀러 전하)     | 9nC            |
| tr (라이즈 타임)    | 7ns            |
| tf (폴 타임)        | 4ns            |
| 게이트 전압         | 10V ~ 15V 권장 |

### D2PAK 패키지 장점

- **손납땜 가능**: 핀 피치 2.54mm, 드레인 패드 대형
- **방열 우수**: 큰 금속 탭으로 PCB 히트싱크 연결 용이
- **TO-220 대비**: 리드 인덕턴스 감소 (~5nH vs ~15nH)

### 게이트 드라이브 요구사항

```
게이트 충전 시간 = Qg / Ig_avg
               = 31nC / 2A (UCC21520 평균 출력)
               ≈ 15.5ns

권장 게이트 전압: 12V ~ 15V (Rds(on) 최적화)
최대 Vgs: ±20V
```

## 4. 절연 DC-DC 컨버터

### VDDA 전원 공급 (하이사이드 - 필수 절연)

VSSA = SW 노드이므로 **반드시 절연된 전원** 필요.

#### 옵션 1: RECOM R1215S (권장)

| 항목   | 사양                   |
| ------ | ---------------------- |
| 입력   | 12V DC (10.8V ~ 13.2V) |
| 출력   | 15V DC, 67mA           |
| 절연   | 1kVDC (기본 절연)      |
| 패키지 | SIP-4 (1W 급)          |
| 효율   | 80%                    |

```
    시스템 12V ────┬────▶ R1215S ────┬────▶ VDDA (15V)
                   │      (절연)     │
                   │                 └────▶ VSSA (= SW 노드에 연결)
                   │
                   └────────────────────────▶ VDDB (직접)
                                              VSSB (= 파워 GND)
```

#### 옵션 2: SN6505A + 트랜스포머 (고절연)

고절연(5.7kVrms) 요구 시 사용.

| 항목        | 사양                      |
| ----------- | ------------------------- |
| 드라이버 IC | SN6505A (TI)              |
| 입력        | 3.3V 또는 5V              |
| 트랜스포머  | Würth 750316830 (1:1)     |
| 정류        | 쇼트키 다이오드 + LC 필터 |

### VDDB 전원 공급 (로우사이드 - 직접 가능)

VSSB = 파워 GND이므로 **시스템 전원에서 직접 공급 가능**.

```
    시스템 12V~15V ────────────────────▶ VDDB
                                        │
    파워 GND ──────────────────────────▶ VSSB
```

## 6. 설계 시 주의사항

### 2.2MHz에서의 고려사항

| 항목            | Si MOSFET (IPB200N15N3) | GaN FET (EPC2036) |
| --------------- | ----------------------- | ----------------- |
| **스위칭 시간** | ~7ns                    | ~2ns              |
| **데드타임**    | 50~100ns 권장           | 5~20ns 권장       |
| **게이트 전압** | 10~15V                  | 4.5~5.5V          |
| **스위칭 손실** | 높음 (MHz에서)          | 낮음              |
| **역회복 전류** | 있음                    | 없음              |
| **납땜 난이도** | 쉬움 (D2PAK)            | 어려움 (BGA)      |

> ⚠️ **2.2MHz에서 Si MOSFET 사용 시 스위칭 손실이 증가**합니다.
> 저출력 애플리케이션에서는 괜찮으나, 고출력 시 발열 관리 필수.

### PCB 레이아웃 권장사항

1. **게이트 루프 최소화**: OUTA → Q1 게이트 → Q1 소스 → VSSA 경로를 가능한 짧게
2. **전력 루프 최소화**: Vbus → Q1 → Q2 → GND 경로를 최단 거리로
3. **바이패스 커패시터**: VDDA/VSSA, VDDB/VSSB 핀 바로 옆에 배치
4. **GND 플레인**: 2층 또는 4층 PCB에서 연속된 GND 플레인 확보
5. **절연 거리**: 1차측/2차측 사이 최소 8mm 이격 (5.7kV 절연)

### HRTIM 데드타임 설정

UCC21520의 DT 핀으로 데드타임을 설정할 경우, **HRTIM 내장 데드타임은 0으로 설정**:

```c
// HRTIM 데드타임 비활성화 (UCC21520 DT 핀 사용 시)
sConfigDT.Prescaler = HRTIM_DEADTIME_PRESCALER_OFF;
sConfigDT.RisingValue = 0;
sConfigDT.RisingSign = HRTIM_DEADTIME_SIGN_POSITIVE;
sConfigDT.RisingLock = HRTIM_DEADTIME_LOCK_WRITE;
sConfigDT.FallingValue = 0;
sConfigDT.FallingSign = HRTIM_DEADTIME_SIGN_POSITIVE;
sConfigDT.FallingLock = HRTIM_DEADTIME_LOCK_WRITE;
HAL_HRTIM_DeadTimeConfig(&hhrtim, HRTIM_TIMERINDEX_TIMER_A, &sConfigDT);
```

또는 **UCC21520 DT 핀을 GND에 연결 (최소 35ns)**하고 HRTIM에서 추가 데드타임 설정:

```c
// HRTIM 데드타임 50ns 추가 (UCC21520 DT=GND 시)
// 50ns = 50e-9 * 5.44e9 = 272 틱
sConfigDT.RisingValue = 272;
sConfigDT.FallingValue = 272;
```

## 7. 대안 구성

### 옵션 A: 부트스트랩 방식 (저비용, 저주파용)

저주파수(~500kHz) 또는 듀티 50% 고정 시 사용 가능.

```
    VDD (15V) ────[D: 쇼트키]────┬──── VDDA
                                │
                              [C_boot: 10µF]
                                │
                              VSSA ──── SW 노드
```

> ⚠️ **2.2MHz에서는 충전 시간 부족 가능** → 절연 DC-DC 권장

### 옵션 B: GaN FET + LMG1210 (최고 성능)

MHz 대역 최적 성능이 필요한 경우.

| 구성            | 부품                                             |
| --------------- | ------------------------------------------------ |
| 게이트 드라이버 | LMG1210 (부트스트랩 내장, 100V)                  |
| FET             | EPC2036 (100V, 솔더볼) 또는 GS61004B (100V, QFN) |
| 장점            | 스위칭 2ns, 역회복 없음                          |
| 단점            | 납땜 어려움, 고가                                |

## 9. BUCK 전력 컨버터 (LM5005)

### 개요

48V SMPS 전원에서 가변 DC 전압을 생성하여 하프브리지 인버터의 출력 전력을 0.1W~10W 범위로 제어.

| 항목          | 사양                                   |
| ------------- | -------------------------------------- |
| 제조사        | Texas Instruments                      |
| 패키지        | **HTSSOP-20** (노출 패드, 손납땜 가능) |
| 입력 전압     | 7V ~ 75V                               |
| 출력 전압     | 1.23V ~ 68V (가변)                     |
| 스위칭 주파수 | 50kHz ~ 500kHz                         |
| 제어 방식     | Current Mode PWM                       |
| FB 기준 전압  | 1.23V                                  |
| 외부 FET      | Hi-side FET 1개 + 환류 쇼트키 다이오드 |

### 핀 배치

```
        ┌───────────────────────────────────────────────────────────────────┐
        │                    LM5005MH  HTSSOP-20                           │
        ├───────────────────────────────────────────────────────────────────┤
  VCC    1 ───┤ 5.25V regulator output              Bootstrap cap          ├─── 20 BST
  SD     2 ───┤ Shutdown input                      PRE tied to SW         ├─── 19 PRE
  VIN    3 ───┤ Input supply 7V~75V                 SW node (parallel)     ├─── 18 SW_1
  VIN_1  4 ───┤ Input supply parallel               SW node                ├─── 17 SW
  SYNC   5 ───┤ Ext sync input, NC if unused        D5 anode sense         ├─── 16 IS
  COMP   6 ───┤ Error amp / RC compensation         Same as IS             ├─── 15 IS_1
  FB     7 ───┤ Feedback input 1.23V                Power GND              ├─── 14 PGND
  RT     8 ───┤ f_sw set by RT                      Power GND              ├─── 13 PGND
  RAMP   9 ───┤ Ramp capacitor input                Low-side gate NC       ├─── 12 OUT
  AGND  10 ───┤ Analog GND                          Soft-start cap         ├─── 11 SS
        └───────────────────────────────────────────────────────────────────┘
          EP (exposed pad) -> solder to PCB GND thermal vias
```

| --- | ----- | -------------------------------------- | ------------------------------------------------------------------------ |
| 1   | VCC   | 내부 5.25V 레귤레이터 출력 (25mA)      | C_VCC=1µF/25V 바이패스 → GND                                             |
| 2   | SD    | 셧다운: LOW=정지, HIGH=동작            | 10kΩ → VIN 풀업 (항상 동작 시 VIN 직결 가능)                             |

| 5   | SYNC  | 외부 동기 클럭 입력 (양방향)           | **NC/Open** (미사용 시 — GND 연결 금지)                                  |

| 9   | RAMP  | 슬로프 보상 커패시터                   | C_RAMP=**330pF**/50V, C0G → GND (L=33µH 기준)                            |

### 출력 전압 조정 방법

#### 기본 원리

```
V_out = V_ref × (1 + R1/R2) = 1.23V × (1 + R1/R2)
```

FB 핀 전압이 1.23V가 되도록 출력을 조절함.

#### STM32 DAC를 이용한 가변 전압 제어

> ⚠️ **기존 LM5005MH 회로(R79=5.11K, R80=16K)의 분압비는 고정 +5V 출력용이므로 사용 불가!**
> 0.1W~10W 가변 제어를 위해 아래 R1, R2, R3 값을 새로 설계해야 합니다.

```
     V_out (2.2V ~ 22.4V 가변)
         │
        [R1] = 11kΩ (최대 전압 결정)
         │
         ├─────────────────────▶ FB (핀 7, LM5005)
         │                            │
        [R2] = 1kΩ                   [R3] = 1.8kΩ
         │                            │
        GND                    DAC1_OUT1 (PA4)
                               STM32G474
```

**동작 원리 (FB 노드 KCL)**:

```
(V_out - V_fb)/R1 = V_fb/R2 + (V_fb - V_dac)/R3

V_fb = 1.23V (FB 기준 전압) 일 때:
V_out = 1.23 × (1 + R1/R2) + (1.23 - V_dac) × R1/R3
```

- DAC = 0V: R3를 통해 전류 유출 → V_out = 최대
- DAC = V_fb(1.23V): R3 영향 없음 (전류=0)
- DAC > V_fb: R3를 통해 FB에 전류 주입 → V_out 감소

**설계값 (R1=11K, R2=1K, R3=1.8K) 계산**:

```
V_out = 1.23 × 12 + (1.23 - V_dac) × 6.11
V_out = 14.76 + (1.23 - V_dac) × 6.11

예시:
├── DAC = 0V    → V_out = 14.76 + 7.52 = 22.3V → P = 9.9W  ✓
├── DAC = 1.0V  → V_out = 14.76 + 1.41 = 16.2V → P = 5.2W
├── DAC = 1.23V → V_out = 14.76 + 0    = 14.8V → P = 4.4W (FB와 동일)
├── DAC = 2.0V  → V_out = 14.76 - 4.70 = 10.1V → P = 2.0W
├── DAC = 3.0V  → V_out = 14.76 - 10.8 = 3.9V  → P = 0.31W
└── DAC = 3.3V  → V_out = 14.76 - 12.7 = 2.1V  → P = 0.09W ✓
```

> ✅ **DAC 0~3.3V로 출력 0.1W ~ 10W 범위 제어 가능!**

> **Hi/Lo-side 동작 원리**:
>
> | 상태        | Hi-side 내장 FET | Lo-side D5       | 전류 방향                                    |
> | ----------- | ---------------- | ---------------- | -------------------------------------------- |
> | **FET ON**  | 도통             | 역바이어스(차단) | 48V → FET(D→S) → SW → L2 → VOUT → GND        |
> | **FET OFF** | 차단             | 순방향(도통)     | PGND → 내부R → IS → D5(A→K) → SW → L2 → VOUT |
>
> **C43 (22nF/100V, C0G)**: 부트스트랩 캐패시터. BST(20)~SW 노드 사이. FET OFF 시 VCC→내부다이오드→BST로 충전. 내압 100V 필수.
> **C_RAMP (330pF/50V, C0G)**: 슬로프 보상 커패시터. RAMP 핀(9번)→GND. **데이터시트 실수계산값: CRAMP = L × 10⁻⁵ = 33µH × 10⁻⁵ = 330pF**.
> 이 값을 사용하면 에뮤레이트드 전류 롱프의 스케일팩터(0.5V/A)가 DC 샘플&홀드 레벨과 일치. 100kHz 시 50V 내압 충분.
> **C_SS (100pF/10V, C0G)**: 소프트스타트 타이밍. SS 핀(11번)→GND. ~5µs 소프트 기동 (TI 참조회로 기준값).
> **C40 (330pF/100V)**: SW 노드 스너버. FET 스위칭 시 전압 링잉(ringing) 억제.
> **D5 (B3100-13-F, Diodes Inc, LCSC C134447)**: SMC(DO-214AB) SMD 쇼트키. 캐소드(K)→SW 노드, **애노드(A)→IS 핀(16/15)**. 100V/3A, Vf=790mV@3A. ⚠️ **울트라패스트 다이오드 사용 금지** (쇼트키 다이오드만 — 데이터시트 권장).
> **IS 핀 전류감지**: LM5005 내부 sense R이 IS(16)~PGND 사이에 내장 → D5 애노드 전류를 IC 내부에서 감지. 이 경로는 **LM5005 PWM 제어 전용** — MCU ADC(PA6) 연결 불가.
> **PRE(19) → SW 노드 묶음**: 데이터시트 권장. 경부하·DCM 진입 시 250ns 동안 C43 강제 재충전. CCM에서는 내부 역류방지로 손실 없음.
> ⚠️ **출력 캐패시터 내압**: 최대 출력 전압 22V이므로 **C44/C45 내압 35V 이상** 필수.
>
> ### MCU 출력 전류 측정 회로 (PA6 보호 필수)
>
> VOUT은 최대 22.4V — R_sense 양단을 MCU ADC에 **직결하면 MCU 즉시 파손**.
> 반드시 **전류감지 증폭기(INA190A4)**를 경유해 GND 기준 신호로 변환해야 함.
> ⚠️ INA181A1(공통모드 26V)은 VOUT_max 22.4V 대비 여유 3.6V뿐 — 과도 응답 시 파손 위험. **INA190A4(40V, Gain=200V/V)으로 교체.**
>
> ```
> VOUT ─[R110: 10mΩ/2512]─┬── 하프브리지 입력 (부하)
>                          │
>               ┌──────────┴──────────┐
>               │   INA190A4 (SOT-23-5)│
>               │   IN+ = R110 고전위  │
>               │   IN- = R110 저전위  │  ← 차동 입력: 공통모드 최대 40V (여유 17.6V)
>               │   Gain = 200V/V     │
>               │   VS = 3.3V, GND    │
>               │   OUT ─────────────────▶ PA6 (ADC2_IN3)
>               └─────────────────────┘
>                       ↑
>               [100nF 바이패스, VS핀]
>
> 전류 → ADC 전압 변환:
>   V_PA6 = Gain × I × R_sense = 200 × I × 0.01Ω = 2.0V/A
>   I = 100mA → V_PA6 = 200mV  (분해능 우수)
>   I = 450mA → V_PA6 = 900mV  ← ADC 3.3V 기준 27.3% 사용, 안전 ✓
>   I = 1A    → V_PA6 = 2.0V   ← ADC 3.3V 기준 60.6% 사용, 안전 ✓
> ```
>
> | 항목               | INA190A4 사양                                 |
> | ------------------ | --------------------------------------------- |
> | 패키지             | SOT-23-5 (INA181A1과 핀 호환)                 |
> | 공통모드 입력 전압 | **-4V ~ +40V** ✓ (VOUT 22.4V 대비 여유 17.6V) |
> | 게인               | **200V/V** (INA190**A4** 고정)                |
> | 전원               | 2.7V~5.5V (3.3V 직결 가능)                    |
> | 출력               | GND 기준 0~VS                                 |
>
> > ⚠️ **INA190A4을 생략하고 R110 → PA6 직결은 절대 금지.** R110의 고전위 측 전압 = VOUT(최대 22.4V) → MCU VDD(3.3V) 초과 → GPIO 보호 다이오드 파괴.

### MCU BUCK 출력 전압 측정 회로 (PA7 보호 필수)

VOUT 최대 설계값은 22.4V이나, 과도 응답·전원 투입 시 **최대 35V**까지 발생할 수 있음.
PA7(ADC2_IN4) VDDA = 3.3V이므로 **저항 분압기** 필수.

```
VOUT (0 ~ 35V 최대)
     │
    [R_VH: 100kΩ, 1%, 0603]   ← 상단 분압 저항
     │
     ├──────────╾────────────▶ PA7 (ADC2_IN4, MCU 3.3V)
     │          ↑
     │     [100nF C0G]        ← 노이즈 필터 (선택)
     │          │
    [R_VL: 8.2kΩ, 1%, 0603]   ← 하단 분압 저항
     │
    GND
```

**분압비 계산 (VOUT_max = 35V 기준):**

```
분압비 = R_VL / (R_VH + R_VL) = 8.2 / 108.2 = 0.0758

VOUT → V_PA7 변환:
├── VOUT = 35V (과도 최대) → V_PA7 = 35 × 0.0758 = 2.65V ✓  (ADC 3.3V 마진 0.65V)
├── VOUT = 22.4V (정상 최대) → V_PA7 = 22.4 × 0.0758 = 1.70V ✓
├── VOUT = 10V  → V_PA7 = 0.758V ✓
└── VOUT = 2V   → V_PA7 = 0.152V ✓

ADC count → VOUT 역산:
VOUT = ADC_count × (3.3V / 4096) × (R_VH + R_VL) / R_VL
     = ADC_count × (3.3 / 4096) × (108.2 / 8.2)
     = ADC_count × 0.1063V/count

ADC 분해능: 35V / 4096 ≈ 8.5mV/count (VOUT 환산)
```

> ⚠️ **PA7 직결은 절대 금지.** VOUT(최대 35V) → MCU VDDA(3.3V) 초과 → ADC 입력 파괴.
> ⚠️ **R_VH + R_VL 분압 저항은 1% 정밀도** 사용 (전압 측정 오차 직접 반영).

| 항목                   | R_VH(100kΩ) + R_VL(8.2kΩ)          |
| ---------------------- | ---------------------------------- |
| 분압비                 | 1/13.2 (≈1/13)                     |
| 35V 입력 시 PA7 전압   | **2.65V** (MCU 안전 ✓)             |
| 22.4V 입력 시 PA7 전압 | 1.70V                              |
| 소비 전류 (35V 시)     | 35V / 108.2kΩ ≈ 0.32mA (무시 가능) |

> **PA7(ADC2_IN4) 용도**: BUCK 출력 전압 실시간 모니터링 → PI 피드백 루프 보조, 출력 과전압 감지(SW 보호).
> 현재 `adc_control.c`에서 ADC2 채널은 미구현 상태 — 향후 구현 시 ADC2_IN4 채널 설정 필요.

### Type II 루프 보상망 (COMP ↔ FB)

```
COMP(6) ──[R4: 49.9kΩ]──── FB(7) ──[R1=11kΩ]── VOUT
                              │
                           [C5: 10nF]    ← Type II zero
                              │
                            AGND(10)

선택: COMP(6) ──[C6: ≤100pF]── AGND(10)  (노이즈 억제 고주파 폴)
```

**보상망 계산 (데이터시트 참조값과 동일):**

```
Type II zero:  fZ = 1/(2π × R4 × C5) = 1/(2π × 49.9k × 10nF) = 319Hz

모듈레이터 극점: fp = 1/(2π × RLOAD × COUT) = 1/(2π × 50Ω × 242µF) = 13Hz

목표 크로스오버: f_cross = f_sw/10 = 100kHz/10 = 10kHz
  → fZ(319Hz) << f_cross(10kHz) ✓  데이터시트 조건 만족
```

> **⚠️ R4=49.9kΩ 적정성 검토:**
>
> - 데이터시트 예제(RLOAD=5Ω)와 우리 설계(RLOAD=50Ω)의 모듈레이터 DC Gain 차이:
>   - 데이터시트: 2×5 = 10 (20dB) / 우리 설계: **2×50 = 100 (40dB)** → 20dB 더 높음
> - 루프 게인이 높으므로 실제 보드에서 발진 확인 필요.
>   불안정 시 **R4를 22kΩ~33kΩ으로 낮추어** 에러앰프 게인 감소.
> - C5=10nF은 변경 불필요 (fZ만 R4에 비례하여 이동).

### 스위칭 주파수 설정 (RT 핀)

```
f_sw (kHz) = 3300 / R_T (kΩ)

예시:
├── R_T = 6.6kΩ  → f_sw = 500kHz
├── R_T = 10kΩ   → f_sw = 330kHz
├── R_T = 16.5kΩ → f_sw = 200kHz
├── R_T = 21kΩ   → f_sw = 157kHz  ← ★ 참고 회로도 표준값
└── R_T = 33kΩ   → f_sw = 100kHz  ← 48V 입력 시 권장 (최소 ON시간 여유확보)
```

> **R_T = 21kΩ vs 33kΩ 비교:**
>
> - **21kΩ (157kHz)**: 참고 회로도 표준. 48V에서 D_min=5.6%→t_on=357ns, LM5005 최소ON(200~300ns) 대비 여유 있음.
> - **33kΩ (100kHz)**: 동일 조건 t_on=560ns → 2배 여유. **48V 입력 전범위(0.15W~10W)에 더 안정적.**

### ⚠️ DC 48V 입력 시 주의사항

#### 듀티 사이클 제한

48V 입력에서 저출력(저전압) 시 최소 듀티 한계에 도달할 수 있습니다.

```
듀티 = V_out / V_in (이상적 BUCK)

├── V_out = 22.4V (10W)   → 듀티 = 22.4/48 = 46.7% ✓ 정상
├── V_out = 10V (2W)      → 듀티 = 10/48 = 20.8%   ✓ 정상
├── V_out = 5V (0.5W)     → 듀티 = 5/48 = 10.4%    ✓ 정상
├── V_out = 2.7V (0.15W)  → 듀티 = 2.7/48 = 5.6%   ⚠️ 경계
└── V_out = 2.2V (0.1W)   → 듀티 = 2.2/48 = 4.6%   ❌ 한계 초과 가능
```

#### 최소 ON 시간 검증

```
LM5005 최소 ON 시간 ≈ 200~300ns

├── f_sw = 330kHz → 주기 = 3µs
│   └── 듀티 4.6% → ON = 138ns ❌ 부족
│
└── f_sw = 100kHz → 주기 = 10µs

### 전력 제어 펌웨어 예시

```c
#include "stm32g4xx_hal.h"
#include <math.h>

#define LOAD_IMPEDANCE      50.0f    // 트랜스듀서 임피던스 (Ω)
#define V_REF               1.23f    // LM5005 FB 기준 전압
#define R1_VALUE            11000.0f // FB 상단 분압 저항 (Ω)
#define R2_VALUE            1000.0f  // FB 하단 분압 저항 (Ω)
#define R3_VALUE            1800.0f  // DAC 주입 저항 (Ω)
#define DAC_MAX_VALUE       4095     // 12비트 DAC
#define DAC_VREF            3.3f     // DAC 기준 전압

// V_out = V_REF × (1 + R1/R2) + (V_REF - V_dac) × R1/R3
// V_out = 14.76 + (1.23 - V_dac) × 6.11

// 전력 → DAC 값 변환 테이블 (캘리브레이션용)
static const struct {
    float power;    // W
    uint16_t dac;   // 12-bit
} power_table[] = {
    {0.1f,  4095},  // V_out ≈ 2.1V  (DAC=3.3V)
    {0.3f,  3700},  // V_out ≈ 3.9V  (DAC=3.0V)
    {0.5f,  3400},  // V_out ≈ 5.0V
    {1.0f,  3000},  // V_out ≈ 7.1V
    {2.0f,  2500},  // V_out ≈ 10V   (DAC=2.0V)
    {5.0f,  1500},  // V_out ≈ 15.8V (DAC=1.2V)
    {10.0f, 0},     // V_out ≈ 22.3V (DAC=0V)
};

/**
 * @brief 출력 전력 설정 (0.1W ~ 10W, 0.1W 단위)
 * @param power_watts 목표 전력 (W)
 */
void BUCK_SetPower(float power_watts)
{
    // 범위 제한
    if (power_watts < 0.1f) power_watts = 0.1f;
    if (power_watts > 10.0f) power_watts = 10.0f;

    // 전력 → 전압 변환 (P = V² / R)
    float target_voltage = sqrtf(power_watts * LOAD_IMPEDANCE);

    // 전압 → DAC 값 변환
    // V_out = V_OUT_MAX - (V_dac × DAC_GAIN)
    // V_dac = (V_OUT_MAX - target_voltage) / DAC_GAIN
    float v_dac = (V_OUT_MAX - target_voltage) / DAC_GAIN;

    // DAC 전압 → DAC 레지스터 값 (0~3.3V → 0~4095)
    if (v_dac < 0.0f) v_dac = 0.0f;
    if (v_dac > 3.3f) v_dac = 3.3f;

    uint16_t dac_value = (uint16_t)(v_dac / 3.3f * (float)DAC_MAX_VALUE);

    // DAC 출력 설정 (PA4)
    HAL_DAC_SetValue(&hdac1, DAC_CHANNEL_1, DAC_ALIGN_12B_R, dac_value);
}

/**
 * @brief 전력 피드백 제어 루프 (100ms 주기 호출)
 */
void BUCK_PowerControlLoop(void)
{
    static float error_integral = 0.0f;
    const float Kp = 0.5f;
    const float Ki = 0.1f;

    // ADC로 실제 전력 측정
    float actual_power = ADC_GetForwardPower();
    float target_power = g_target_power;

    // PI 제어
    float error = target_power - actual_power;
    error_integral += error * 0.1f;  // dt = 100ms

    // 적분 제한
    if (error_integral > 5.0f) error_integral = 5.0f;
    if (error_integral < -5.0f) error_integral = -5.0f;

    // 보정된 전력값 계산
    float corrected_power = target_power + (Kp * error) + (Ki * error_integral);

    // BUCK 출력 조정
    BUCK_SetPower(corrected_power);
}
```

## 10. PZT 출력 전력 검출 회로 (2단계 공진 탐색)

### 개요

PZT 트랜스듀서의 공진주파수 탐색 및 출력 전력 모니터링을 위한 **2단계 검출 구조**.

**1차 공진 탐색 — 전류 기반 주파수 스윕**:
발진 주파수를 공칭 주파수 ±10% 범위에서 스윕하며 BUCK 출력 전류(PA6, INA190A4)를
모니터링하여 **전류 최대점 = 공진점**으로 판단한다.
공진 시 직렬 임피던스가 최소가 되어 전류가 최대가 되는 물리 원리를 이용.

**2차 정밀 탐색 — CT + 쇼트키 브리지 검파**:
1차 탐색 결과 근처에서 CT + BAT54W 쇼트키 브리지 검파로
좌/우 채널(PA0, PA1) 밸런스 오차를 측정하여 **|PA0-PA1| 최소점**을 정밀 공진점으로 찾는다.
기존 LTC5507(RF 전력 검출기 IC)은 오버스펙·조달 불안정으로 제거하고,
0.5~2 MHz 대역에 적합한 단순 다이오드 검파 방식으로 대체한다.

| 항목      | 1차 (전류 기반)              | 2차 (CT + 다이오드)                 |
| --------- | ---------------------------- | ----------------------------------- | ------- | ----- |

### 10.1 1차 공진 탐색: 전류 기반 주파수 스윕

BUCK 출력 전류를 PA6 (ADC2_IN3, INA190A4 경유)으로 읽어 공진점을 탐색한다.
전류 센싱 하드웨어는 §9의 INA190A4 + R110(10mΩ) 경로를 그대로 사용.

### 10.2 2차 정밀 탐색: CT + 쇼트키 다이오드 검파기

1차 탐색 결과 주파수 근처(±2%)에서 브리지 좌/우 채널의 밸런스(|PA0-PA1|)로 정밀 공진점을 찾는다.

### 브릿지 회로 동작 원리

CT 2차측 전류가 브릿지 저항망을 통해 분배됨.
출력 트랜스 2차측 전류 조건에 따라 좌/우 브리지 노드에 차동 전압이 발생한다.

**R41~R46 표기 기준 연결 규칙 (PZT_OUTPUT_DETECT_Schottky.jpg 기준):**

- Pin1 노드 = CT2 2차측 Pin1 = R41 상단 = R43/R44 상단
- Pin2 노드 = CT2 2차측 Pin2 = R42 상단 = R45/R46 상단
- R41, R42 하단만 GNDPZT 공통 노드로 연결
- R43, R44는 좌측 브리지 내부 병렬 (우측과 직접 연결 금지)
- R45, R46는 우측 브리지 내부 병렬 (좌측과 직접 연결 금지)
- C14, C15는 Pin1/Pin2가 아니라 CT 1차측(고전력 라인)에서 GND로 샘플링
- 즉, R41~R46의 한쪽이 모두 하나의 공통 노드로 합쳐지면 안 됨

**등가 저항 계산:**

```
R43 // R44 = 200Ω // 200Ω = 100Ω   (좌측 브리지 분기 임피던스)
R45 // R46 = 200Ω // 200Ω = 100Ω   (우측 브리지 분기 임피던스)
R_load = R41 // (R43 // R44) = 200Ω // 100Ω = 66.7Ω  (CT Pin1 기준 합성 부하)
```

**채널 방향 라벨 주의:**

- 좌측(PA0)을 순방향, 우측(PA1)을 역방향으로 고정 정의하지 않는다.
- CT 권선 방향/배선 극성에 따라 좌우 의미가 바뀔 수 있으므로, 초기 캘리브레이션으로 라벨을 확정한다.
- 공진점 판정은 라벨과 무관하게 **|PA0-PA1| 최소** 기준으로 수행한다.

> **R41~R46 전부 200Ω 통일**: BOM 1종으로 단순화.
> 기존 50Ω/25Ω 대비 신호 레벨 55%(500kHz)~220%(2MHz) 향상.

**AC 커플링 고역통과 필터 (C73, C74 = 100nF):**

```
fc = 1 / (2π × R_source × C)
   = 1 / (2π × 100Ω × 100nF)
   ≈ 16 kHz

500kHz (Ch0)에서: -0.005 dB (완전 평탄)
2MHz (Ch6)에서: -0.0003 dB (완전 평탄)
```

> R_source가 12.5Ω→100Ω로 증가했으나, fc가 오히려 127kHz→16kHz로 낮아져
> 전 채널에서 더욱 평탄한 응답 확보.

#### 다이오드 검파 신호 레벨 계산

하프브리지 출력 → CT(1:10) → 브릿지 → 다이오드 검파 → ADC 변환:

```
부하 50Ω 가정, BAT54W Vf ≈ 240mV @1mA
CT 합성 부하: R_load = 200Ω // 100Ω = 66.7Ω
CT 인덕턴스: L_m = 6.8μH (61재, 10턴)

             ────── 500kHz ──────   ────── 1MHz ──────   ────── 2MHz ──────
전력  I_sec  V_pk   V_det   ADC    V_pk   V_det   ADC    V_pk   V_det   ADC
(W)   (mA)   (mV)   (mV)   (12b)  (mV)   (mV)   (12b)  (mV)   (mV)   (12b)
────  ─────  ─────  ─────  ─────  ─────  ─────  ─────  ─────  ─────  ─────
0.15  5.48   157    0      0      209    0      0      277    37     46
0.30  7.75   222    0      0      296    56     70     392    152    189
0.50  10.0   287    47     58     382    142    176    506    266    330
1.00  14.1   405    165    205    540    300    372    715    475    590

  V_pk(f) = I_sec × R_load × ωL_m / √(R_load² + (ωL_m)²) × √2
  V_detect ≈ max(0, V_pk - Vf)
  ADC = V_detect / 3.3V × 4095
```

**감도 고려:**

- **500kHz**: 0.5W 이상에서 검파 동작 시작 → **2차 탐색 시 0.5W 권장**
- **1MHz 이상**: 0.3W에서도 56mV 이상 → **0.3W로 충분**
- **주파수별 2차 탐색 전력 자동 조정** 권장 (500~700kHz: 0.5W, 700kHz~2MHz: 0.3W)
- 1차 탐색은 전류(PA6)로 수행하므로 다이오드 저전력 한계는 문제 없음

### BAT54W 다이오드 검파 출력 특성

BAT54W는 **쇼트키 다이오드 반파 피크 검출기**로, 출력이 입력 피크 전압에 **선형 비례**.
(LTC5507의 로그 출력과 달리 역변환 불필요)

```
                  BAT54W 반파 검파 전달 특성

    V_out (mV DC)
       │
   800 ┼                                    ╱
       │                                  ╱
   600 ┼                                ╱     ← 선형 영역
       │                              ╱        (V_in > Vf 이후)
   400 ┼                            ╱
       │                          ╱
   200 ┼                        ╱
       │                  ····· ← Vf 사각지대 (~240mV)
     0 ┼──────────────···
       └────┬────┬────┬────┬────┬────┬────▶ V_in_peak (mV)
           100  200  300  500  700  1000

    V_out ≈ V_in_peak - Vf    [V_in > Vf 영역]
    V_out ≈ 0                  [V_in < Vf 영역]
    Vf(BAT54W) ≈ 240mV @1mA, ~200mV @0.1mA
```

**ADC 변환 (12비트, 3.3V 기준):**

```
ADC_value = V_detect / 3.3V × 4095

V_detect → V_bridge_peak 역변환 (필요 시):
  V_bridge_peak = V_detect + Vf ≈ ADC × 3.3 / 4095 + 0.24 [V]

VSWR 계산에는 역변환 불필요 — FWD/REV ADC값의 비율만 사용:
  Γ ≈ √(ADC_rev / ADC_fwd)
  VSWR = (1 + Γ) / (1 - Γ)
```

> **핵심 장점**: 선형 검파기이므로 FWD/REV ADC 원시값으로 직접 VSWR 계산 가능.
> LTC5507 로그 출력처럼 dBm→mW 역변환이 불필요하여 펌웨어 단순화.

### 공진주파수 탐색 알고리즘 (2단계)

PZT 트랜스듀서의 직렬 공진점에서 **임피던스 최소 → 전류 최대 → VSWR 최소**.

#### 1단계: 전류 기반 공진 탐색 (I_max)

```c
/**
 * @brief 전류 기반 공진주파수 탐색 (1차)
 *        공칭 주파수 ±10% 범위를 스윕, 전류 최대점 = 공진점
 * @param nominal_khz  공칭 주파수 (kHz), 예: 2000
 * @param step_khz     스윕 스텝 (kHz), 예: 5
 * @return 공진주파수 (kHz), 0이면 탐색 실패
 */
uint32_t FindResonance_CurrentMax(uint32_t nominal_khz, uint16_t step_khz)
{
    uint32_t start = nominal_khz * 90 / 100;   // -10%
    uint32_t end   = nominal_khz * 110 / 100;  // +10%
    uint32_t best_freq = 0;
    uint16_t max_current = 0;

    if (start < FREQ_MIN_KHZ) start = FREQ_MIN_KHZ;
    if (end > FREQ_MAX_KHZ)   end = FREQ_MAX_KHZ;

    MegasonicCtrl_SetPower(RESCAN_POWER_01W);  // 저전력 (0.15W)
    HRTIM_OutputStart();

    for (uint32_t freq = start; freq <= end; freq += step_khz) {
        HRTIM_SetFrequency(freq);
        HAL_Delay(RESCAN_FREQ_SETTLE_MS);      // 10ms 안정화

        uint16_t current = ADC_GetCurrent_Raw(); // PA6 (INA190A4)

        if (current > max_current) {
            max_current = current;
            best_freq = freq;
        }
    }

    HRTIM_OutputStop();
    return best_freq;
}
```

**탐색 파라미터 예시 (Ch6, 2MHz PZT):**

```
공칭 주파수: 2000 kHz
스윕 범위:   1800 ~ 2200 kHz (±10%)
스텝:        5 kHz
포인트 수:   (2200-1800)/5 + 1 = 81포인트
소요 시간:   81 × 10ms = ~0.8초
```

#### 2단계: 브리지 밸런스 정밀 탐색 (|PA0-PA1| 최소)

1차 결과 근처(±2%)에서 다이오드 검파 좌/우 채널 차이(|PA0-PA1|) 최소점을 정밀 탐색.

```c
/**
 * @brief 브리지 밸런스 오차 계산
 *        공진점 근처에서 좌/우 채널이 평형이므로 |PA0-PA1| 최소값을 사용
 */
uint16_t CalculateBalanceError(uint16_t left, uint16_t right)
{
  if (left > right) return (uint16_t)(left - right);
  return (uint16_t)(right - left);
}

/**
 * @brief 2단계 브리지 밸런스 정밀 탐색
 *        1차 결과 ±2% 범위를 2kHz 스텝으로 정밀 스윕
 * @param coarse_khz  1차 탐색 결과 주파수 (kHz)
 * @return 정밀 공진주파수 (kHz), 0이면 탐색 실패
 */
uint32_t FindResonance_BalanceFine(uint32_t coarse_khz)
{
    uint32_t start = coarse_khz * 98 / 100;    // -2%
    uint32_t end   = coarse_khz * 102 / 100;   // +2%
    uint32_t best_freq = coarse_khz;
  uint16_t min_err = 0xFFFF;

    if (start < FREQ_MIN_KHZ) start = FREQ_MIN_KHZ;
    if (end > FREQ_MAX_KHZ)   end = FREQ_MAX_KHZ;

    // 주파수별 전력 자동 조정: 500~700kHz는 0.5W, 700kHz 이상은 0.3W
    uint16_t tune_power = (coarse_khz < 700) ? RESCAN_POWER_VSWR_LF : RESCAN_POWER_VSWR;
    MegasonicCtrl_SetPower(tune_power);
    HRTIM_OutputStart();

    for (uint32_t freq = start; freq <= end; freq += 2) {
        HRTIM_SetFrequency(freq);
        HAL_Delay(RESCAN_FREQ_SETTLE_MS);

      uint16_t left = ADC_GetForwardPower_Raw();   // PA0 (좌측 채널)
      uint16_t right = ADC_GetReversePower_Raw();  // PA1 (우측 채널)
      uint16_t err = CalculateBalanceError(left, right);

      if (err < min_err) {
        min_err = err;
            best_freq = freq;
        }
    }

    HRTIM_OutputStop();
    return (min_err <= RESCAN_BALANCE_OK_ADC) ? best_freq : 0;
}
```

#### 통합 2단계 탐색 흐름

```c
/**
 * @brief 2단계 공진주파수 탐색 (전류 1차 → 밸런스 2차)
 */
uint32_t FindResonance_TwoPhase(uint32_t nominal_khz)
{
    // 1단계: 전류 기반 광역 탐색 (±10%, 5kHz 스텝)
    uint32_t coarse = FindResonance_CurrentMax(nominal_khz, 5);
    if (coarse == 0) return 0;

    // 2단계: 밸런스 기반 정밀 탐색 (±2%, 2kHz 스텝)
    uint32_t fine = FindResonance_BalanceFine(coarse);

    return (fine != 0) ? fine : coarse;  // 2차 실패 시 1차 결과 사용
}
```

```
탐색 흐름:

  ┌─────────────────────────────────────────────────┐
  │  1차: 전류 기반 광역 탐색                       │
  │  범위: f₀ ±10%, 스텝: 5kHz, 전력: 0.15W        │
  │  기준: PA6 전류 최대점 → f_coarse               │
  └──────────────────────┬──────────────────────────┘
                         │

  ┌─────────────────────────────────────────────────┐
  │  2차: 브리지 밸런스 정밀 탐색 (옵션)            │
  │  범위: f_coarse ±2%, 스텝: 2kHz                 │
  │  전력: 500~700kHz→0.5W, 700kHz~2MHz→0.3W       │
  │  기준: |PA0-PA1| 최소점 → f_fine                │
  └──────────────────────┬──────────────────────────┘

              f_resonance = f_fine (또는 f_coarse)

```

### 주의사항

1. **안정화 시간**: 주파수 변경 후 최소 10ms 대기 필요 (LC 매칭 + BUCK 출력 안정화)
2. **1차 탐색 전력**: 0.15W 저전력으로 수행 (비공진 상태에서 트랜스듀서 과전류 방지)
3. **2차 탐색 전력**: 주파수별 자동 조정 — 500~700kHz: 0.5W, 700kHz~2MHz: 0.3W
   (CT L_m=6.8μH에 의해 저주파 신호가 약하므로, 500~700kHz에서는 전력을 높여 다이오드 Vf 초과 확보)
4. **온도 의존성**: PZT 공진주파수는 온도에 따라 ±1~2% 변동 → 주기적 재탐색 권장
5. **케이블 영향**: 케이블 길이에 따라 위상 지연 발생 → 설치 후 초기 캘리브레이션 필수
6. **2차 탐색 생략**: 1차 전류 탐색만으로도 대부분의 PZT에서 충분한 공진점 검출 가능.
   2차 밸런스 탐색은 고정밀 요구 시 또는 브리지 대칭점 확인 목적으로 사용

#### 브리지 밸런스 검출 원리 (저항 브릿지 방식)

센터탭 대신 **외부 저항 브릿지(R43~R46)**로 좌/우 채널 차동 출력을 생성합니다:

```
  메인 전력선 (T2 → PZT)                    메인 전력선 (T2 → PZT)
         │                                        │
        [C14]                                    [C15]
        10pF ← ⚠️ 메인 전력선에 연결!           10pF ← ⚠️ 메인 전력선에 연결!
         │                                        │
        GND                                      GND

                    CT 2차측 (10T, 센터탭 없음)
                           │           │
                         Pin1        Pin2
                           │           │
                          [R41]      [R42]
                         200Ω       200Ω
                           │           │
                          GND         GND

                    ┌────┴────┐              ┌────┴────┐
                    │         │              │         │
                  [R43]     [R44]          [R45]     [R46]
                  200Ω      200Ω           200Ω      200Ω
                    │         │              │         │
                    └────┬────┘              └────┬────┘
                         │                        │
                     Bridge-A (좌측)         Bridge-B (우측)
                         │                        │
                       [C73]                    [C74]
                       100nF                    100nF
                         │                        │
                     ──|>|── D12             ──|>|── D13
                      BAT54W                  BAT54W
                         │                        │
                       [R47]  [C75]             [R48]  [C76]
                       10kΩ   100nF             10kΩ   100nF
                         │     │                  │     │
                        GND   GND                GND   GND
                         │                        │
                        PA0                      PA1
```

**동작 원리:**

- CT의 Pin1, Pin2를 각각 R41, R42(200Ω)로 GND에 종단
- R43//R44 = R45//R46 = 100Ω 병렬 저항이 브릿지 형성 (R41~R46 전부 200Ω 통일)
- CT 합성 부하: R_load = 200Ω // 100Ω = 66.7Ω
- 좌/우 채널의 절대 라벨(FWD/REV)은 CT 권선 극성에 따라 바뀔 수 있으므로 고정하지 않음
- 공진점 판정은 **|PA0-PA1| 최소** 조건으로 수행

> **⚠️ [CRITICAL] C14/C15 (10pF) 연결 위치 주의**
>
> C14/C15는 **반드시 메인 전력선(CT 1차측, OUTPUT_TRANS → PZT 사이의 고전력 라인)** 에서 GND로 연결해야 합니다.
> CT 2차측 Pin1/Pin2에 연결하면 **전압 커플링이 사라져 좌/우 브리지 채널의 차동 특성이 무너집니다**.
>
> ```
> ✅ 올바른 연결:  메인 전력선(CT 1차측 L1/L2 옆) ── C14/C15 ── GND
> ❌ 잘못된 연결:  CT 2차측 Pin1/Pin2 ── C14/C15 ── GND
> ```
>
> 브리지 원리: **전압 샘플(C14/C15, 메인 라인)** + **전류 샘플(CT 2차측)** 이 결합되어
> 좌/우 채널 차동 신호를 만듭니다. 공진점 근처에서는 두 채널의 차이가 최소가 됩니다.
> C14/C15가 CT 2차측에 연결되면 전압 기준이 사라져 두 채널 분해능이 크게 떨어집니다.

**센터탭이 필요 없는 이유:**
| 센터탭 방식 | 저항 브릿지 방식 (현재 회로) |
|------------|------------------------------|
| CT 권선 복잡 (센터탭 정밀 위치) | **CT 권선 단순 (양 끝단만)** |
| CT 내부에서 방향 분리 | **외부 저항으로 방향 분리** |
| 밸런스 조정 어려움 | **저항값으로 쉽게 조정** |
| 자작 시 정밀도 요구 | **자작 용이** |

### 11.1 LCD1602 + LED 제어 (74HCT574 ×2 — 방법 C)

#### 설계 방식

디스플레이 보드를 MCU 보드와 분리하여 별도 PCB로 제작.
**74HCT574 ×2 (8비트 D 플립플롭 래치)** 를 사용하여 공유 6비트 데이터 버스로
LCD(6핀)와 LED(6개)를 각각 독립 래치를 통해 제어.

- **74HCT574 전원: 5V** — HCT 계열은 TTL 입력 문턱(V_IH = 2.0V 고정)을 사용하므로
  MCU 3.3V 출력(최소 2.97V)이 5V VCC에서도 HIGH로 정상 인식됨
- **LCD1602 VDD: 5V 필수** — HD44780 동작 범위 4.5~5.5V. 3.3V 공급 시 동작 불안정
- **LCD 백라이트(LED+)**: 5V + 33Ω 직렬 저항 → 전류 제한 ~60mA 이내
- **버튼 풀업: 외부 10kΩ 풀업 저항** (3.3V ← R_PU 10kΩ ← 버튼 핀) — MCU 내부 풀업 미사용
- **액티브 부저**: KEC105S (NPN, 내장 베이스 저항) 트랜지스터 구동 — 디스플레이 보드 탑재
- **커넥터: 20핀** — 기존 16핀 + ⑰ BUZZER(PB5) + ⑱ 5V + ⑲ 3.3V + ⑳ GND

### 11.2 외부 제어 입력 절연 (TLP281-4 광커플러)

#### 필요성

D-SUB 25핀 커넥터를 통해 외부 PLC, 릴레이 접점 등에서 24V 신호가 인가될 수 있음.
**MCU 보호 및 그라운드 루프 차단**을 위해 광커플러 절연 필수.

#### 외부 입력 5개 (광커플러 TLP281-4 ×1 + TLP181 ×1)

| 채널 | 신호         | 기능                     | MCU 핀 | IC              |
| ---- | ------------ | ------------------------ | ------ | --------------- |
| CH1  | REMOTE       | 발진 ON/OFF (4-COM 겸용) | PA15   | TLP281-4 (CH1)  |
| CH2  | BCD_BIT1     | 8단계 출력 선택 비트1    | PC10   | TLP281-4 (CH2)  |
| CH3  | BCD_BIT2     | 8단계 출력 선택 비트2    | PC11   | TLP281-4 (CH3)  |

#### 저항값 계산

```
외부 24V 기준:
├── LED 전류 = (24V - 1.2V) / 4.7kΩ ≈ 4.9mA ✓ (권장 5~10mA)
├── CTR(전류전달비) = 50% (최소) → 출력 2.5mA 이상
└── 풀업 10kΩ → VOL ≈ 0.3V, VOH ≈ 3.3V ✓

외부 12V 기준:
├── LED 전류 = (12V - 1.2V) / 4.7kΩ ≈ 2.3mA
└── 저전류이지만 동작 가능 (CTR 여유 있음)

외부 5V 기준:
├── LED 전류 = (5V - 1.2V) / 4.7kΩ ≈ 0.8mA
└── 권장: R_LED = 1kΩ으로 변경 (3.8mA)
```

#### 입력 극성 (Active LOW)

- **접점 OPEN**: LED OFF → 트랜지스터 OFF → MCU 입력 = HIGH (풀업)
- **접점 SHORT**: LED ON → 트랜지스터 ON → MCU 입력 = LOW

> 매뉴얼과 동일: REMOTE **SHORT=발진ON**, **OPEN=발진OFF**

## 13. 주의사항 요약

### 클럭 소스 (HSI vs HSE 상세 분석)

#### 개요

STM32G474MET6는 두 가지 고속 클럭 소스를 지원합니다:

| 항목           | HSI (내부 RC)                | HSE (외부 크리스탈)          |
| -------------- | ---------------------------- | ---------------------------- |
| 주파수         | 16 MHz                       | 4~48 MHz (외부 부품)         |
| 정밀도 (25°C)  | ±1% (공장 트리밍)            | ±20ppm (크리스탈 의존)       |
| 온도 드리프트  | ±3% (-40~105°C 전 범위)      | ±50ppm (-40~85°C, AT컷 기준) |
| 외부 부품      | 없음                         | 크리스탈 + 부하 커패시터 2개 |
| 시작 시간      | ~2µs                         | ~2ms (크리스탈 안정화)       |
| 사용 핀        | 없음                         | **PF0-OSC_IN, PF1-OSC_OUT**  |
| PLL 입력       | ✅                           | ✅                           |
| HRTIM DLL 영향 | DLL이 자체 캘리브레이션 수행 | 더 안정적인 기준 클럭 제공   |

#### 핵심 문제: PF0/PF1 핀 충돌

LQFP80 패키지에서 HSE 크리스탈을 사용하면 **PF0, PF1이 GPIO로 사용 불가**합니다:

```
STM32G474MET6 LQFP80

    핀 11: PF0 / OSC_IN   ← HSE 사용 시 크리스탈 연결 (GPIO 불가)
    핀 12: PF1 / OSC_OUT  ← HSE 사용 시 크리스탈 연결 (GPIO 불가)
```

**현재 설계에서 PF0/PF1은 LC탱크 릴레이로 할당**:

| 핀  | 현재 용도        | HSE 사용 시     |
| --- | ---------------- | --------------- |
| PF0 | LC_RELAY3 (GPIO) | ❌ OSC_IN 전용  |
| PF1 | LC_RELAY4 (GPIO) | ❌ OSC_OUT 전용 |

#### HSI 선택 시 영향 분석

**1. HRTIM PWM 주파수 정밀도**

```
PLL 설정: HSI 16MHz → PLL → 170MHz SYSCLK
HRTIM DLL: 170MHz × 32 = 5.44GHz 유효 클럭

HSI ±1% 오차가 PWM 주파수에 미치는 영향:
├── 목표 500kHz → 실제 495~505 kHz (±5kHz)
├── 목표 1000kHz → 실제 990~1010 kHz (±10kHz)
└── 목표 2000kHz → 실제 1980~2020 kHz (±20kHz)

HSE ±20ppm 일 경우:
├── 목표 500kHz → 실제 499.99~500.01 kHz (±0.01kHz)
├── 목표 1000kHz → 실제 999.98~1000.02 kHz
└── 목표 2000kHz → 실제 1999.96~2000.04 kHz
```

> 메가소닉 트랜스듀서는 **공진주파수 대역폭(Q값)이 수 kHz~수십 kHz**이므로,
> HSI ±1% 오차(±5~20kHz)는 대역폭 내에서 충분히 허용됩니다.
> 또한 **실시간 전력 피드백(전류 센싱 + 다이오드 검파)**으로 공진점을 추적하므로 절대 정밀도보다 상대 분해능이 중요합니다.

**2. Modbus RTU 보레이트 정밀도**

```
USART2 보레이트: 9600 bps (또는 19200/38400)

RS-485 허용 오차: ±3% (일반적 UART 동기화 마진)

HSI ±1% → USART 보레이트 오차 ±1% → ✅ 3% 이내 (충분)
HSI ±3% (극한 온도) → USART 보레이트 오차 ±3% → ⚠️ 경계

HSE ±20ppm → USART 보레이트 오차 무시 수준
```

> 산업 현장 온도(0~60°C)에서 HSI 오차는 ±1.5% 이내이므로 Modbus 통신에 문제 없습니다.
> 극한 환경(-40~105°C)에서만 HSE가 필요합니다.

**3. ADC 샘플링 타이밍**

```
ADC 클럭 = SYSCLK / 분주 (보통 /4 = 42.5MHz)

HSI ±1% → ADC 변환 시간 ±1% → 측정값 자체에는 영향 없음
(ADC는 S&H 방식이므로 클럭 주파수가 아닌 기준 전압 정밀도가 중요)
```

> ADC 정밀도는 VREF+ 전압에 의존하며, 클럭 오차의 영향은 무시할 수 있습니다.

#### HSE가 필요한 경우와 대응

만약 향후 HSE가 필요하게 되면:

**대응 방안 A: LC_RELAY3/4 핀 재배치**

| 기존 핀 | 기존 용도 | 이동 핀 | 비고        |
| ------- | --------- | ------- | ----------- |
| PF0     | LC_RELAY3 | **PC8** | 여유핀 활용 |
| PF1     | LC_RELAY4 | **PC9** | 여유핀 활용 |

```
HSE 크리스탈 회로:
        PF0 (OSC_IN)          PF1 (OSC_OUT)
            │                      │
       ┌────┴────┐            ┌────┴────┐
       │ [CL1]   │            │ [CL2]   │
       │ 10~20pF │            │ 10~20pF │
       │         │            │         │
       └────┬────┘            └────┬────┘
            │                      │
            └──────────────────────┘
                      │
                     GND

    CL1, CL2: 부하 커패시터 (크리스탈 데이터시트 참조)
    권장 크리스탈: 8MHz (PLL로 170MHz 생성)
```

**대응 방안 B: HSI 캘리브레이션 강화**

HSE 없이 정밀도를 높이는 방법:

1. **공장 트리밍 활용**: RCC_ICSCR 레지스터의 HSITRIM[6:0] 비트로 ±0.5% 이내 조정 가능
2. **온도 보상 테이블**: 온도 센서(ADC 내장)로 HSI 편차를 소프트웨어 보정
3. **외부 기준 신호**: Modbus 마스터 통신 타이밍으로 주기적 보정

#### 결론 — HSE 8MHz 크리스탈 채택 (v3)

| 항목                | HSE (현재 설계)                            | HSI (이전 설계) |
| ------------------- | ------------------------------------------ | --------------- |
| **정밀도**          | ⭐ ±20ppm                                  | ±1% (10,000ppm) |
| **PF0/PF1**         | 크리스탈 전용                              | GPIO 사용 가능  |
| **LC탱크 릴레이**   | PC8/PC9로 이동                             | PF0/PF1 할당    |
| **PWM 500kHz 오차** | ±0.01kHz                                   | ±5kHz           |
| **Modbus 통신**     | 무시 수준                                  | ±1%             |
| **외부 부품**       | 크리스탈+C×2 추가                          | 없음            |
| **여유핀**          | 7개 (PA5, PA10, PB2, PB8, PB9, PC13, PC14) | 5개             |

> ✅ **현재 설계는 HSE 8MHz 크리스탈 사용 (v3)**.
> PF0(OSC_IN)/PF1(OSC_OUT)에 8MHz 크리스탈 연결.
> LC_RELAY3/4는 PC8/PC9로 재배치 완료.

#### HSE 크리스탈 회로

```
                     3.3V (또는 VDD)
                           │
                     ┌─────┴─────┐
                     │ [R_fb]    │
                     │ 1MΩ      │ ← 피드백 저항 (발진 안정화)
                     │           │
    PF0 (OSC_IN) ────┼─────┬─────┼──── PF1 (OSC_OUT)
         핀 11       │     │     │      핀 12
                     │   ┌─┴─┐   │
                     │   │   │   │
                     │   │ Y1│   │ ← 8 MHz 크리스탈
                     │   │   │   │   HC49/S 또는 3215 패키지
                     │   └─┬─┘   │
                     │     │     │
                ┌────┴─┐   │   ┌─┴────┐
                │[CL1] │   │   │[CL2] │
                │12pF  │   │   │12pF  │ ← 부하 커패시터 (C0G/NP0)
                └──┬───┘   │   └──┬───┘
                   │       │      │
                   └───────┴──────┘
                           │

```

**부품:**

| 부품  | 사양                        | 수량 | 비고                        |
| ----- | --------------------------- | ---- | --------------------------- |
| Y1    | 8 MHz, HC49/S 또는 3215 SMD | 1    | CL = 10~20pF, ESR < 100Ω    |
| CL1/2 | 12pF, C0G/NP0, 50V          | 2    | CL = 2×(C_load - C_stray)   |
| R_fb  | 1MΩ (옵션)                  | 1    | 발진 안정화, 일부 설계 생략 |

**부하 커패시터 계산:**

```
CL = 크리스탈 데이터시트 Load Capacitance (보통 10~20pF)
C_stray = PCB 기생 커패시턴스 (보통 2~5pF)

CL_ext = 2 × (CL - C_stray)
       = 2 × (10pF - 3pF) = 14pF → 표준값 12pF 선택

예시 (CL=10pF 크리스탈):  CL1 = CL2 = 12pF
예시 (CL=20pF 크리스탈):  CL1 = CL2 = 33pF
```

**PCB 배치 주의:**

1. CL1, CL2는 PF0/PF1 핀에서 **5mm 이내** 배치
2. 크리스탈~커패시터 GND 비아는 **최단 거리**
3. 고주파 신호 배선(HRTIM 등)과 **격리** (크로스토크 방지)
4. 크리스탈 아래 GND 플레인 확보 (차폐 효과)

### 15.2 리셋 회로 (NRST)

```
        3.3V
         │
        [R_RST: 10kΩ]  ← 풀업 저항
         │
         ├───────────┬─────────────────────────▶ NRST (핀 7)
         │           │
        [C_RST]     [D_RST]
        100nF       BAV99                      ← ESD 보호 다이오드
         │           │
        GND        3.3V
         │
         │
       ┌─┴─┐
       │ ○ │  ← SW_RST (리셋 버튼, 옵션)
       │   │     택트 스위치
       └─┬─┘
         │
        GND
```

**리셋 시간 상수:**

```
τ = R × C = 10kΩ × 100nF = 1ms

전원 투입 시 NRST가 LOW로 유지되는 시간:
t_reset ≈ 5τ = 5ms (안전한 리셋 보장)
```

**참고:**

- STM32G4 내부에는 NRST 풀업이 있으나, 외부 풀업 추가 권장
- 리셋 버튼은 PCB 테스트/디버그용 (양산 시 생략 가능)

## 17. LED 직접 구동 회로

### 17.1 LED 구동 방식 (74HCT574 래치)

현재 설계의 LED는 STM32 GPIO 직결이 아니라, 74HCT574 #2 래치를 통해 5V 도메인에서 구동한다.

```
STM32 (3.3V)                   74HCT574 #2 (5V)                 LED
PB10~PB15(DATA) ─────────────▶ D0~D5                     Q0~Q5 ──[560Ω]──▶ LED ── GND
PD9 (CLK_LED)  ──────────────▶ CLK
```

> LED ON/OFF는 `PB10~PB15` 데이터 비트와 `PD9` 래치 펄스로 제어된다.

### 17.2 저항값 계산 (5V 레일 기준)

**공식:**

```
R = (V_LED_RAIL - V_f) / I_f

V_LED_RAIL = 5.0V
V_f = LED 순방향 전압
I_f = LED 전류
```

**560Ω 단일값 사용 시 전류 예시:**

| LED 색상      | V_f (typ) | I_f 계산               | 결과 |
| ------------- | --------- | ---------------------- | ---- |
| 적색 (Red)    | 2.0V      | (5.0-2.0)/560 = 5.36mA | ✅   |
| 황색 (Yellow) | 2.1V      | (5.0-2.1)/560 = 5.18mA | ✅   |
| 녹색 (Green)  | 2.2V      | (5.0-2.2)/560 = 5.00mA | ✅   |

> ⚠️ **청색/백색 LED**: Vf가 3.2V로 높아 전압 마진이 0.1V밖에 안됨.
> GPIO VOH가 2.9V까지 떨어질 수 있으므로 **적색/녹색/황색** 권장.

### 17.3 6개 LED 구성

| LED 번호 | MCU 제어 비트 (74HCT574 #2) | 기능    | LED 색상 | 저항값 |
| -------- | --------------------------- | ------- | -------- | ------ |
| LED1     | PB10(DATA0) + PD9(CLK_LED)  | NORMAL  | 녹색     | 560Ω   |
| LED2     | PB11(DATA1) + PD9(CLK_LED)  | H/L SET | 황색     | 560Ω   |
| LED3     | PB12(DATA2) + PD9(CLK_LED)  | 8POWER  | 황색     | 560Ω   |
| LED4     | PB13(DATA3) + PD9(CLK_LED)  | REMOTE  | 녹색     | 560Ω   |
| LED5     | PB14(DATA4) + PD9(CLK_LED)  | EXT     | 녹색     | 560Ω   |
| LED6     | PB15(DATA5) + PD9(CLK_LED)  | RX      | 적색     | 560Ω   |

### 17.4 전류 합계 검증

```

├── 개별 GPIO 핀: 최대 25mA → 10mA ✓ 충분

```

**권장 TVS (아래 3종 중 1개만 선택 실장):**

| 부품      | 사양              | 특징                   |
| --------- | ----------------- | ---------------------- |
| SM712     | -7V~+12V (비대칭) | RS-485 전용, A/B 차동  |
| SMBJ6.0CA | ±6V 클램핑        | 범용 양방향            |

> 위 3개는 대체 옵션이며 동시에 3개를 모두 실장하지 않는다.
> 권장 우선순위: SM712 1순위, 수급 이슈 시 PESD1CAN 또는 SMBJ6.0CA 대체.

> DE/RE 제어 정책: 기본은 RX 모드(DE=LOW, /RE=LOW), 송신 프레임 직전에만 DE=HIGH, /RE=HIGH.
> 송신 완료 후 즉시 RX 모드로 복귀하여 반이중 충돌을 방지한다.

## 21. 하프브리지 출력단 부품 최적화

> C58, C59, FC1, FC2, R29, R30, R31, C60 — 캐패시터 결합 하프브리지 출력 네트워크.
> MOSFET: STD25N10F7 (100V/25A, TO-252), 변압기: T2 OUTPUT_Transformer.

### 21.1 회로 토폴로지

Q2(하이사이드)와 Q3(로우사이드)가 VBUS~GNDB 사이에서 스위칭한다.
분할 캐패시터(FC1/FC2)의 중간점과 SW 노드 사이에서 DC 블로킹 캐패시터(C60)를 통해 T2 변압기 1차측을 구동한다.

```
VBUS ──┬── Q2(D) ── Q2(S) ── SW node ────────────────────┐
       │                                                   │
       ├── C58(0.1µF) ─┬── FC1 ──┬── Cap Midpoint ──── T2:1
       │                │         │                        │
       │                │    R29(bleed)                    T2:2 ── PZT (50Ω)
       │                │         │                        │
       ├── C59(0.1µF) ─┴── FC2 ──┤── R30(bleed)          │
       │                          │                        │
       │   Q3(D) ── Q3(S) ── GNDB│── R31(bleed)          │
       │                          │                  C60(DC block)
       └──────────────────────────┘                        │
                                                        GNDPZT
```

### 21.2 동작 조건

| 항목 | 값 |
| --- | --- |
| 주파수 | 500 kHz ~ 2 MHz (7채널, params.h) |
| VBUS | 2.5V ~ 25V (LM5005 BUCK 출력) |

| MOSFET | STD25N10F7 (100V/25A, TO-252) |

### 21.3 부품별 분석

#### C58, C59 — 고주파 바이패스 캐패시터

MOSFET 스위칭 과도 전류(di/dt)를 흡수하는 고주파 바이패스.

| 주파수 | 100nF Xc | 10nF Xc |
| --- | --- | --- |
| 500 kHz | 3.2Ω | 32Ω |
| 1 MHz | 1.6Ω | 16Ω |
| 2 MHz | 0.8Ω | 8Ω |

- **현재값**: 0.1µF / 100V
- **권장값**: **0.1µF / 100V C0G(NP0) MLCC** (현재값 유지)
- **추가**: 각각 **10nF / 100V C0G** 병렬 (고차 고조파 10MHz+ 대역 ESL 보완)
- C0G/NP0 유전체 필수 — X5R/X7R는 DC 바이어스·온도 계수에 의해 실효 용량 30~50% 감소
- MOSFET 드레인-소스 직근 배치 (전류 루프 면적 최소화)

#### FC1, FC2 — 벌크 전해 캐패시터

DC 버스 분할 중간점 전압 안정화 + 과도 에너지 저장.

| 항목 | 1000µF (현재) | 470µF (권장) | 220µF (최소) |
| --- | --- | --- | --- |
| 리플 전압 (10W, 500kHz) | 0.8mV | 1.7mV | 3.6mV |
| 방전 시정수 (R=15kΩ) | 15s | 7.1s | 3.3s |
| 중간점 드리프트 (5mA 불균형) | 5 V/s | 10.6 V/s | 22.7 V/s |

- **현재값**: 1000µF / 63V
- **권장값**: **330µF ~ 470µF / 50V, 저ESR 전해** (Panasonic FR/FC 시리즈)
- **추가**: 각각 **1µF / 50V X7R 세라믹** 병렬 (전해 ESR/ESL과 세라믹 바이패스 사이 임피던스 갭 브릿지)
- 1000µF은 과대 — 500kHz 이상에서 전해의 실효 임피던스는 ESL 지배적, 고주파 바이패스는 C58/C59가 담당
- 50V 내압: VBUS max 25V 대비 2:1 디레이팅 확보

#### R29, R30 — 블리더/밸런싱 저항

FC1/FC2 방전(안전) + 중간점 전압 균등화.

| 저항값 | 전류 (12.5V) | 소비 전력 | 방전 시정수 (470µF) |
| --- | --- | --- | --- |
| 15kΩ | 0.83mA | 10.4mW | 7.1s |
| 10kΩ | 1.25mA | 15.6mW | 4.7s |

- **현재값**: 15kΩ / 3W
- **권장값**: **10kΩ ~ 15kΩ / 0.5W** (와트 정격만 축소)
- 최대 소비 전력 ~16mW — 3W 정격은 190배 과잉, 0.5W이면 32배 여유
- 블리더 전류(0.8~1.3mA) > 전해 누설 전류(수µA~수십µA) → 중간점 균등 유지 충분

#### R31 — 출력측 블리더/댐핑 저항

변압기/PZT 측 잔류 전하 방전 경로.

- **현재값**: 22kΩ / 5W
- **권장값**: **22kΩ / 1W ~ 2W** (와트 정격만 축소)
- 22kΩ은 50Ω 부하 대비 440배 — AC 동작에 영향 없음
- 최대 소비 전력: 25² / 22000 = 28mW — 5W는 180배 과잉

#### C60 — DC 블로킹 캐패시터 ⚠️ 핵심 변경 대상

SW 노드 DC 성분(VBUS/2)이 변압기로 유입되는 것을 차단. **신호 경로 직렬 삽입 부품**.

| 용량 | 500kHz Xc | 1MHz Xc | 2MHz Xc | 500kHz 전압강하율 (50Ω 기준) |
| --- | --- | --- | --- | --- |
| 0.1µF (현재) | **3.18Ω** | 1.59Ω | 0.80Ω | **6.4%** |
| 0.22µF | 1.45Ω | 0.72Ω | 0.36Ω | 2.9% |
| **0.47µF** | **0.68Ω** | 0.34Ω | 0.17Ω | **1.4%** |
| 1.0µF | 0.32Ω | 0.16Ω | 0.08Ω | 0.6% |

- **현재값**: 0.1µF / 100V
- **권장값**: **0.47µF / 100V 폴리프로필렌(PP) 필름 캐패시터**
- 폴리프로필렌 필름 권장 이유:
  - 낮은 ESR/ESL (전력 경로 적합)
  - 자기 치유(self-healing) 특성
  - DC 바이어스 의존성 없음 (X7R 세라믹은 DC 인가 시 30~50% 감소)
  - 높은 리플 전류 내량
- **대안**: X7R 세라믹 사용 시 DC 디레이팅 고려하여 **1µF / 100V X7R** (실효 0.5~0.7µF)
- 내압: VBUS/2(DC) + VBUS/2(AC peak) = max 25V → 100V로 4:1 디레이팅
