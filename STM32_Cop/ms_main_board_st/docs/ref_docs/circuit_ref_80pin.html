데드타임 설정 (DT 핀)

DT 핀 연결 데드타임
GND 35ns (최소)
저항 50kΩ~200kΩ 가변 설정
VCC 100ns 이상
권장 (IPB200N15N3) 100kΩ → ~80ns

Si MOSFET은 GaN 대비 스위칭이 느리므로 50~100ns 데드타임 권장.
HRTIM 내장 데드타임과 UCC21520 DT 핀 중 하나만 사용 권장 (이중 적용 시 과도한 듀티 손실).

3. IPB200N15N3 Si MOSFET

개요

항목 사양
제조사 Infineon
패키지 D2PAK (TO-263)
Vds (최대) 150V
Rds(on) 20mΩ @ Vgs=10V
Id (연속) 50A @ 25°C
Qg (총 게이트 전하) 31nC
Qgd (밀러 전하) 9nC
tr (라이즈 타임) 7ns
tf (폴 타임) 4ns
게이트 전압 10V ~ 15V 권장

D2PAK 패키지 장점

  • 손납땜 가능: 핀 피치 2.54mm, 드레인 패드 대형
  • 방열 우수: 큰 금속 탭으로 PCB 히트싱크 연결 용이
  • TO-220 대비: 리드 인덕턴스 감소 (~5nH vs ~15nH)

게이트 드라이브 요구사항

계산식 / 파라미터
게이트 충전 시간 = Qg / Ig_avg
               = 31nC / 2A (UCC21520 평균 출력)
               ≈ 15.5ns

권장 게이트 전압: 12V ~ 15V (Rds(on) 최적화)
최대 Vgs: ±20V

6. 설계 시 주의사항

2.2MHz에서의 고려사항

항목 Si MOSFET (IPB200N15N3) GaN FET (EPC2036)
스위칭 시간 ~7ns ~2ns
데드타임 50~100ns 권장 5~20ns 권장
게이트 전압 10~15V 4.5~5.5V
스위칭 손실 높음 (MHz에서) 낮음
역회복 전류 있음 없음
납땜 난이도 쉬움 (D2PAK) 어려움 (BGA)

⚠️ 2.2MHz에서 Si MOSFET 사용 시 스위칭 손실이 증가합니다.
저출력 애플리케이션에서는 괜찮으나, 고출력 시 발열 관리 필수.

PCB 레이아웃 권장사항

  1. 게이트 루프 최소화: OUTA → Q1 게이트 → Q1 소스 → VSSA 경로를 가능한 짧게
  2. 전력 루프 최소화: Vbus → Q1 → Q2 → GND 경로를 최단 거리로
  3. 바이패스 커패시터: VDDA/VSSA, VDDB/VSSB 핀 바로 옆에 배치
  4. GND 플레인: 2층 또는 4층 PCB에서 연속된 GND 플레인 확보
  5. 절연 거리: 1차측/2차측 사이 최소 8mm 이격 (5.7kV 절연)

HRTIM 데드타임 설정

UCC21520의 DT 핀으로 데드타임을 설정할 경우, HRTIM 내장 데드타임은 0으로 설정:

코드 예시
// HRTIM 데드타임 비활성화 (UCC21520 DT 핀 사용 시)
sConfigDT.Prescaler = HRTIM_DEADTIME_PRESCALER_OFF;
sConfigDT.RisingValue = 0;
sConfigDT.RisingSign = HRTIM_DEADTIME_SIGN_POSITIVE;
sConfigDT.RisingLock = HRTIM_DEADTIME_LOCK_WRITE;
sConfigDT.FallingValue = 0;
sConfigDT.FallingSign = HRTIM_DEADTIME_SIGN_POSITIVE;
sConfigDT.FallingLock = HRTIM_DEADTIME_LOCK_WRITE;
HAL_HRTIM_DeadTimeConfig(&hhrtim, HRTIM_TIMERINDEX_TIMER_A, &sConfigDT);

또는 UCC21520 DT 핀을 GND에 연결 (최소 35ns)하고 HRTIM에서 추가 데드타임 설정:

코드 예시
// HRTIM 데드타임 50ns 추가 (UCC21520 DT=GND 시)
// 50ns = 50e-9 * 5.44e9 = 272 틱
sConfigDT.RisingValue = 272;
sConfigDT.FallingValue = 272;

STM32 DAC를 이용한 가변 전압 제어

⚠️ 기존 LM5005MH 회로(R79=5.11K, R80=16K)의 분압비는 고정 +5V 출력용이므로 사용 불가!
0.1W~10W 가변 제어를 위해 아래 R1, R2, R3 값을 새로 설계해야 합니다.

동작 원리 (FB 노드 KCL):

계산식 / 파라미터
(V_out - V_fb)/R1 = V_fb/R2 + (V_fb - V_dac)/R3

V_fb = 1.23V (FB 기준 전압) 일 때:
V_out = 1.23 × (1 + R1/R2) + (1.23 - V_dac) × R1/R3
  • DAC = 0V: R3를 통해 전류 유출 → V_out = 최대
  • DAC = V_fb(1.23V): R3 영향 없음 (전류=0)
  • DAC > V_fb: R3를 통해 FB에 전류 주입 → V_out 감소

설계값 (R1=11K, R2=1K, R3=1.8K) 계산:

회로도 / 블록도
V_out = 1.23 × 12 + (1.23 - V_dac) × 6.11
V_out = 14.76 + (1.23 - V_dac) × 6.11

├── DAC = 0V    → V_out = 14.76 + 7.52 = 22.3V → P = 9.9W  ✓
├── DAC = 1.0V  → V_out = 14.76 + 1.41 = 16.2V → P = 5.2W
├── DAC = 1.23V → V_out = 14.76 + 0    = 14.8V → P = 4.4W (FB와 동일)
├── DAC = 2.0V  → V_out = 14.76 - 4.70 = 10.1V → P = 2.0W
├── DAC = 3.0V  → V_out = 14.76 - 10.8 = 3.9V  → P = 0.31W
└── DAC = 3.3V  → V_out = 14.76 - 12.7 = 2.1V  → P = 0.09W ✓

    [R_VH: 100kΩ, 1%, 0603]   ← 상단 분압 저항
     │
     ├──────────╾────────────▶ PA7 (ADC2_IN4, MCU 3.3V)
     │          ↑
     │     [100nF C0G]        ← 노이즈 필터 (선택)
     │          │
    [R_VL: 8.2kΩ, 1%, 0603]   ← 하단 분압 저항
     │
    GND

분압비 계산 (VOUT_max = 35V 기준):

회로도 / 블록도
분압비 = R_VL / (R_VH + R_VL) = 8.2 / 108.2 = 0.0758

VOUT → V_PA7 변환:
├── VOUT = 35V (과도 최대) → V_PA7 = 35 × 0.0758 = 2.65V ✓  (ADC 3.3V 마진 0.65V)
├── VOUT = 22.4V (정상 최대) → V_PA7 = 22.4 × 0.0758 = 1.70V ✓
├── VOUT = 10V  → V_PA7 = 0.758V ✓
└── VOUT = 2V   → V_PA7 = 0.152V ✓

ADC count → VOUT 역산:
VOUT = ADC_count × (3.3V / 4096) × (R_VH + R_VL) / R_VL
     = ADC_count × (3.3 / 4096) × (108.2 / 8.2)
     = ADC_count × 0.1063V/count

ADC 분해능: 35V / 4096 ≈ 8.5mV/count (VOUT 환산)

⚠️ PA7 직결은 절대 금지. VOUT(최대 35V) → MCU VDDA(3.3V) 초과 → ADC 입력 파괴.
⚠️ R_VH + R_VL 분압 저항은 1% 정밀도 사용 (전압 측정 오차 직접 반영).

항목 R_VH(100kΩ) + R_VL(8.2kΩ)
분압비 1/13.2 (≈1/13)
35V 입력 시 PA7 전압 2.65V (MCU 안전 ✓)
22.4V 입력 시 PA7 전압 1.70V
소비 전류 (35V 시) 35V / 108.2kΩ ≈ 0.32mA (무시 가능)

PA7(ADC2_IN4) 용도: BUCK 출력 전압 실시간 모니터링 → PI 피드백 루프 보조, 출력 과전압 감지(SW 보호).

전력 제어 펌웨어 예시

코드 예시
#include "stm32g4xx_hal.h"
#include <math.h>

#define LOAD_IMPEDANCE      50.0f    // 트랜스듀서 임피던스 (Ω)
#define V_REF               1.23f    // LM5005 FB 기준 전압
#define R1_VALUE            11000.0f // FB 상단 분압 저항 (Ω)
#define R2_VALUE            1000.0f  // FB 하단 분압 저항 (Ω)
#define R3_VALUE            1800.0f  // DAC 주입 저항 (Ω)
#define DAC_MAX_VALUE       4095     // 12비트 DAC
#define DAC_VREF            3.3f     // DAC 기준 전압

// V_out = V_REF × (1 + R1/R2) + (V_REF - V_dac) × R1/R3
// V_out = 14.76 + (1.23 - V_dac) × 6.11

// 전력 → DAC 값 변환 테이블 (캘리브레이션용)
static const struct {
    float power;    // W
    uint16_t dac;   // 12-bit
} power_table[] = {
    {0.1f,  4095},  // V_out ≈ 2.1V  (DAC=3.3V)
    {0.3f,  3700},  // V_out ≈ 3.9V  (DAC=3.0V)
    {0.5f,  3400},  // V_out ≈ 5.0V
    {1.0f,  3000},  // V_out ≈ 7.1V
    {2.0f,  2500},  // V_out ≈ 10V   (DAC=2.0V)
    {5.0f,  1500},  // V_out ≈ 15.8V (DAC=1.2V)
    {10.0f, 0},     // V_out ≈ 22.3V (DAC=0V)
};

/**
 * @brief 출력 전력 설정 (0.1W ~ 10W, 0.1W 단위)
 * @param power_watts 목표 전력 (W)
 */
void BUCK_SetPower(float power_watts)
{
    // 범위 제한
    if (power_watts < 0.1f) power_watts = 0.1f;
    if (power_watts > 10.0f) power_watts = 10.0f;

    // 전력 → 전압 변환 (P = V² / R)
    float target_voltage = sqrtf(power_watts * LOAD_IMPEDANCE);

    // 전압 → DAC 값 변환
    // V_out = V_OUT_MAX - (V_dac × DAC_GAIN)
    // V_dac = (V_OUT_MAX - target_voltage) / DAC_GAIN
    float v_dac = (V_OUT_MAX - target_voltage) / DAC_GAIN;

    // DAC 전압 → DAC 레지스터 값 (0~3.3V → 0~4095)
    if (v_dac < 0.0f) v_dac = 0.0f;
    if (v_dac > 3.3f) v_dac = 3.3f;

    uint16_t dac_value = (uint16_t)(v_dac / 3.3f * (float)DAC_MAX_VALUE);

    // DAC 출력 설정 (PA4)
    HAL_DAC_SetValue(&hdac1, DAC_CHANNEL_1, DAC_ALIGN_12B_R, dac_value);
}

/**
 * @brief 전력 피드백 제어 루프 (100ms 주기 호출)
 */
void BUCK_PowerControlLoop(void)
{
    static float error_integral = 0.0f;
    const float Kp = 0.5f;
    const float Ki = 0.1f;

    // ADC로 실제 전력 측정
    float actual_power = ADC_GetForwardPower();
    float target_power = g_target_power;

    // PI 제어
    float error = target_power - actual_power;
    error_integral += error * 0.1f;  // dt = 100ms

    // 적분 제한
    if (error_integral > 5.0f) error_integral = 5.0f;
    if (error_integral < -5.0f) error_integral = -5.0f;

    // 보정된 전력값 계산
    float corrected_power = target_power + (Kp * error) + (Ki * error_integral);

    // BUCK 출력 조정
    BUCK_SetPower(corrected_power);
}

10. PZT 출력 전력 검출 회로 (2단계 공진 탐색)

개요

PZT 트랜스듀서의 공진주파수 탐색 및 출력 전력 모니터링을 위한 2단계 검출 구조.

1차 공진 탐색 — 전류 기반 주파수 스윕:
발진 주파수를 공칭 주파수 ±10% 범위에서 스윕하며 BUCK 출력 전류(PA6, INA190A4)를
모니터링하여 전류 최대점 = 공진점으로 판단한다.
공진 시 직렬 임피던스가 최소가 되어 전류가 최대가 되는 물리 원리를 이용.

2차 정밀 탐색 — CT + 쇼트키 브리지 검파:
1차 탐색 결과 근처에서 CT + BAT54W 쇼트키 브리지 검파로
좌/우 채널(PA0, PA1) 밸런스 오차를 측정하여 |PA0-PA1| 최소점을 정밀 공진점으로 찾는다.
기존 LTC5507(RF 전력 검출기 IC)은 오버스펙·조달 불안정으로 제거하고,
0.5~2 MHz 대역에 적합한 단순 다이오드 검파 방식으로 대체한다.

| 항목 | 1차 (전류 기반) | 2차 (CT + 다이오드) |
| --------- | ---------------------------- | ----------------------------------- | ------- | ----- |
| 측정 대상 | BUCK 출력 전류 (PA6) | 브리지 좌/우 채널 (PA0, PA1) |
| 검출 소자 | INA190A4 전류 센스 앰프 (§9) | BAT54W 쇼트키 다이오드 피크 검파 ×2 |
| 스윕 범위 | 공칭 주파수 ±10% | 1차 결과 ±2% (정밀) |
| 판정 기준 | 전류 최대점 (Z_min → I_max) | 브리지 밸런스점 ( | PA0-PA1 | 최소) |
| 필요 전력 | 저전력 (0.15W)에서도 동작 | 중전력 (≥0.3W) 권장 |
| 정밀도 | 중 (채널/LC 조합 단위) | 상 (kHz 단위) |

10.2 2차 정밀 탐색: CT + 쇼트키 다이오드 검파기

1차 탐색 결과 주파수 근처(±2%)에서 브리지 좌/우 채널의 밸런스(|PA0-PA1|)로 정밀 공진점을 찾는다.

브릿지 회로 동작 원리

CT 2차측 전류가 브릿지 저항망을 통해 분배됨.
출력 트랜스 2차측 전류 조건에 따라 좌/우 브리지 노드에 차동 전압이 발생한다.

R41~R46 표기 기준 연결 규칙 (PZT_OUTPUT_DETECT_Schottky.jpg 기준):

  • Pin1 노드 = CT2 2차측 Pin1 = R41 상단 = R43/R44 상단
  • Pin2 노드 = CT2 2차측 Pin2 = R42 상단 = R45/R46 상단
  • R41, R42 하단만 GNDPZT 공통 노드로 연결
  • R43, R44는 좌측 브리지 내부 병렬 (우측과 직접 연결 금지)
  • R45, R46는 우측 브리지 내부 병렬 (좌측과 직접 연결 금지)
  • C14, C15는 Pin1/Pin2가 아니라 CT 1차측(고전력 라인)에서 GND로 샘플링
  • 즉, R41~R46의 한쪽이 모두 하나의 공통 노드로 합쳐지면 안 됨

등가 저항 계산:

계산식 / 파라미터
R43 // R44 = 200Ω // 200Ω = 100Ω   (좌측 브리지 분기 임피던스)
R45 // R46 = 200Ω // 200Ω = 100Ω   (우측 브리지 분기 임피던스)
R_load = R41 // (R43 // R44) = 200Ω // 100Ω = 66.7Ω  (CT Pin1 기준 합성 부하)

채널 방향 라벨 주의:

  • 좌측(PA0)을 순방향, 우측(PA1)을 역방향으로 고정 정의하지 않는다.
  • CT 권선 방향/배선 극성에 따라 좌우 의미가 바뀔 수 있으므로, 초기 캘리브레이션으로 라벨을 확정한다.
  • 공진점 판정은 라벨과 무관하게 |PA0-PA1| 최소 기준으로 수행한다.

R41~R46 전부 200Ω 통일: BOM 1종으로 단순화.
기존 50Ω/25Ω 대비 신호 레벨 55%(500kHz)~220%(2MHz) 향상.

AC 커플링 고역통과 필터 (C73, C74 = 100nF):

계산식 / 파라미터
fc = 1 / (2π × R_source × C)
   = 1 / (2π × 100Ω × 100nF)
   ≈ 16 kHz

500kHz (Ch0)에서: -0.005 dB (완전 평탄)
2MHz (Ch6)에서: -0.0003 dB (완전 평탄)

R_source가 12.5Ω→100Ω로 증가했으나, fc가 오히려 127kHz→16kHz로 낮아져
전 채널에서 더욱 평탄한 응답 확보.

ADC 변환 (12비트, 3.3V 기준):

계산식 / 파라미터
ADC_value = V_detect / 3.3V × 4095

V_detect → V_bridge_peak 역변환 (필요 시):
  V_bridge_peak = V_detect + Vf ≈ ADC × 3.3 / 4095 + 0.24 [V]

VSWR 계산에는 역변환 불필요 — FWD/REV ADC값의 비율만 사용:
  Γ ≈ √(ADC_rev / ADC_fwd)
  VSWR = (1 + Γ) / (1 - Γ)

핵심 장점: 선형 검파기이므로 FWD/REV ADC 원시값으로 직접 VSWR 계산 가능.
LTC5507 로그 출력처럼 dBm→mW 역변환이 불필요하여 펌웨어 단순화.

공진주파수 탐색 알고리즘 (2단계)

PZT 트랜스듀서의 직렬 공진점에서 임피던스 최소 → 전류 최대 → VSWR 최소.

1단계: 전류 기반 공진 탐색 (I_max)

코드 예시
/**
 * @brief 전류 기반 공진주파수 탐색 (1차)
 *        공칭 주파수 ±10% 범위를 스윕, 전류 최대점 = 공진점
 * @param nominal_khz  공칭 주파수 (kHz), 예: 2000
 * @param step_khz     스윕 스텝 (kHz), 예: 5
 * @return 공진주파수 (kHz), 0이면 탐색 실패
 */
uint32_t FindResonance_CurrentMax(uint32_t nominal_khz, uint16_t step_khz)
{
    uint32_t start = nominal_khz * 90 / 100;   // -10%
    uint32_t end   = nominal_khz * 110 / 100;  // +10%
    uint32_t best_freq = 0;
    uint16_t max_current = 0;

    if (start < FREQ_MIN_KHZ) start = FREQ_MIN_KHZ;
    if (end > FREQ_MAX_KHZ)   end = FREQ_MAX_KHZ;

    MegasonicCtrl_SetPower(RESCAN_POWER_01W);  // 저전력 (0.15W)
    HRTIM_OutputStart();

    for (uint32_t freq = start; freq <= end; freq += step_khz) {
        HRTIM_SetFrequency(freq);
        HAL_Delay(RESCAN_FREQ_SETTLE_MS);      // 10ms 안정화

        uint16_t current = ADC_GetCurrent_Raw(); // PA6 (INA190A4)

        if (current > max_current) {
            max_current = current;
            best_freq = freq;
        }
    }

    HRTIM_OutputStop();
    return best_freq;
}

탐색 파라미터 예시 (Ch6, 2MHz PZT):

계산식 / 파라미터
공칭 주파수: 2000 kHz
스윕 범위:   1800 ~ 2200 kHz (±10%)
스텝:        5 kHz
포인트 수:   (2200-1800)/5 + 1 = 81포인트
소요 시간:   81 × 10ms = ~0.8초

2단계: 브리지 밸런스 정밀 탐색 (|PA0-PA1| 최소)

1차 결과 근처(±2%)에서 다이오드 검파 좌/우 채널 차이(|PA0-PA1|) 최소점을 정밀 탐색.

코드 예시
/**
 * @brief 브리지 밸런스 오차 계산
 *        공진점 근처에서 좌/우 채널이 평형이므로 |PA0-PA1| 최소값을 사용
 */
uint16_t CalculateBalanceError(uint16_t left, uint16_t right)
{
  if (left > right) return (uint16_t)(left - right);
  return (uint16_t)(right - left);
}

/**
 * @brief 2단계 브리지 밸런스 정밀 탐색
 *        1차 결과 ±2% 범위를 2kHz 스텝으로 정밀 스윕
 * @param coarse_khz  1차 탐색 결과 주파수 (kHz)
 * @return 정밀 공진주파수 (kHz), 0이면 탐색 실패
 */
uint32_t FindResonance_BalanceFine(uint32_t coarse_khz)
{
    uint32_t start = coarse_khz * 98 / 100;    // -2%
    uint32_t end   = coarse_khz * 102 / 100;   // +2%
    uint32_t best_freq = coarse_khz;
  uint16_t min_err = 0xFFFF;

    if (start < FREQ_MIN_KHZ) start = FREQ_MIN_KHZ;
    if (end > FREQ_MAX_KHZ)   end = FREQ_MAX_KHZ;

    // 주파수별 전력 자동 조정: 500~700kHz는 0.5W, 700kHz 이상은 0.3W
    uint16_t tune_power = (coarse_khz < 700) ? RESCAN_POWER_VSWR_LF : RESCAN_POWER_VSWR;
    MegasonicCtrl_SetPower(tune_power);
    HRTIM_OutputStart();

    for (uint32_t freq = start; freq <= end; freq += 2) {
        HRTIM_SetFrequency(freq);
        HAL_Delay(RESCAN_FREQ_SETTLE_MS);

      uint16_t left = ADC_GetForwardPower_Raw();   // PA0 (좌측 채널)
      uint16_t right = ADC_GetReversePower_Raw();  // PA1 (우측 채널)
      uint16_t err = CalculateBalanceError(left, right);

      if (err < min_err) {
        min_err = err;
            best_freq = freq;
        }
    }

    HRTIM_OutputStop();
    return (min_err <= RESCAN_BALANCE_OK_ADC) ? best_freq : 0;
}

/**
 * @brief 2단계 공진주파수 탐색 (전류 1차 → 밸런스 2차)
 */
uint32_t FindResonance_TwoPhase(uint32_t nominal_khz)
{
    // 1단계: 전류 기반 광역 탐색 (±10%, 5kHz 스텝)
    uint32_t coarse = FindResonance_CurrentMax(nominal_khz, 5);
    if (coarse == 0) return 0;

    // 2단계: 밸런스 기반 정밀 탐색 (±2%, 2kHz 스텝)
    uint32_t fine = FindResonance_BalanceFine(coarse);

    return (fine != 0) ? fine : coarse;  // 2차 실패 시 1차 결과 사용
}

클럭 소스 (HSI vs HSE 상세 분석)

개요

STM32G474MET6는 두 가지 고속 클럭 소스를 지원합니다:

항목 HSI (내부 RC) HSE (외부 크리스탈)
주파수 16 MHz 4~48 MHz (외부 부품)
정밀도 (25°C) ±1% (공장 트리밍) ±20ppm (크리스탈 의존)
온도 드리프트 ±3% (-40~105°C 전 범위) ±50ppm (-40~85°C, AT컷 기준)
외부 부품 없음 크리스탈 + 부하 커패시터 2개
시작 시간 ~2µs ~2ms (크리스탈 안정화)
사용 핀 없음 PF0-OSC_IN, PF1-OSC_OUT
PLL 입력 ✅ ✅
HRTIM DLL 영향 DLL이 자체 캘리브레이션 수행 더 안정적인 기준 클럭 제공

핵심 문제: PF0/PF1 핀 충돌

LQFP80 패키지에서 HSE 크리스탈을 사용하면 PF0, PF1이 GPIO로 사용 불가합니다:

참고 블록
STM32G474MET6 LQFP80

    핀 11: PF0 / OSC_IN   ← HSE 사용 시 크리스탈 연결 (GPIO 불가)
    핀 12: PF1 / OSC_OUT  ← HSE 사용 시 크리스탈 연결 (GPIO 불가)

현재 설계에서 PF0/PF1은 LC탱크 릴레이로 할당:

핀 현재 용도 HSE 사용 시
PF0 LC_RELAY3 (GPIO) ❌ OSC_IN 전용
PF1 LC_RELAY4 (GPIO) ❌ OSC_OUT 전용

HSI 선택 시 영향 분석

1. HRTIM PWM 주파수 정밀도

회로도 / 블록도
PLL 설정: HSI 16MHz → PLL → 170MHz SYSCLK
HRTIM DLL: 170MHz × 32 = 5.44GHz 유효 클럭

HSI ±1% 오차가 PWM 주파수에 미치는 영향:
├── 목표 500kHz → 실제 495~505 kHz (±5kHz)
├── 목표 1000kHz → 실제 990~1010 kHz (±10kHz)
└── 목표 2000kHz → 실제 1980~2020 kHz (±20kHz)

HSE ±20ppm 일 경우:
├── 목표 500kHz → 실제 499.99~500.01 kHz (±0.01kHz)
├── 목표 1000kHz → 실제 999.98~1000.02 kHz
└── 목표 2000kHz → 실제 1999.96~2000.04 kHz

메가소닉 트랜스듀서는 공진주파수 대역폭(Q값)이 수 kHz~수십 kHz이므로,
HSI ±1% 오차(±5~20kHz)는 대역폭 내에서 충분히 허용됩니다.
또한 실시간 전력 피드백(전류 센싱 + 다이오드 검파)으로 공진점을 추적하므로 절대 정밀도보다 상대 분해능이 중요합니다.

2. Modbus RTU 보레이트 정밀도

참고 블록
USART2 보레이트: 9600 bps (또는 19200/38400)

RS-485 허용 오차: ±3% (일반적 UART 동기화 마진)

HSI ±1% → USART 보레이트 오차 ±1% → ✅ 3% 이내 (충분)
HSI ±3% (극한 온도) → USART 보레이트 오차 ±3% → ⚠️ 경계

HSE ±20ppm → USART 보레이트 오차 무시 수준

산업 현장 온도(0~60°C)에서 HSI 오차는 ±1.5% 이내이므로 Modbus 통신에 문제 없습니다.
극한 환경(-40~105°C)에서만 HSE가 필요합니다.

3. ADC 샘플링 타이밍

계산식 / 파라미터
ADC 클럭 = SYSCLK / 분주 (보통 /4 = 42.5MHz)

HSI ±1% → ADC 변환 시간 ±1% → 측정값 자체에는 영향 없음
(ADC는 S&H 방식이므로 클럭 주파수가 아닌 기준 전압 정밀도가 중요)

ADC 정밀도는 VREF+ 전압에 의존하며, 클럭 오차의 영향은 무시할 수 있습니다.

17.1 LED 구동 방식 (74HCT574 래치)

현재 설계의 LED는 STM32 GPIO 직결이 아니라, 74HCT574 #2 래치를 통해 5V 도메인에서 구동한다.

회로도 / 블록도

PB10~PB15(DATA) ─────────────▶ D0~D5                     Q0~Q5 ──[560Ω]──▶ LED ── GND
PD9 (CLK_LED)  ──────────────▶ CLK

LED ON/OFF는 PB10~PB15 데이터 비트와 PD9 래치 펄스로 제어된다.

17.2 저항값 계산 (5V 레일 기준)

공식:

계산식 / 파라미터
R = (V_LED_RAIL - V_f) / I_f

V_LED_RAIL = 5.0V
V_f = LED 순방향 전압
I_f = LED 전류

560Ω 단일값 사용 시 전류 예시:

LED 색상 V_f (typ) I_f 계산 결과
적색 (Red) 2.0V (5.0-2.0)/560 = 5.36mA ✅
황색 (Yellow) 2.1V (5.0-2.1)/560 = 5.18mA ✅
녹색 (Green) 2.2V (5.0-2.2)/560 = 5.00mA ✅

⚠️ 청색/백색 LED: Vf가 3.2V로 높아 전압 마진이 0.1V밖에 안됨.
