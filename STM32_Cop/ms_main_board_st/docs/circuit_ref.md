# 메가소닉 게이트 드라이브 회로 참조 문서

> 2.2MHz 메가소닉 발진기용 하프브리지 게이트 드라이브 회로 설계 참조.
> UCC21520 절연 게이트 드라이버 + IPB200N15N3 Si MOSFET + 절연 DC-DC 컨버터 구성.

## 2. UCC21520 절연 게이트 드라이버

### 개요

| 항목                  | 사양                                      |
| --------------------- | ----------------------------------------- |
| 제조사                | Texas Instruments                         |
| 패키지                | SOIC-16W (DW)                             |
| 절연 내압             | 5.7kVrms (강화 절연)                      |
| 전파 지연             | 18ns (typ)                                |
| CMTI                  | 150kV/µs (Common Mode Transient Immunity) |
| 피크 출력 전류        | 4A (source/sink)                          |
| 동작 주파수           | DC ~ 5MHz                                 |
| 입력 전압 (VCC)       | 3.0V ~ 5.5V                               |
| 출력 전압 (VDDA/VDDB) | 9.2V ~ 25V                                |

### 핀 배치 및 기능

```
                    ┌─────────────────────────────────────┐
                    │           UCC21520DW                │
     1차측 (MCU)    │                                     │    2차측 (FET)
                    │                                     │
     VCCI ──────────┤8  VCCI                   VDDA  16├──────── VDDA (하이사이드)
            (+3.3V) │                                     │
     DIS  ──────────┤7  DISABLE                OUTA  15├──────── Q1 게이트
                    │                                     │
     INB  ──────────┤6  INB                    VSSA  14├──────── SW 노드
            (PA9)   │                                     │
     INA  ──────────┤5  INA                    NC    13├────────
            (PA8)   │                                     │
     GND  ──────────┤4  GND                    NC    12├────────
                    │                                     │
     NC   ──────────┤3  NC                     VDDB  11├──────── VDDB (로우사이드)
                    │                                     │
     DT   ──────────┤2  DT                     OUTB  10├──────── Q2 게이트
                    │                                     │
     VCCI ──────────┤1  VCCI                   VSSB   9├──────── 파워 GND
                    │                                     │
                    └─────────────────────────────────────┘
```

### 전원 도메인 (3개 독립 절연)

| 도메인                   | 전원 핀 | GND 핀 | 기준점       | 전압 범위   |
| ------------------------ | ------- | ------ | ------------ | ----------- |
| **1차측 (입력)**         | VCC     | GND    | 시스템 GND   | 3.0V ~ 5.5V |
| **2차측 A (하이사이드)** | VDDA    | VSSA   | **SW 노드**  | 9.2V ~ 25V  |
| **2차측 B (로우사이드)** | VDDB    | VSSB   | **파워 GND** | 9.2V ~ 25V  |

> ⚠️ **VSSA ≠ VSSB**: VSSA는 SW 노드에, VSSB는 파워 GND에 연결. 절대 공유 금지!

### 데드타임 설정 (DT 핀)

| DT 핀 연결             | 데드타임          |
| ---------------------- | ----------------- |
| GND                    | 35ns (최소)       |
| 저항 50kΩ~200kΩ        | 가변 설정         |
| VCC                    | 100ns 이상        |
| **권장 (IPB200N15N3)** | **100kΩ → ~80ns** |

> Si MOSFET은 GaN 대비 스위칭이 느리므로 50~100ns 데드타임 권장.
> HRTIM 내장 데드타임과 UCC21520 DT 핀 중 **하나만 사용** 권장 (이중 적용 시 과도한 듀티 손실).

## 3. IPB200N15N3 Si MOSFET

### 개요

| 항목                | 사양           |
| ------------------- | -------------- |
| 제조사              | Infineon       |
| 패키지              | D2PAK (TO-263) |
| Vds (최대)          | 150V           |
| Rds(on)             | 20mΩ @ Vgs=10V |
| Id (연속)           | 50A @ 25°C     |
| Qg (총 게이트 전하) | 31nC           |
| Qgd (밀러 전하)     | 9nC            |
| tr (라이즈 타임)    | 7ns            |
| tf (폴 타임)        | 4ns            |
| 게이트 전압         | 10V ~ 15V 권장 |

### D2PAK 패키지 장점

- **손납땜 가능**: 핀 피치 2.54mm, 드레인 패드 대형
- **방열 우수**: 큰 금속 탭으로 PCB 히트싱크 연결 용이
- **TO-220 대비**: 리드 인덕턴스 감소 (~5nH vs ~15nH)

### 게이트 드라이브 요구사항

```
게이트 충전 시간 = Qg / Ig_avg
               = 31nC / 2A (UCC21520 평균 출력)
               ≈ 15.5ns

권장 게이트 전압: 12V ~ 15V (Rds(on) 최적화)
최대 Vgs: ±20V
```

## 4. 절연 DC-DC 컨버터

### VDDA 전원 공급 (하이사이드 - 필수 절연)

VSSA = SW 노드이므로 **반드시 절연된 전원** 필요.

#### 옵션 1: RECOM R1215S (권장)

| 항목   | 사양                   |
| ------ | ---------------------- |
| 입력   | 12V DC (10.8V ~ 13.2V) |
| 출력   | 15V DC, 67mA           |
| 절연   | 1kVDC (기본 절연)      |
| 패키지 | SIP-4 (1W 급)          |
| 효율   | 80%                    |

```
    시스템 12V ────┬────▶ R1215S ────┬────▶ VDDA (15V)
                   │      (절연)     │
                   │                 └────▶ VSSA (= SW 노드에 연결)
                   │
                   └────────────────────────▶ VDDB (직접)
                                              VSSB (= 파워 GND)
```

#### 옵션 2: SN6505A + 트랜스포머 (고절연)

고절연(5.7kVrms) 요구 시 사용.

| 항목        | 사양                      |
| ----------- | ------------------------- |
| 드라이버 IC | SN6505A (TI)              |
| 입력        | 3.3V 또는 5V              |
| 트랜스포머  | Würth 750316830 (1:1)     |
| 정류        | 쇼트키 다이오드 + LC 필터 |

### VDDB 전원 공급 (로우사이드 - 직접 가능)

VSSB = 파워 GND이므로 **시스템 전원에서 직접 공급 가능**.

```
    시스템 12V~15V ────────────────────▶ VDDB
                                        │
    파워 GND ──────────────────────────▶ VSSB
```

## 6. 설계 시 주의사항

### 2.2MHz에서의 고려사항

| 항목            | Si MOSFET (IPB200N15N3) | GaN FET (EPC2036) |
| --------------- | ----------------------- | ----------------- |
| **스위칭 시간** | ~7ns                    | ~2ns              |
| **데드타임**    | 50~100ns 권장           | 5~20ns 권장       |
| **게이트 전압** | 10~15V                  | 4.5~5.5V          |
| **스위칭 손실** | 높음 (MHz에서)          | 낮음              |
| **역회복 전류** | 있음                    | 없음              |
| **납땜 난이도** | 쉬움 (D2PAK)            | 어려움 (BGA)      |

> ⚠️ **2.2MHz에서 Si MOSFET 사용 시 스위칭 손실이 증가**합니다.
> 저출력 애플리케이션에서는 괜찮으나, 고출력 시 발열 관리 필수.

### PCB 레이아웃 권장사항

1. **게이트 루프 최소화**: OUTA → Q1 게이트 → Q1 소스 → VSSA 경로를 가능한 짧게
2. **전력 루프 최소화**: Vbus → Q1 → Q2 → GND 경로를 최단 거리로
3. **바이패스 커패시터**: VDDA/VSSA, VDDB/VSSB 핀 바로 옆에 배치
4. **GND 플레인**: 2층 또는 4층 PCB에서 연속된 GND 플레인 확보
5. **절연 거리**: 1차측/2차측 사이 최소 8mm 이격 (5.7kV 절연)

### HRTIM 데드타임 설정

UCC21520의 DT 핀으로 데드타임을 설정할 경우, **HRTIM 내장 데드타임은 0으로 설정**:

```c
// HRTIM 데드타임 비활성화 (UCC21520 DT 핀 사용 시)
sConfigDT.Prescaler = HRTIM_DEADTIME_PRESCALER_OFF;
sConfigDT.RisingValue = 0;
sConfigDT.RisingSign = HRTIM_DEADTIME_SIGN_POSITIVE;
sConfigDT.RisingLock = HRTIM_DEADTIME_LOCK_WRITE;
sConfigDT.FallingValue = 0;
sConfigDT.FallingSign = HRTIM_DEADTIME_SIGN_POSITIVE;
sConfigDT.FallingLock = HRTIM_DEADTIME_LOCK_WRITE;
HAL_HRTIM_DeadTimeConfig(&hhrtim, HRTIM_TIMERINDEX_TIMER_A, &sConfigDT);
```

또는 **UCC21520 DT 핀을 GND에 연결 (최소 35ns)**하고 HRTIM에서 추가 데드타임 설정:

```c
// HRTIM 데드타임 50ns 추가 (UCC21520 DT=GND 시)
// 50ns = 50e-9 * 5.44e9 = 272 틱
sConfigDT.RisingValue = 272;
sConfigDT.FallingValue = 272;
```

## 9. BUCK 전력 컨버터 (LM5005)

### 개요

48V SMPS 전원에서 가변 DC 전압을 생성하여 하프브리지 인버터의 출력 전력을 0.1W~10W 범위로 제어.

| 항목          | 사양                      |
| ------------- | ------------------------- |
| 제조사        | Texas Instruments         |
| 패키지        | **SOIC-14** (손납땜 용이) |
| 입력 전압     | 7V ~ 75V                  |
| 출력 전압     | 1.23V ~ 68V (가변)        |
| 스위칭 주파수 | 50kHz ~ 500kHz            |
| 제어 방식     | Current Mode PWM          |
| FB 기준 전압  | 1.23V                     |
| 외부 FET      | Hi-side + Lo-side 필요    |

### 핀 배치

```
                    ┌─────────────────────────────────────────┐
                    │              LM5005                     │
                    │             SOIC-14                     │
                    │                                         │
     VIN ───────────┤1  VIN                          HO  14├──── 하이사이드 게이트
     EN ────────────┤2  EN                           HS  13├──── 스위치 노드
     RT ────────────┤3  RT                          BST  12├──── 부트스트랩
     AGND ──────────┤4  AGND                        VCC  11├──── 내부 레귤레이터
     SS ────────────┤5  SS (소프트스타트)            CS  10├──── 전류 감지
     COMP ──────────┤6  COMP                       PGND   9├──── 파워 GND
     FB ────────────┤7  FB (피드백)                  LO   8├──── 로우사이드 게이트
                    │                                         │
                    └─────────────────────────────────────────┘
```

### 출력 전압 조정 방법

#### 기본 원리

```
V_out = V_ref × (1 + R1/R2) = 1.23V × (1 + R1/R2)
```

FB 핀 전압이 1.23V가 되도록 출력을 조절함.

#### STM32 DAC를 이용한 가변 전압 제어

> ⚠️ **기존 LM5005MH 회로(R79=5.11K, R80=16K)의 분압비는 고정 +5V 출력용이므로 사용 불가!**
> 0.1W~10W 가변 제어를 위해 아래 R1, R2, R3 값을 새로 설계해야 합니다.

```
     V_out (2.2V ~ 22.4V 가변)
         │
        [R1] = 11kΩ (최대 전압 결정)
         │
         ├─────────────────────▶ FB (핀 7, LM5005)
         │                            │
        [R2] = 1kΩ                   [R3] = 1.8kΩ
         │                            │
        GND                    DAC1_OUT1 (PA4)
                               STM32G474
```

**동작 원리 (FB 노드 KCL)**:

```
(V_out - V_fb)/R1 = V_fb/R2 + (V_fb - V_dac)/R3

V_fb = 1.23V (FB 기준 전압) 일 때:
V_out = 1.23 × (1 + R1/R2) + (1.23 - V_dac) × R1/R3
```

- DAC = 0V: R3를 통해 전류 유출 → V_out = 최대
- DAC = V_fb(1.23V): R3 영향 없음 (전류=0)
- DAC > V_fb: R3를 통해 FB에 전류 주입 → V_out 감소

**설계값 (R1=11K, R2=1K, R3=1.8K) 계산**:

```
V_out = 1.23 × 12 + (1.23 - V_dac) × 6.11
V_out = 14.76 + (1.23 - V_dac) × 6.11

예시:
├── DAC = 0V    → V_out = 14.76 + 7.52 = 22.3V → P = 9.9W  ✓
├── DAC = 1.0V  → V_out = 14.76 + 1.41 = 16.2V → P = 5.2W
├── DAC = 1.23V → V_out = 14.76 + 0    = 14.8V → P = 4.4W (FB와 동일)
├── DAC = 2.0V  → V_out = 14.76 - 4.70 = 10.1V → P = 2.0W
├── DAC = 3.0V  → V_out = 14.76 - 10.8 = 3.9V  → P = 0.31W
└── DAC = 3.3V  → V_out = 14.76 - 12.7 = 2.1V  → P = 0.09W ✓
```

> ✅ **DAC 0~3.3V로 출력 0.1W ~ 10W 범위 제어 가능!**

### 전력 제어 펌웨어 예시

```c
#include "stm32g4xx_hal.h"
#include <math.h>

#define LOAD_IMPEDANCE      50.0f    // 트랜스듀서 임피던스 (Ω)
#define V_REF               1.23f    // LM5005 FB 기준 전압
#define R1_VALUE            11000.0f // FB 상단 분압 저항 (Ω)
#define R2_VALUE            1000.0f  // FB 하단 분압 저항 (Ω)
#define R3_VALUE            1800.0f  // DAC 주입 저항 (Ω)
#define DAC_MAX_VALUE       4095     // 12비트 DAC
#define DAC_VREF            3.3f     // DAC 기준 전압

// V_out = V_REF × (1 + R1/R2) + (V_REF - V_dac) × R1/R3
// V_out = 14.76 + (1.23 - V_dac) × 6.11

// 전력 → DAC 값 변환 테이블 (캘리브레이션용)
static const struct {
    float power;    // W
    uint16_t dac;   // 12-bit
} power_table[] = {
    {0.1f,  4095},  // V_out ≈ 2.1V  (DAC=3.3V)
    {0.3f,  3700},  // V_out ≈ 3.9V  (DAC=3.0V)
    {0.5f,  3400},  // V_out ≈ 5.0V
    {1.0f,  3000},  // V_out ≈ 7.1V
    {2.0f,  2500},  // V_out ≈ 10V   (DAC=2.0V)
    {5.0f,  1500},  // V_out ≈ 15.8V (DAC=1.2V)
    {10.0f, 0},     // V_out ≈ 22.3V (DAC=0V)
};

/**
 * @brief 출력 전력 설정 (0.1W ~ 10W, 0.1W 단위)
 * @param power_watts 목표 전력 (W)
 */
void BUCK_SetPower(float power_watts)
{
    // 범위 제한
    if (power_watts < 0.1f) power_watts = 0.1f;
    if (power_watts > 10.0f) power_watts = 10.0f;

    // 전력 → 전압 변환 (P = V² / R)
    float target_voltage = sqrtf(power_watts * LOAD_IMPEDANCE);

    // 전압 → DAC 값 변환
    // V_out = V_OUT_MAX - (V_dac × DAC_GAIN)
    // V_dac = (V_OUT_MAX - target_voltage) / DAC_GAIN
    float v_dac = (V_OUT_MAX - target_voltage) / DAC_GAIN;

    // DAC 전압 → DAC 레지스터 값 (0~3.3V → 0~4095)
    if (v_dac < 0.0f) v_dac = 0.0f;
    if (v_dac > 3.3f) v_dac = 3.3f;

    uint16_t dac_value = (uint16_t)(v_dac / 3.3f * (float)DAC_MAX_VALUE);

    // DAC 출력 설정 (PA4)
    HAL_DAC_SetValue(&hdac1, DAC_CHANNEL_1, DAC_ALIGN_12B_R, dac_value);
}

/**
 * @brief 전력 피드백 제어 루프 (100ms 주기 호출)
 */
void BUCK_PowerControlLoop(void)
{
    static float error_integral = 0.0f;
    const float Kp = 0.5f;
    const float Ki = 0.1f;

    // ADC로 실제 전력 측정
    float actual_power = ADC_GetForwardPower();
    float target_power = g_target_power;

    // PI 제어
    float error = target_power - actual_power;
    error_integral += error * 0.1f;  // dt = 100ms

    // 적분 제한
    if (error_integral > 5.0f) error_integral = 5.0f;
    if (error_integral < -5.0f) error_integral = -5.0f;

    // 보정된 전력값 계산
    float corrected_power = target_power + (Kp * error) + (Ki * error_integral);

    // BUCK 출력 조정
    BUCK_SetPower(corrected_power);
}
```

### 브릿지 회로 동작 원리

CTL-6-P 전류 트랜스포머의 2차측 전류가 브릿지 저항망을 통해 분배됨.
출력 트랜스 2차측 전류의 방향에 따라 FWD/REV 노드에 차동 전압 발생.

**등가 저항 계산:**

```
R21 // R22 = 25Ω // 25Ω = 12.5Ω
R23 // R24 = 25Ω // 25Ω = 12.5Ω

FWD 경로 소스 임피던스: 12.5Ω
REV 경로 소스 임피던스: 12.5Ω

LTC5507 입력 종단: R25 = R26 = 50Ω
```

**신호 감쇠 (전압 분배):**

```
감쇠 = R_load / (R_source + R_load)
     = 50 / (12.5 + 50)
     = 0.8 (-1.9 dB)
```

**AC 커플링 고역통과 필터:**

```
fc = 1 / (2π × (Rs + RL) × C)
   = 1 / (2π × 62.5Ω × 10nF)
   ≈ 255 kHz

500kHz (Ch0)에서 추가 감쇠: -1.0 dB
2MHz (Ch6)에서 추가 감쇠: -0.1 dB (평탄)
```

**Period 계산 공식:**

```

├── 500 kHz  → 5.44e9 / 500e3  = 10,880 틱
├── 1000 kHz → 5.44e9 / 1000e3 = 5,440 틱
└── 2000 kHz → 5.44e9 / 2000e3 = 2,720 틱
```

### LTC5507 출력 특성

LTC5507은 **로그 검출기**이며, 출력 전압과 입력 RF 전력의 관계:

```
                    LTC5507 전달 특성 (100kHz~1GHz)
    V_out (V)
       │
   2.2 ┼────────────────────────────────────────╮
       │                                         ╲
   2.0 ┼                                          ╲
       │                                           ╲
   1.5 ┼                                            ╲
       │                                             ╲
   1.0 ┼                                              ╲
       │                                               ╲
   0.5 ┼                                                ╲
       │                                                 ╲
   0.1 ┼─────────────────────────────────────────────────╲
       └─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬──▶ P_in (dBm)
           -34   -25   -15    -5     0    +5   +10   +14

V_out ≈ 0.044 × P_in(dBm) + 1.67  [근사식, 25°C]
```

**ADC 변환 (12비트, 3.3V 기준):**

```
ADC_value = V_out / 3.3V × 4095

V_out → P_in 역변환:
P_in(dBm) = (V_out - 1.67) / 0.044
          = (ADC × 3.3 / 4095 - 1.67) / 0.044
```

### 공진주파수 탐색 알고리즘

PZT 트랜스듀서의 공진점에서 **순방향 전력 최대, 역방향(반사) 전력 최소**.

#### 방법 1: 순방향 전력 최대점 탐색

```c
/**
 * @brief 순방향 전력 최대점 탐색 (단순 스윕)
 * @param start_ch  시작 채널 (0~6)
 * @param end_ch    종료 채널 (0~6)
 * @param step_khz  스윕 스텝 (kHz), 0이면 채널 단위
 * @return 공진주파수 (kHz), 0이면 탐색 실패
 */
uint32_t FindResonance_FwdMax(uint8_t start_ch, uint8_t end_ch, uint16_t step_khz)
{
    uint32_t best_freq = 0;
    uint16_t max_fwd = 0;

    // 채널 단위 스윕 또는 세밀한 스윕
    for (uint32_t freq = FreqTable[start_ch]; freq <= FreqTable[end_ch];
         freq += (step_khz ? step_khz : 250))
    {
        // 주파수 설정
        HRTIM_SetFrequency(freq);
        HAL_Delay(10);  // 안정화 대기

        // 순방향 전력 측정
        uint16_t fwd = ADC_GetForwardPower_Raw();

        if (fwd > max_fwd) {
            max_fwd = fwd;
            best_freq = freq;
        }
    }

    return best_freq;
}
```

#### 방법 2: VSWR 최소점 탐색 (권장)

```c
/**
 * @brief VSWR(정재파비) 계산
 * @param fwd  순방향 ADC 값
 * @param rev  역방향 ADC 값
 * @return VSWR (×100, 정수), 예: 150 = VSWR 1.50
 */
uint16_t CalculateVSWR(uint16_t fwd, uint16_t rev)
{
    if (fwd <= rev) return 9999;  // 비정상 (무한대)

    // 반사계수 Γ = √(P_rev / P_fwd)
    // ADC 값은 전력에 비례하므로:
    // Γ = √(rev / fwd)

    float gamma = sqrtf((float)rev / (float)fwd);

    // VSWR = (1 + Γ) / (1 - Γ)
    if (gamma >= 0.99f) return 9999;

    float vswr = (1.0f + gamma) / (1.0f - gamma);

    return (uint16_t)(vswr * 100.0f);
}

/**
 * @brief VSWR 최소점 탐색 (공진주파수)
 */
uint32_t FindResonance_VSWRMin(uint8_t start_ch, uint8_t end_ch)
{
    uint32_t best_freq = 0;
    uint16_t min_vswr = 9999;

    for (uint8_t ch = start_ch; ch <= end_ch; ch++)
    {
        uint32_t freq = FreqTable[ch];

        HRTIM_SetFrequency(freq);
        HAL_Delay(10);

        uint16_t fwd = ADC_GetForwardPower_Raw();
        uint16_t rev = ADC_GetReversePower_Raw();
        uint16_t vswr = CalculateVSWR(fwd, rev);

        if (vswr < min_vswr) {
            min_vswr = vswr;
            best_freq = freq;
        }
    }

    return best_freq;
}
```

#### 방법 3: 2단계 탐색 (빠른 스윕 + 정밀 스윕)

```c
/**
 * @brief 2단계 공진주파수 탐색
 * 1단계: 채널 단위로 빠르게 스윕 → 대략적 공진 채널 탐색
 * 2단계: 해당 채널 ±125kHz 범위에서 10kHz 단위 정밀 스윕
 */
uint32_t FindResonance_TwoPhase(void)
{
    // 1단계: 채널 단위 스윕 (250kHz 간격)
    uint32_t coarse_freq = FindResonance_FwdMax(0, 6, 0);

    if (coarse_freq == 0) return 0;

    // 2단계: 정밀 스윕 (±125kHz 범위, 10kHz 스텝)
    uint32_t start = (coarse_freq > 125) ? coarse_freq - 125 : 500;
    uint32_t end = (coarse_freq < 1875) ? coarse_freq + 125 : 2000;

    uint32_t best_freq = coarse_freq;
    uint16_t max_fwd = 0;

    for (uint32_t freq = start; freq <= end; freq += 10)
    {
        HRTIM_SetFrequency(freq);
        HAL_Delay(5);

        uint16_t fwd = ADC_GetForwardPower_Raw();

        if (fwd > max_fwd) {
            max_fwd = fwd;
            best_freq = freq;
        }
    }

    return best_freq;
}
```

### 주의사항

1. **안정화 시간**: 주파수 변경 후 최소 5~10ms 대기 필요 (BUCK 출력 안정화)
2. **출력 전력**: 공진 탐색 시 저전력(0.1~0.3W)으로 수행 권장 (트랜스듀서 보호)
3. **온도 의존성**: PZT 공진주파수는 온도에 따라 ±1~2% 변동 가능 → 주기적 재탐색 권장
4. **케이블 영향**: 케이블 길이에 따라 위상 지연 발생 → 설치 후 초기 캘리브레이션 필수

                           │           │
                         Pin1        Pin2
                           │           │

                           │           │           │

**동작 원리:**

- CT의 Pin1, Pin2를 각각 R17, R19(50Ω)로 GND에 종단

**센터탭이 필요 없는 이유:**

### 11.1 LCD1602 + LED 제어 (74HCT574 ×2 — 방법 C)

#### 설계 방식

디스플레이 보드를 MCU 보드와 분리하여 별도 PCB로 제작.
**74HCT574 ×2 (8비트 D 플립플롭 래치)** 를 사용하여 공유 6비트 데이터 버스로
LCD(6핀)와 LED(6개)를 각각 독립 래치를 통해 제어.

- **74HCT574 전원: 5V** — HCT 계열은 TTL 입력 문턱(V_IH = 2.0V 고정)을 사용하므로
  MCU 3.3V 출력(최소 2.97V)이 5V VCC에서도 HIGH로 정상 인식됨
- **LCD1602 VDD: 5V 필수** — HD44780 동작 범위 4.5~5.5V. 3.3V 공급 시 동작 불안정
- **LCD 백라이트(LED+)**: 5V + 33Ω 직렬 저항 → 전류 제한 ~60mA 이내
- **버튼 풀업: 외부 10kΩ 풀업 저항** (3.3V ← R_PU 10kΩ ← 버튼 핀) — MCU 내부 풀업 미사용

- **커넥터: 20핀** — 기존 16핀 + ⑰ BUZZER(PB5) + ⑱ 5V + ⑲ 3.3V + ⑳ GND

### 11.2 외부 제어 입력 절연 (TLP281-4 광커플러)

#### 필요성

D-SUB 25핀 커넥터를 통해 외부 PLC, 릴레이 접점 등에서 24V 신호가 인가될 수 있음.
**MCU 보호 및 그라운드 루프 차단**을 위해 광커플러 절연 필수.

#### 외부 입력 5개 (광커플러 TLP281-4 ×1 + TLP181 ×1)

| 채널 | 신호         | 기능                     | MCU 핀 | IC              |
| ---- | ------------ | ------------------------ | ------ | --------------- |
| CH1  | REMOTE       | 발진 ON/OFF (4-COM 겸용) | PC6    | TLP281-4 (CH1)  |
| CH2  | BCD_BIT1     | 8단계 출력 선택 비트1    | PC10   | TLP281-4 (CH2)  |
| CH3  | BCD_BIT2     | 8단계 출력 선택 비트2    | PC11   | TLP281-4 (CH3)  |
| CH4  | BCD_BIT3     | 8단계 출력 선택 비트3    | PC12   | TLP281-4 (CH4)  |
| CH5  | SENSOR_INPUT | 외부 센서/비상 정지 입력 | PA10   | TLP181 (단채널) |

> SENSOR_INPUT(PA10)은 기존 RELAY_ADDR에서 재할당. 센서 OPEN 시 Err7/Err8 발생.
> TLP181: TLP281-4 단채널 등가 부품 (SOT-23-4 또는 DIP-4).

#### 저항값 계산

```
외부 24V 기준:
├── LED 전류 = (24V - 1.2V) / 4.7kΩ ≈ 4.9mA ✓ (권장 5~10mA)
├── CTR(전류전달비) = 50% (최소) → 출력 2.5mA 이상
└── 풀업 10kΩ → VOL ≈ 0.3V, VOH ≈ 3.3V ✓

외부 12V 기준:
├── LED 전류 = (12V - 1.2V) / 4.7kΩ ≈ 2.3mA
└── 저전류이지만 동작 가능 (CTR 여유 있음)

외부 5V 기준:
├── LED 전류 = (5V - 1.2V) / 4.7kΩ ≈ 0.8mA
└── 권장: R_LED = 1kΩ으로 변경 (3.8mA)
```

#### 입력 극성 (Active LOW)

- **접점 OPEN**: LED OFF → 트랜지스터 OFF → MCU 입력 = HIGH (풀업)
- **접점 SHORT**: LED ON → 트랜지스터 ON → MCU 입력 = LOW

> 매뉴얼과 동일: REMOTE **SHORT=발진ON**, **OPEN=발진OFF**

## 13. 주의사항 요약

### 클럭 소스 (HSI vs HSE 상세 분석)

#### 개요

STM32G474MET6는 두 가지 고속 클럭 소스를 지원합니다:

| 항목           | HSI (내부 RC)                | HSE (외부 크리스탈)          |
| -------------- | ---------------------------- | ---------------------------- |
| 주파수         | 16 MHz                       | 4~48 MHz (외부 부품)         |
| 정밀도 (25°C)  | ±1% (공장 트리밍)            | ±20ppm (크리스탈 의존)       |
| 온도 드리프트  | ±3% (-40~105°C 전 범위)      | ±50ppm (-40~85°C, AT컷 기준) |
| 외부 부품      | 없음                         | 크리스탈 + 부하 커패시터 2개 |
| 시작 시간      | ~2µs                         | ~2ms (크리스탈 안정화)       |
| 사용 핀        | 없음                         | **PF0-OSC_IN, PF1-OSC_OUT**  |
| PLL 입력       | ✅                           | ✅                           |
| HRTIM DLL 영향 | DLL이 자체 캘리브레이션 수행 | 더 안정적인 기준 클럭 제공   |

#### 핵심 문제: PF0/PF1 핀 충돌

LQFP80 패키지에서 HSE 크리스탈을 사용하면 **PF0, PF1이 GPIO로 사용 불가**합니다:

```
STM32G474MET6 LQFP80

    핀 11: PF0 / OSC_IN   ← HSE 사용 시 크리스탈 연결 (GPIO 불가)
    핀 12: PF1 / OSC_OUT  ← HSE 사용 시 크리스탈 연결 (GPIO 불가)
```

**현재 설계에서 PF0/PF1은 LC탱크 릴레이로 할당**:

| 핀  | 현재 용도        | HSE 사용 시     |
| --- | ---------------- | --------------- |
| PF0 | LC_RELAY3 (GPIO) | ❌ OSC_IN 전용  |
| PF1 | LC_RELAY4 (GPIO) | ❌ OSC_OUT 전용 |

#### HSI 선택 시 영향 분석

**1. HRTIM PWM 주파수 정밀도**

```
PLL 설정: HSI 16MHz → PLL → 170MHz SYSCLK
HRTIM DLL: 170MHz × 32 = 5.44GHz 유효 클럭

HSI ±1% 오차가 PWM 주파수에 미치는 영향:
├── 목표 500kHz → 실제 495~505 kHz (±5kHz)
├── 목표 1000kHz → 실제 990~1010 kHz (±10kHz)
└── 목표 2000kHz → 실제 1980~2020 kHz (±20kHz)

HSE ±20ppm 일 경우:
├── 목표 500kHz → 실제 499.99~500.01 kHz (±0.01kHz)
├── 목표 1000kHz → 실제 999.98~1000.02 kHz
└── 목표 2000kHz → 실제 1999.96~2000.04 kHz
```

> 메가소닉 트랜스듀서는 **공진주파수 대역폭(Q값)이 수 kHz~수십 kHz**이므로,
> HSI ±1% 오차(±5~20kHz)는 대역폭 내에서 충분히 허용됩니다.
> 또한 **실시간 전력 피드백(LTC5507)**으로 공진점을 추적하므로 절대 정밀도보다 상대 분해능이 중요합니다.

**2. Modbus RTU 보레이트 정밀도**

```
USART2 보레이트: 9600 bps (또는 19200/38400)

RS-485 허용 오차: ±3% (일반적 UART 동기화 마진)

HSI ±1% → USART 보레이트 오차 ±1% → ✅ 3% 이내 (충분)
HSI ±3% (극한 온도) → USART 보레이트 오차 ±3% → ⚠️ 경계

HSE ±20ppm → USART 보레이트 오차 무시 수준
```

> 산업 현장 온도(0~60°C)에서 HSI 오차는 ±1.5% 이내이므로 Modbus 통신에 문제 없습니다.
> 극한 환경(-40~105°C)에서만 HSE가 필요합니다.

**3. ADC 샘플링 타이밍**

```
ADC 클럭 = SYSCLK / 분주 (보통 /4 = 42.5MHz)

HSI ±1% → ADC 변환 시간 ±1% → 측정값 자체에는 영향 없음
(ADC는 S&H 방식이므로 클럭 주파수가 아닌 기준 전압 정밀도가 중요)
```

> ADC 정밀도는 VREF+ 전압에 의존하며, 클럭 오차의 영향은 무시할 수 있습니다.

#### HSE가 필요한 경우와 대응

만약 향후 HSE가 필요하게 되면:

**대응 방안 A: LC_RELAY3/4 핀 재배치**

| 기존 핀 | 기존 용도 | 이동 핀 | 비고        |
| ------- | --------- | ------- | ----------- |
| PF0     | LC_RELAY3 | **PC7** | 여유핀 활용 |
| PF1     | LC_RELAY4 | **PC8** | 여유핀 활용 |

```
HSE 크리스탈 회로:
        PF0 (OSC_IN)          PF1 (OSC_OUT)
            │                      │
       ┌────┴────┐            ┌────┴────┐
       │ [CL1]   │            │ [CL2]   │
       │ 10~20pF │            │ 10~20pF │
       │         │            │         │
       └────┬────┘            └────┬────┘
            │                      │
            └──────────────────────┘
                      │
                     GND

    CL1, CL2: 부하 커패시터 (크리스탈 데이터시트 참조)
    권장 크리스탈: 8MHz (PLL로 170MHz 생성)
```

**대응 방안 B: HSI 캘리브레이션 강화**

HSE 없이 정밀도를 높이는 방법:

1. **공장 트리밍 활용**: RCC_ICSCR 레지스터의 HSITRIM[6:0] 비트로 ±0.5% 이내 조정 가능
2. **온도 보상 테이블**: 온도 센서(ADC 내장)로 HSI 편차를 소프트웨어 보정
3. **외부 기준 신호**: Modbus 마스터 통신 타이밍으로 주기적 보정

#### 결론 — HSE 8MHz 크리스탈 채택 (v3)

| 항목                | HSE (현재 설계)     | HSI (이전 설계) |
| ------------------- | ------------------- | --------------- |
| **정밀도**          | ⭐ ±20ppm           | ±1% (10,000ppm) |
| **PF0/PF1**         | 크리스탈 전용       | GPIO 사용 가능  |
| **LC탱크 릴레이**   | PC7/PC8로 이동      | PF0/PF1 할당    |
| **PWM 500kHz 오차** | ±0.01kHz            | ±5kHz           |
| **Modbus 통신**     | 무시 수준           | ±1%             |
| **외부 부품**       | 크리스탈+C×2 추가   | 없음            |
| **여유핀**          | 3개 (PA5, PC9, PB9) | 5개             |

> ✅ **현재 설계는 HSE 8MHz 크리스탈 사용 (v3)**.
> PF0(OSC_IN)/PF1(OSC_OUT)에 8MHz 크리스탈 연결.
> LC_RELAY3/4는 PC7/PC8로 재배치 완료.

#### HSE 크리스탈 회로

```
                     3.3V (또는 VDD)
                           │
                     ┌─────┴─────┐
                     │ [R_fb]    │
                     │ 1MΩ      │ ← 피드백 저항 (발진 안정화)
                     │           │
    PF0 (OSC_IN) ────┼─────┬─────┼──── PF1 (OSC_OUT)
         핀 11       │     │     │      핀 12
                     │   ┌─┴─┐   │
                     │   │   │   │
                     │   │ Y1│   │ ← 8 MHz 크리스탈
                     │   │   │   │   HC49/S 또는 3215 패키지
                     │   └─┬─┘   │
                     │     │     │
                ┌────┴─┐   │   ┌─┴────┐
                │[CL1] │   │   │[CL2] │
                │12pF  │   │   │12pF  │ ← 부하 커패시터 (C0G/NP0)
                └──┬───┘   │   └──┬───┘
                   │       │      │
                   └───────┴──────┘

```

**부품:**

| 부품  | 사양                        | 수량 | 비고                        |
| ----- | --------------------------- | ---- | --------------------------- |
| Y1    | 8 MHz, HC49/S 또는 3215 SMD | 1    | CL = 10~20pF, ESR < 100Ω    |
| CL1/2 | 12pF, C0G/NP0, 50V          | 2    | CL = 2×(C_load - C_stray)   |
| R_fb  | 1MΩ (옵션)                  | 1    | 발진 안정화, 일부 설계 생략 |

**부하 커패시터 계산:**

```
CL = 크리스탈 데이터시트 Load Capacitance (보통 10~20pF)
C_stray = PCB 기생 커패시턴스 (보통 2~5pF)

CL_ext = 2 × (CL - C_stray)
       = 2 × (10pF - 3pF) = 14pF → 표준값 12pF 선택

예시 (CL=10pF 크리스탈):  CL1 = CL2 = 12pF
예시 (CL=20pF 크리스탈):  CL1 = CL2 = 33pF
```

**PCB 배치 주의:**

1. CL1, CL2는 PF0/PF1 핀에서 **5mm 이내** 배치
2. 크리스탈~커패시터 GND 비아는 **최단 거리**
3. 고주파 신호 배선(HRTIM 등)과 **격리** (크로스토크 방지)
4. 크리스탈 아래 GND 플레인 확보 (차폐 효과)

### 15.2 리셋 회로 (NRST)

```
        3.3V
         │
        [R_RST: 10kΩ]  ← 풀업 저항
         │
         ├───────────┬─────────────────────────▶ NRST (핀 7)
         │           │
        [C_RST]     [D_RST]
        100nF       BAV99                      ← ESD 보호 다이오드
         │           │
        GND        3.3V
         │
         │
       ┌─┴─┐
       │ ○ │  ← SW_RST (리셋 버튼, 옵션)
       │   │     택트 스위치
       └─┬─┘
         │
        GND
```

**리셋 시간 상수:**

```
τ = R × C = 10kΩ × 100nF = 1ms

전원 투입 시 NRST가 LOW로 유지되는 시간:
t_reset ≈ 5τ = 5ms (안전한 리셋 보장)
```

**참고:**

- STM32G4 내부에는 NRST 풀업이 있으나, 외부 풀업 추가 권장
- 리셋 버튼은 PCB 테스트/디버그용 (양산 시 생략 가능)

## 17. LED 직접 구동 회로

### 17.1 STM32G474 GPIO 직접 구동 방식

GPIO 출력(3.3V, 최대 25mA/핀)으로 LED를 직접 구동.

```
3.3V (VDD)
    │
    ├─────────────────────────────────────────────────────────────┐
    │                                                             │
    │   GPIO (PB0~PB2, PB10, PB11, PB14)                         │
    │     │                                                       │
    │     │                                                       │
    │   ┌─┴─┐                                                     │
    │   │   │ ← GPIO 핀 (푸시풀 출력)                             │
    │   │ ● │                                                     │
    │   │   │                                                     │
    │   └─┬─┘                                                     │
    │     │                                                       │
    │    [R_LED]  ← 전류 제한 저항                                │
    │     │                                                       │
    │   ┌─┴─┐                                                     │
    │   │ ▼ │ ← LED (애노드가 저항 쪽)                            │
    │   │LED│                                                     │
    │   └─┬─┘                                                     │
    │     │                                                       │
    │    GND                                                      │
    │                                                             │
    └─────────────────────────────────────────────────────────────┘
```

> **Active HIGH 구동**: GPIO HIGH → LED ON

### 17.2 저항값 계산

**공식:**

```
R = (V_GPIO - V_f) / I_f

V_GPIO = 3.3V (STM32 출력 VOH)
V_f = 순방향 전압 (LED 색상별 상이)
I_f = 순방향 전류 (10mA 권장, 밝기와 수명 균형)
```

**LED 색상별 계산:**

| LED 색상      | V_f (typ) | I_f  | R 계산               | 권장 표준값 | 소비전력 |
| ------------- | --------- | ---- | -------------------- | ----------- | -------- |
| 적색 (Red)    | 2.0V      | 10mA | (3.3-2.0)/10m = 130Ω | **150Ω**    | 15mW     |
| 황색 (Yellow) | 2.1V      | 10mA | (3.3-2.1)/10m = 120Ω | **120Ω**    | 14.4mW   |
| 녹색 (Green)  | 2.2V      | 10mA | (3.3-2.2)/10m = 110Ω | **100Ω**    | 11mW     |
| 청색 (Blue)   | 3.2V      | 10mA | (3.3-3.2)/10m = 10Ω  | **22Ω**     | 2.2mW    |
| 백색 (White)  | 3.2V      | 10mA | (3.3-3.2)/10m = 10Ω  | **22Ω**     | 2.2mW    |

> ⚠️ **청색/백색 LED**: Vf가 3.2V로 높아 전압 마진이 0.1V밖에 안됨.
> GPIO VOH가 2.9V까지 떨어질 수 있으므로 **적색/녹색/황색** 권장.
