1.2 LCD1602 + LED 제어 (방법 C: 74HCT574 ×2) v4 신규

MCU 보드와 분리된 디스플레이 보드에 74HCT574 ×2를 탑재. 공유 6비트 DATA 버스(PB10~PB15)에 CLK_LCD(PD0) / CLK_LED(PD1) 상승엣지로 LCD와 LED를 독립 래치 제어. 버튼 외부 10kΩ 풀업, 부저(KEC105S) 포함. 커넥터: 20핀 (기존 16핀 + ⑰BUZZER + ⑱5V + ⑲3.3V + ⑳GND).
74HCT574 VCC: 5V (HCT 계열: V_IH=2.0V 고정 → MCU 3.3V 출력으로 정상 구동). LCD1602 VDD: 5V 필수 (HD44780: 4.5~5.5V 범위).

2시스템 클럭 — HSE 8MHz → PLL → 170MHz v3 HSE 적용

HSE 8MHz 크리스탈 사용으로 정밀도 ±20ppm 확보 (HSI ±1% 대비 50배 개선). HRTIM 주파수 오차 최소화.

2.1 PLL 설정

입력  : HSE = 8 MHz (PF0/PF1 크리스탈)
PLLM  : ÷2  → 8 / 2 = 4 MHz (PLL 입력)
PLLN  : ×85 → 4 × 85 = 340 MHz (VCO)
PLLR  : ÷2  → 340 / 2 = 170 MHz (SYSCLK)

AHB   = 170 MHz  (RCC_SYSCLK_DIV1)
APB1  = 170 MHz  (RCC_HCLK_DIV1)
APB2  = 170 MHz  (RCC_HCLK_DIV1)
Flash Latency = 4 WS (150~170 MHz, Range1 Boost)

HRTIM 유효 클럭 = APB2×32(DLL) = 170×32 = 5,440 MHz
1 틱 = 1/5.44 GHz ≈ 184 ps

2.2 HAL 초기화 코드 요점

/* 전원 스케일 설정 (170MHz를 위해 Range1 Boost 필수) */
HAL_PWREx_ControlVoltageScaling(PWR_REGULATOR_VOLTAGE_SCALE1_BOOST);

/* HSE + PLL 설정 */
rcc_osc.OscillatorType = RCC_OSCILLATORTYPE_HSE;
rcc_osc.HSEState       = RCC_HSE_ON;
rcc_osc.PLL.PLLSource  = RCC_PLLSOURCE_HSE;
rcc_osc.PLL.PLLM       = RCC_PLLM_DIV2;   /* ÷2 → 4MHz */
rcc_osc.PLL.PLLN       = 85U;              /* ×85 → 340MHz VCO */
rcc_osc.PLL.PLLR       = RCC_PLLR_DIV2;   /* ÷2 → 170MHz SYSCLK */
HAL_RCC_OscConfig(&rcc_osc);

/* Flash Latency 4WS */
HAL_RCC_ClockConfig(&rcc_clk, FLASH_LATENCY_4);

/* HRTIM DLL 캘리브레이션 (반드시 HRTIM 사용 전 호출) */
HAL_HRTIM_DLLCalibrationStart(&hhrtim, HRTIM_CALIBRATIONRATE_14);
HAL_HRTIM_PollForDLLCalibration(&hhrtim, HAL_MAX_DELAY);

8BUCK 전력 컨버터 (LM5005)

8.1 개요

48V SMPS 입력에서 가변 DC 전압(2.2V~22.4V)을 생성하여 하프브리지의 Vbus를 조절, 출력 전력 0.1W~10W를 제어.

항목 사양

출력 전압 2.2V ~ 22.4V (가변, DAC 제어)
출력 전력 (@50Ω 부하) 0.1W ~ 10W
스위칭 주파수 100kHz (R_T=330kΩ, 48V 입력 최적)
FB 기준 전압 1.23V
패키지 SOIC-14 (손납땜)

8.2 DAC 전압 제어 회로

     V_out (2.2V ~ 22.4V)
         │
        [R1: 11kΩ, 1%] ◀── 최대 전압 결정
         │
         ├──────────────────────────────▶ FB (LM5005 핀7)
         │                                     │
        [R2: 1kΩ, 1%]                         [R3: 1.8kΩ, 1%]
         │                                     │
        GND                            DAC1_OUT1 (PA4) / STM32G474

V_out = 1.23 × (1 + R1/R2) + (1.23 - V_dac) × R1/R3
      = 14.76 + (1.23 - V_dac) × 6.11

DAC = 0.0V  → V_out = 22.3V → P = 9.9W   ✓
DAC = 1.0V  → V_out = 16.2V → P = 5.2W
DAC = 1.23V → V_out = 14.8V → P = 4.4W   (FB=DAC, 전류=0)
DAC = 2.0V  → V_out = 10.1V → P = 2.0W
DAC = 3.0V  → V_out =  3.9V → P = 0.31W
DAC = 3.3V  → V_out =  2.1V → P = 0.09W  ✓

※ 48V 입력 시 최소 듀티 한계: V_out 최소 2.7V 권장 (0.15W)
   → R_T = 330kΩ (100kHz) 사용 시 ON-time margin 충분

SWD 핀 (PA13/PA14) 테스트 패드 또는 2.54mm 헤더 노출. 주변 신호와 간섭 없도록 별도 영역 확보.
BOOT0 핀 10kΩ 풀다운 저항으로 GND 고정 (플래시 부트). 점퍼/버튼으로 HIGH 전환 옵션 추가.

14최종 회로 설계 점검 체크리스트

회로 설계 착수 전 아래 항목을 반드시 확인하세요. Critical 항목 미충족 시 기능 불가.

🔴 Critical — 기능 불가 수준

  • PF0/PF1을 GPIO로 사용하지 않았는가? (HSE 크리스탈 전용, GPIO_Init에서 제외)
  • PA13/PA14을 GPIO로 사용하지 않았는가? (SWD 전용)
  • PA8/PA9 핀을 AF13 (HRTIM1)으로 설정하고 VERY_HIGH 속도로 구성했는가?
  • HRTIM DLL 캘리브레이션 코드가 HRTIM 사용 전에 반드시 호출되는가?
  • UCC21520 VSSA가 SW 노드에, VSSB가 파워 GND에 각각 연결되었는가? (절대 공유 금지)
  • 하이사이드 게이트 전원 VDDA가 반드시 절연된 DC-DC (R1215S 등)에서 공급되는가?
  • LM5005 FB 분압 저항이 R1=11kΩ, R2=1kΩ, R3=1.8kΩ으로 설계되었는가? (기존 회로 R79/R80 값 사용 불가)
  • LC_RELAY3/4가 PC7/PC8에 연결되었는가? (PF0/PF1 아님)
  • RS-485 DE가 PC13에, /RE가 PC14에 연결되었는가? (PA4/PA5 아님)
  • PA4가 DAC1_OUT1로만 사용되고 다른 신호와 충돌 없는가?

🟠 Important — 오동작 가능

  • HSE 크리스탈 부하 커패시터(CL1/CL2)가 크리스탈 데이터시트 기준으로 선택되었는가? (일반적으로 9~22pF)
  • VDDA/VSSA, VDDB/VSSB 핀 바로 옆에 바이패스 커패시터(10µF + 100nF) 배치되는가?
  • STM32 VDDA 핀에 100nF + 1µF 바이패스 커패시터가 있는가?
  • ADC 캘리브레이션 HAL_ADCEx_Calibration_Start() 호출이 펌웨어에 있는가?
  • UCC21520 DT 핀(100kΩ)과 HRTIM 내장 데드타임 중 하나만 사용하는가? (이중 사용 금지)
  • RS-485 버스 끝단에만 120Ω 종단 저항이 있는가?
  • RS-485 송신 완료(USART2 TC 플래그) 확인 후 DE→LOW 전환 코드가 있는가?
  • BOOT0 핀에 10kΩ 풀다운 저항이 있는가? (프로그래밍 후 GND 고정 필요)
  • LCD1602의 R/W 핀이 GND에 고정되었는가? (쓰기 전용)
  • LCD1602 V0 핀에 대비 조절용 가변저항(10kΩ)이 있는가?
  • 버튼 6개 (PC0~PC5) 내부 풀업 설정으로 GPIO_INPUT 구성되었는가?
  • LED 6개에 전류 제한 저항(330Ω~1kΩ)이 있는가?
  • ULN2003A COM 핀이 릴레이 코일 전원(5V)에 연결되었는가?
  • 알람 릴레이 접점 사양이 AC 100V/DC 100V, 0.3A 이내인가?
  • 광커플러(TLP281-4) 외부 전류 제한 저항이 적절한 값으로 설계되었는가?

🟡 권장 — 성능 및 신뢰성

  • 2MHz 고주파 동작 시 Si MOSFET(IPB200N15N3) 발열 검토 → 방열판 필요 여부 확인
  • HRTIM PWM 트레이스(PA8→UCC21520, PA9→UCC21520)가 최단 거리로 배선되었는가?
  • 전력 루프(Vbus→Q1→SW노드→Q2→GND) 면적이 최소화되었는가?
  • UCC21520 1차측/2차측 사이 PCB 절연 거리 8mm 이상 확보되었는가?
  • HSE 크리스탈과 MCU 사이 트레이스 길이가 20mm 이내인가?
  • RF 전력 검출기 출력 전압이 VDDA(3.3V) 이내인지 확인되었는가?
  • BUCK 컨버터 인덕터(47µH)와 출력 커패시터(100µF×2) 배치가 루프 면적 최소화되었는가?
  • 4층 PCB 또는 연속 GND 플레인이 확보되었는가?
  • LM5005 R_T = 330kΩ (100kHz, 48V 입력 최적)으로 설정되었는가?
  • RELAY_OPERATE에 24V 0.1A 이상 접점을 사용하는가?
  • DAC1_OUT1 (PA4) 출력 전압(0~3.3V)이 LM5005 FB 노드를 초과하지 않도록 설계되었는가?
  • 디버그 USART1 (PB6/PB7)이 테스트 포인트로 노출되었는가?
  • NRST 핀이 ST-Link 커넥터에 연결되었는가?
