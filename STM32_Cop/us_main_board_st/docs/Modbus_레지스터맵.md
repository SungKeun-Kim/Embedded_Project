# Modbus RTU 통신 프로토콜 사양서

**적용 장치**: STM32G474RC 초음파 발진기 메인보드  
**프로토콜**: Modbus RTU (표준 준거)  
**문서 버전**: 1.0  
**최종 수정**: 2026-03-13

> **참고**: 본 문서는 기존 RS485 커스텀 프로토콜(RS485_protocol.md)의 모든 기능을 표준 Modbus RTU로 재정의한 것입니다.

---

## 1. 물리 계층 (Physical Layer)

### 1.1 하드웨어 인터페이스

| 항목         | 사양                                        |
| ------------ | ------------------------------------------- |
| 통신 규격    | RS-485 반이중 (Half-Duplex)                 |
| 트랜시버     | MAX3485 / SP3485 (3.3V 동작)                |
| 커넥터       | 2선식 (A, B) + GND                          |
| 종단 저항    | 120Ω (RTERM 핀으로 소프트웨어 제어 가능)    |
| 최대 노드 수 | 32 (표준), 256 (고입력 임피던스 IC 사용 시) |
| 최대 케이블  | 1,200 m (9600 bps 기준)                     |

### 1.2 UART 설정

| 항목      | 기본값        | 설정 범위                        |
| --------- | ------------- | -------------------------------- |
| 통신 속도 | 9600 bps      | 9600 / 19200 / 38400 / 115200    |
| 데이터    | 8 bits        | 고정                             |
| 패리티    | None          | None / Even / Odd                |
| 정지 비트 | 2 bits (N 시) | N→2비트, E/O→1비트 (Modbus 규격) |

> **Modbus 규격 참고**: 패리티 None일 때 Stop bit 2개, Even 또는 Odd일 때 Stop bit 1개를 사용하여 11비트 프레임을 유지합니다.

### 1.3 RS-485 방향 제어 핀

| 핀   | 포트        | 기능                                                      |
| ---- | ----------- | --------------------------------------------------------- |
| PA2  | USART2_TX   | RS485 TXD                                                 |
| PA3  | USART2_RX   | RS485 RXD                                                 |
| PA5  | GPIO_Output | DE (Driver Enable, HIGH=송신 활성화) — USART2 인접 배치   |
| PA6  | GPIO_Output | /RE (Receiver Enable, LOW=수신 활성화) — USART2 인접 배치 |
| PC11 | GPIO_Output | RTERM (종단 저항 ON/OFF)                                  |
| PC12 | GPIO_Input  | Board_Detect (옵션 보드 장착 인식, 메뉴 표시 연동용)      |

> **핀 변경 이력**: DE를 PC9→PA5, /RE를 PC10→PA6으로 이동하여 USART2(PA2/PA3) 인접 배치. MAX3485 배선 최단 거리 확보.

**방향 전환 타이밍:**

```
대기/수신: DE=LOW,  /RE=LOW   → RXNE 인터럽트로 바이트 수신
송신 시작: DE=HIGH, /RE=HIGH  → 데이터 전송
송신 완료: TC 플래그 확인 후 DE=LOW, /RE=LOW 복귀 (필수!)
```

## 2. 데이터 링크 계층 (Modbus RTU 프레임)

### 2.1 프레임 구조

표준 Modbus RTU 프레임 형식을 따릅니다.

```
┌──────────┬──────────┬────────────────┬───────────┐
│ 슬레이브 │ 펑션코드 │   데이터 필드  │  CRC-16   │
│ 주소     │          │  (가변 길이)   │           │
├──────────┼──────────┼────────────────┼───────────┤
│ 1 byte   │ 1 byte   │  N bytes       │ 2 bytes   │
└──────────┴──────────┴────────────────┴───────────┘
```

> **기존 프로토콜과의 핵심 차이점:**
>
> - **바이트 순서**: Modbus RTU는 **Big Endian** (MSB 우선)입니다. 기존 프로토콜의 Little Endian과 반대이므로 주의하십시오.
> - **CRC**: CRC-CCITT(x^16+x^12+x^5+1)가 아닌, **CRC-16/Modbus** (다항식 0xA001, LSB-first) 사용.
> - **펑션코드**: 기존의 쓰기=1, 읽기=2 대신, 표준 Modbus 펑션코드(0x03, 0x06, 0x10 등) 사용.

### 2.2 슬레이브 주소

| 주소    | 용도                             |
| ------- | -------------------------------- |
| 0       | 브로드캐스트 (응답 없음, 쓰기만) |
| 1–247   | 개별 장치 주소 (기본값: 1)       |
| 248–255 | 예약                             |

### 2.3 프레임 간 묵음 구간

| 통신 속도  | 3.5 캐릭터 시간 | 1.5 캐릭터 시간 |
| ---------- | --------------- | --------------- |
| 9600 bps   | 4.01 ms         | 1.72 ms         |
| 19200 bps  | 2.01 ms         | 0.86 ms         |
| 38400 bps  | 1.75 ms (최소)  | 0.75 ms (최소)  |
| 115200 bps | 1.75 ms (최소)  | 0.75 ms (최소)  |

> 19200 bps 이상에서는 타이머 기반 고정 시간(1.75 ms / 0.75 ms)을 사용합니다 (Modbus 표준 권고).

### 2.4 CRC-16 계산

Modbus RTU 표준 CRC-16을 사용합니다.

- **다항식**: 0xA001 (반전 표현), 원래 다항식 x^16+x^15+x^2+1
- **초기값**: 0xFFFF
- **바이트 순서**: CRC Low byte 먼저, CRC High byte 나중 (LSB-first)

```c

uint16_t Modbus_CRC16(const uint8_t *data, uint16_t length)
{
    uint16_t crc = 0xFFFF;
    while (length--) {
        uint8_t index = (uint8_t)(crc ^ *data++);
        crc = (crc >> 8) ^ crc16_table[index];
    }
    return crc;  /* CRC Low 먼저, CRC High 나중에 전송 */
}
```

> **⚠ 기존 프로토콜 마이그레이션 주의**: 기존 RS485_protocol.md에서 사용하던 CRC-CCITT(x^16+x^12+x^5+1, 초기값 0x0000)와 **완전히 다른** 알고리즘입니다. 기존 CRC 테이블(crc_table_high/crc_table_low)은 더 이상 사용하지 않습니다.

## 3. 지원 펑션 코드 (Function Codes)

### 3.1 펑션 코드 목록

| 코드 | 명칭                     | 설명                     | 기존 프로토콜 대응 |
| ---- | ------------------------ | ------------------------ | ------------------ |
| 0x03 | Read Holding Registers   | 1개 이상의 레지스터 읽기 | 기능=0x02 (읽기)   |
| 0x06 | Write Single Register    | 단일 레지스터 쓰기       | 기능=0x01 (쓰기)   |
| 0x10 | Write Multiple Registers | 복수 레지스터 연속 쓰기  | (신규 기능)        |

> **기존 프로토콜 대비 변경점**: 기존의 기능코드 쓰기=1, 읽기=2를 Modbus 표준 FC 0x03, 0x06, 0x10으로 변경합니다.

### 3.2 FC 0x03 — Read Holding Registers (레지스터 읽기)

연속된 1개 이상의 Holding Register(16비트)를 읽습니다.

**요청 프레임:**

```
┌───────┬───────┬─────────────┬─────────────┬────────┐
│ 주소  │ 0x03  │ 시작 레지스터│ 레지스터 수 │ CRC-16 │
│ 1byte │ 1byte │ 2bytes (MSB)│ 2bytes (MSB)│ 2bytes │
└───────┴───────┴─────────────┴─────────────┴────────┘
```

**정상 응답 프레임:**

```
┌───────┬───────┬───────────┬───────────────────┬────────┐
│ 주소  │ 0x03  │ 바이트 수 │ 레지스터 데이터들  │ CRC-16 │
│ 1byte │ 1byte │ 1byte     │ N×2 bytes (MSB)   │ 2bytes │
└───────┴───────┴───────────┴───────────────────┴────────┘
```

**예시 — 주파수 설정값 읽기 (슬레이브 주소 0x01, 레지스터 0x0001, 1개):**

```
요청: [01] [03] [00 01] [00 01] [D5 CA]
응답: [01] [03] [02] [01 1A] [B8 44]
→ 값 0x011A = 282 → 28.2 kHz (×0.1 kHz)
```

**예시 — 운전 제어 레지스터 일괄 읽기 (0x0000부터 10개):**

```
요청: [01] [03] [00 00] [00 0A] [C5 CD]
→ 한 번의 요청으로 동작상태, 주파수, 출력, 모드 등 10개 레지스터를 한꺼번에 읽음
```

### 3.3 FC 0x06 — Write Single Register (단일 레지스터 쓰기)

단일 Holding Register에 값을 기록합니다.

**요청 프레임:**

```
┌───────┬───────┬─────────────┬─────────────┬────────┐
│ 주소  │ 0x06  │ 레지스터주소│ 기록할 값   │ CRC-16 │
│ 1byte │ 1byte │ 2bytes (MSB)│ 2bytes (MSB)│ 2bytes │
└───────┴───────┴─────────────┴─────────────┴────────┘
```

**정상 응답**: 요청 프레임과 동일한 내용을 그대로 에코합니다 (기존 프로토콜의 쓰기 응답과 동일한 원리).

**예시 — 출력값 500으로 설정 (슬레이브 0x01, 레지스터 0x0002):**

```
요청: [01] [06] [00 02] [01 F4] [E9 B1]
응답: [01] [06] [00 02] [01 F4] [E9 B1]  ← 동일 에코
→ 값 0x01F4 = 500 (Fullscale 출력)
```

### 3.4 FC 0x10 — Write Multiple Registers (복수 레지스터 쓰기)

연속된 여러 Holding Register에 한 번에 값을 기록합니다.

**요청 프레임:**

```
┌───────┬───────┬─────────────┬─────────────┬───────────┬────────────────┬────────┐
│ 주소  │ 0x10  │ 시작 레지스터│ 레지스터 수 │ 바이트 수 │ 레지스터 값들  │ CRC-16 │
│ 1byte │ 1byte │ 2bytes      │ 2bytes      │ 1byte     │ N×2 bytes      │ 2bytes │
└───────┴───────┴─────────────┴─────────────┴───────────┴────────────────┴────────┘
```

**정상 응답:**

```
┌───────┬───────┬─────────────┬─────────────┬────────┐
│ 주소  │ 0x10  │ 시작 레지스터│ 레지스터 수 │ CRC-16 │
│ 1byte │ 1byte │ 2bytes      │ 2bytes      │ 2bytes │
└───────┴───────┴─────────────┴─────────────┴────────┘
```

**예시 — 스윕 시작/끝 주파수 동시 설정 (0x0012~0x0013):**

```
요청: [01] [10] [00 12] [00 02] [04] [01 0E] [01 36] [CRC CRC]
→ 0x0012=270 (27.0kHz), 0x0013=310 (31.0kHz) 동시 기록
```

### 3.5 예외 응답 (Exception Response)

요청이 유효하지 않을 경우, 펑션코드의 최상위 비트(MSB)에 0x80을 OR하여 에러 응답합니다.

> **기존 프로토콜 호환**: 기존에도 기능값에 0x80을 OR하여 에러를 알렸으므로, 이 동작 방식은 동일합니다.

```
┌───────┬──────────────┬───────────┬────────┐
│ 주소  │ FC + 0x80    │ 예외 코드 │ CRC-16 │
│ 1byte │ 1byte        │ 1byte     │ 2bytes │
└───────┴──────────────┴───────────┴────────┘
```

| 예외 코드 | 명칭                 | 설명                            | 기존 프로토콜 대응        |
| --------- | -------------------- | ------------------------------- | ------------------------- |
| 0x01      | Illegal Function     | 지원하지 않는 펑션코드          | —                         |
| 0x02      | Illegal Data Address | 존재하지 않는 레지스터 주소     | 명령/값 범위 초과 시 에러 |
| 0x03      | Illegal Data Value   | 허용 범위를 벗어난 값           | 명령/값 범위 초과 시 에러 |
| 0x04      | Slave Device Failure | 장치 내부 오류로 요청 처리 불가 | (신규)                    |

> **무응답 상황**: CRC 불일치, 주소 불일치, 프레임 훼손 등의 경우 어떤 응답도 하지 않습니다. 마스터는 타임아웃으로 통신 에러를 판단합니다 (기존 프로토콜과 동일).

## 4. 레지스터 맵 (Register Map)

레지스터 주소는 기능별로 **16개(0x10) 단위 블록**으로 정렬하여 그룹 내 연속 읽기(`FC 0x03`)에 최적화하였습니다.

### 4.1 레지스터 맵 전체 요약

#### 그룹 0x00: 운전 제어 (Operational Control) — 10개 연속 읽기 가능

| 주소   | R/W | 명칭                | 단위/범위                                 | 기존 프로토콜 대응      |
| ------ | --- | ------------------- | ----------------------------------------- | ----------------------- |
| 0x0000 | R/W | 동작 ON/OFF         | 0=OFF, 1=ON                               | 출력 동작 상태 (0x0201) |
| 0x0001 | R/W | 주파수 설정값       | ×0.1 kHz (200–1680 → 20.0–168.0 kHz)      | 주파수 (0x0100)         |
| 0x0002 | R/W | 출력 설정값         | 0–500 (Fullscale=500 정규화, 기존과 동일) | 출력 (0x0102)           |
| 0x0003 | R   | 현재 출력 주파수    | ×0.1 kHz                                  | 주파수 읽기 (0x0100)    |
| 0x0004 | R   | 현재 출력값         | 0–500                                     | 출력 읽기 (0x0102)      |
| 0x0005 | R   | 동작 상태 플래그    | 비트 필드 (§4.2 참조)                     | 각종 상태 읽기 통합     |
| 0x0006 | R/W | 동작 모드           | 0=연속, 1=펄스, 2=스윕                    | 스윕 동작 상태 (0x0203) |
| 0x0007 | R   | 출력 표시 범위      | 표시용 Fullscale 값 (W 또는 %)            | 표시 출력 범위 (0x0103) |
| 0x0008 | R/W | 로컬 입력 금지      | 0=허용, 1=금지                            | 로컬 입력 금지 (0x0200) |
| 0x0009 | R   | 외부 입력 동작 상태 | 0=OFF, 1=ON                               | 외부 입력 상태 (0x0202) |

#### 그룹 0x10: 펄스/스윕 설정 (Pulse & Sweep) — 5개 연속 읽기/쓰기 가능

| 주소   | R/W | 명칭             | 단위/범위           | 기존 프로토콜 대응         |
| ------ | --- | ---------------- | ------------------- | -------------------------- |
| 0x0010 | R/W | 펄스 ON 시간     | ms (10–10000)       | (신규 기능)                |
| 0x0011 | R/W | 펄스 OFF 시간    | ms (10–10000)       | (신규 기능)                |
| 0x0012 | R/W | 스윕 시작 주파수 | ×0.1 kHz (200–1680) | 스윕 주파수 (0x0104)       |
| 0x0013 | R/W | 스윕 끝 주파수   | ×0.1 kHz (200–1680) | (신규: 기존은 단일 주파수) |
| 0x0014 | R/W | 스윕 시간        | ms (100–60000)      | (신규 기능)                |

#### 그룹 0x20: 에러/진단 (Diagnostics) — 확장성 확보

| 주소   | R/W | 명칭               | 단위/범위                                | 기존 프로토콜 대응 |
| ------ | --- | ------------------ | ---------------------------------------- | ------------------ |

| 0x0022 | R   | 누적 운전 시간 (H) | 시간 (32비트 중 상위 16비트)             | (신규: 추천)       |
| 0x0023 | R   | 누적 운전 시간 (L) | 분 (32비트 중 하위 16비트)               | (신규: 추천)       |
| 0x0024 | R   | MCU 내부 온도      | ×0.1 °C (내부 온도 센서)                 | (신규: 추천)       |
| 0x0025 | R   | 입력 전류값        | ×0.01 A (ACS/CT 센서 ADC 환산)           | (신규: 추천)       |

| 주소   | R/W | 명칭                 | 단위/범위                          | 기존 프로토콜 대응       |

| 0x0030 | R/W | Modbus 슬레이브 주소 | 1–247                              | (신규: 기존은 고정)      |

| 0x0044 | R/W | 정전류 제어 활성화 | 0=수동, 1=자동(정전류)    | (신규: 추천)       |

### 4.2 동작 상태 플래그 (0x0005) 비트 정의

기존 프로토콜에서 분산되어 있던 Boolean 상태 읽기 명령들을 하나의 비트 필드 레지스터로 통합합니다.

| 비트  | 명칭          | 설명                    | 기존 프로토콜 대응      |
| ----- | ------------- | ----------------------- | ----------------------- |
| 0     | RUN           | 출력 동작 중 (Running)  | 출력 동작 상태 (0x0201) |
| 1     | SOFT_START    | 소프트 스타트 진행 중   | (신규)                  |
| 2     | FAULT         | 에러/이상 발생          | 에러 상태 (0x0204)      |
| 3     | MODBUS_ACTIVE | Modbus 통신 활성        | (신규)                  |
| 4     | LOCAL_LOCK    | 로컬 입력 금지 상태     | 로컬 입력 금지 (0x0200) |
| 5     | EXT_INPUT     | 외부 리모트 입력 ON     | 외부 입력 상태 (0x0202) |
| 6     | SWEEP_ACTIVE  | 스윕 동작 중            | 스윕 동작 상태 (0x0203) |
| 7     | PULSE_ACTIVE  | 펄스 모드 동작 중       | (신규)                  |
| 8     | TIMER_ACTIVE  | 타이머 카운트다운 중    | (신규: 추천)            |
| 9     | CC_ACTIVE     | 정전류 제어 동작 중     | (신규: 추천)            |
| 10    | PLL_LOCKED    | PLL 공진 추적 잠금 완료 | (신규: 추천)            |
| 11–15 | (예약)        | 향후 확장용 (0 고정)    | —                       |

> **💡 활용 팁**: `FC 0x03`으로 레지스터 0x0005 하나만 읽으면 장비의 전체적인 운전 상태를 한 번에 파악할 수 있습니다. 기존 프로토콜에서 각각 별도 명령(0x0201, 0x0202, 0x0203, 0x0204)으로 읽어야 했던 4가지 상태를 1회 통신으로 해결합니다.

### 5.3 Boolean 값 전환

기존 프로토콜에서 Boolean 값을 `True=0xFF00`, `False=0x0000`으로 표현하였으나, Modbus RTU Holding Register에서는:

| 기존          | Modbus RTU |
| ------------- | ---------- |
| True = 0xFF00 | 1 (0x0001) |
| False= 0x0000 | 0 (0x0000) |

> 이 방식은 표준 Modbus 레지스터 값 표현과 일치하며, 마스터 측에서 단순 정수 비교로 처리할 수 있어 코드가 간결해집니다.

## 6. 출력값 정규화 규칙

기존 프로토콜의 출력값 정규화(Fullscale=500) 방식을 그대로 유지합니다.

| 항목             | 값        | 설명                            |
| ---------------- | --------- | ------------------------------- |
| 통신상 출력 범위 | 0–500     | Fullscale 500으로 정규화        |
| 실제 출력 관계   | 비례 환산 | 표시 범위(0x0007) 기준으로 환산 |

**환산 공식:**

```
실제 표시값 = (레지스터값 / 500) × 출력표시범위(0x0007)
```

**환산 예시:**
| 출력 표시 범위(0x0007) | 레지스터값(0x0002) | 실제 표시값 |
| ---------------------- | ------------------ | ----------- |
| 1000 (W) | 500 | 1000 W |
| 1000 (W) | 100 | 200 W |
| 100 (%) | 250 | 50 % |

## 9. 기존 프로토콜과의 주요 차이 요약

| 항목           | 기존 (RS485_protocol.md)            | Modbus RTU (본 문서)                          |
| -------------- | ----------------------------------- | --------------------------------------------- |

| CRC 바이트순서 | (문서 미명시)                       | **CRC Low 먼저, CRC High 나중**               |

| 명령 주소공간  | 흩어진 2바이트 코드 (0x01xx~0x02xx) | **순차 16비트 레지스터 (0x0000~)**            |

| 다중 쓰기      | 불가 (1회 1개)                      | **FC 0x10으로 복수 동시 쓰기**                |
| 에러 응답      | 기능값 OR 0x80                      | 기능값 OR 0x80 + **예외코드 포함**            |
| 호환 도구      | 전용 소프트웨어 필요                | ModbusPoll, QModMaster 등 범용 도구 사용 가능 |

## 10. 마스터 구현 참고 코드 (C 언어)

### 10.1 단일 레지스터 읽기 (FC 0x03)

```c
/* 마스터 → 슬레이브: 주파수 설정값(0x0001) 읽기 요청 생성 */
void Modbus_BuildReadRequest(uint8_t *buf, uint8_t slave_addr,
                              uint16_t reg_addr, uint16_t reg_count)
{
    buf[0] = slave_addr;
    buf[1] = 0x03;                    /* FC: Read Holding Registers */
    buf[2] = (reg_addr >> 8) & 0xFF;  /* 시작 레지스터 주소 Hi */
    buf[3] =  reg_addr       & 0xFF;  /* 시작 레지스터 주소 Lo */
    buf[4] = (reg_count >> 8) & 0xFF; /* 레지스터 수 Hi */
    buf[5] =  reg_count      & 0xFF;  /* 레지스터 수 Lo */
    uint16_t crc = Modbus_CRC16(buf, 6);
    buf[6] = crc & 0xFF;             /* CRC Lo (먼저) */
    buf[7] = (crc >> 8) & 0xFF;      /* CRC Hi (나중) */
    /* 총 8바이트 전송 */
}
```

### 10.2 단일 레지스터 쓰기 (FC 0x06)

```c
/* 마스터 → 슬레이브: 출력 ON(0x0000=1) 쓰기 요청 생성 */
void Modbus_BuildWriteRequest(uint8_t *buf, uint8_t slave_addr,
                               uint16_t reg_addr, uint16_t value)
{
    buf[0] = slave_addr;
    buf[1] = 0x06;                    /* FC: Write Single Register */
    buf[2] = (reg_addr >> 8) & 0xFF;  /* 레지스터 주소 Hi */
    buf[3] =  reg_addr       & 0xFF;  /* 레지스터 주소 Lo */
    buf[4] = (value >> 8) & 0xFF;     /* 값 Hi */
    buf[5] =  value       & 0xFF;     /* 값 Lo */
    uint16_t crc = Modbus_CRC16(buf, 6);
    buf[6] = crc & 0xFF;             /* CRC Lo */
    buf[7] = (crc >> 8) & 0xFF;      /* CRC Hi */
    /* 총 8바이트 전송, 정상 응답도 동일한 8바이트 에코 */
}
```
