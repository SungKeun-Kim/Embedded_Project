# 개발 실행 계획 (Execution Plan)

> 에이전트가 "다음에 무엇을 작업해야 하는가"를 판단하기 위한 단계별 우선순위 문서.
> 각 Phase는 이전 단계가 **검증 완료**된 후 진행한다.

---

## Phase 1: 기반 동작 (Foundation)

**목표**: MCU 부팅, 클럭 설정, 기본 GPIO 동작 확인

| #   | 작업                     | 파일             | 검증 방법                          | 상태    |
| --- | ------------------------ | ---------------- | ---------------------------------- | ------- |
| 1.1 | HSI→PLL→170MHz 클럭 설정 | `system_clock.c` | SysTick 1ms 정확도 확인 (LED 토글) | ✅ 완료 |
| 1.2 | 전체 GPIO 초기화         | `gpio_init.c`    | LED(PC10) 점멸, 버튼(PC0~PC3) 읽기 | ✅ 완료 |
| 1.3 | SysTick ISR 구성         | `stm32g4xx_it.c` | HAL_Delay() 정상 동작              | ✅ 완료 |
| 1.4 | 디버그 UART 출력         | `main.c`         | USART1(PA9) 115200bps 메시지 출력  | ✅ 완료 |

## Phase 3: 초음파 PWM 출력 (Core Output)

**목표**: TIM1 센터-얼라인드 PWM, 상보 출력, 소프트 스타트

| #   | 작업                         | 파일                | 검증 방법                        | 상태      |
| --- | ---------------------------- | ------------------- | -------------------------------- | --------- |
| 3.1 | TIM1 PWM 초기화 (40kHz 기본) | `ultrasonic_pwm.c`  | 오실로스코프 PA8/PB13 파형 확인  | ✅ 완료   |
| 3.2 | 데드타임 설정 (BDTR)         | `ultrasonic_pwm.c`  | 상보 출력 간 데드타임 200~500ns  | ✅ 완료   |
| 3.3 | 주파수 가변 (20~168kHz)      | `ultrasonic_pwm.c`  | 메뉴에서 주파수 변경 → 파형 확인 | ✅ 완료   |
| 3.4 | 듀티비 가변                  | `ultrasonic_pwm.c`  | 메뉴/가변저항 → 듀티 변경 확인   | ✅ 완료   |
| 3.5 | 소프트 스타트 구현           | `ultrasonic_ctrl.c` | 0%→목표값 500ms 램프업           | ✅ 완료   |
| 3.6 | Start/Stop 토글 (BTN_OK)     | `ultrasonic_ctrl.c` | OK 버튼으로 출력 ON/OFF          | ✅ 완료   |
| 3.7 | 비상 정지                    | `ultrasonic_ctrl.c` | 비상 조건 시 즉시 MOE 클리어     | ⬜ 미착수 |
